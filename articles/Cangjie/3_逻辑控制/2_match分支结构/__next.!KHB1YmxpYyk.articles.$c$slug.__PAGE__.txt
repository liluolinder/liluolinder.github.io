## 3.2.1 什么是 match？

上一节我们学了 `if`，它可以处理"如果……否则……"的逻辑。但有时候条件分支多了，`if-else if` 一长串写起来很繁琐：

```Cangjie
let day = 3
if (day == 1) {
    println("周一")
} else if (day == 2) {
    println("周二")
} else if (day == 3) {
    println("周三")
} // ... 还要继续写下去
```

`match` 就是解决这个问题的——它让你能**根据一个值的不同情况，清晰地执行不同代码**。

> [!TIP]
> 如果你学过其他语言，`match` 相当于：
> - **C/Java** 的 `switch`
>
> 但仓颉的 `match` 更强大、更安全。

---

## 3.2.2 含匹配值的 match：基本用法

### 语法结构

```Cangjie
match (待匹配的值) {
    case 模式1 => 
        // 匹配模式1时执行的代码
    case 模式2 => 
        // 匹配模式2时执行的代码
    case _ => 
        // 以上都不匹配时执行（类似 if 的 else）
}
```

### 一个简单的例子

```Cangjie
main() {
    let x = 0
    match (x) {
        case 1 => 
            println("x = 1")
        case 0 => 
            println("x = 0")   // 匹配成功，输出这一行
        case _ => 
            println("x 不是 0 也不是 1")
    }
}
```

运行结果：

```Bash
x = 0
```

> [!TIP]
> 可以这样理解：
> - `match (x)` = "来检查一下 x 是什么"
> - `case 值 =>` = "如果 x 是这个值，就……"
> - `case _ =>` = "如果以上都不是（通配符），就……"（就像 `else`）

### 匹配成功后就退出

`match` 的一个关键特性：**一旦某个 `case` 匹配成功，执行完它的代码后，整个 `match` 就结束了**，不会继续检查后面的 `case`。你不需要写 `break`。

```Cangjie
main() {
    let x = 0
    match (x) {
        case 0 => 
            println("匹配到 0")  // ✅ 匹配成功
        case 0 => 
            println("又匹配到 0") // ❌ 不会执行，因为 match 已经结束了
        case _ => 
            println("默认")
    }
}
// 输出：匹配到 0
```

---

## 3.2.3 穷尽性检查：必须覆盖所有可能

`match` 有一个重要的安全机制：**你必须覆盖所有可能的取值**，否则编译会报错。

```Cangjie
func nonExhaustive(x: Int64) {
    match (x) {
        case 0 => 
            print("x = 0") 
        case 1 => 
            print("x = 1") 
        case 2 => 
            print("x = 2") 
    }  // ❌ 编译报错：x 是 Int64，还有无数种可能没覆盖
}
```

> [!WARNING]
> `Int64` 的取值范围是 -2⁶³ ~ 2⁶³-1，你不可能把每个值都写出来。所以 `match` **要求你必须有一个兜底分支**。

### 用 `_` 兜底

`_`（下划线）是**通配符**，可以匹配任何值，用来确保穷尽：

```Cangjie
func nonExhaustive(x: Int64) {
    match (x) {
        case 0 => 
            print("x = 0") 
        case 1 => 
            print("x = 1") 
        case 2 => 
            print("x = 2") 
        case _ => 
            print("其他值")   // ✅ 覆盖所有剩余情况
    }
}
```

---

## 3.2.4 多模式匹配：一个分支匹配多个值

可以用 `|` 将多个模式组合在同一个 `case` 中，只要匹配其中一个就算匹配成功：

```Cangjie
main() {
    let day = 6

    match (day) {
        case 1 | 2 | 3 | 4 | 5 => 
            println("工作日")
        case 6 | 7 => 
            println("周末")    // 匹配成功
        case _ => 
            println("无效的星期")
    }
}
```

运行结果：

```Bash
周末
```

---

## 3.2.5 无匹配值的 match

`match` 也可以**不匹配值**，只用来判断多个 `Bool` 条件。这种写法比 `if-else if` 更清晰：

```Cangjie
main() {
    let x = -1
    match {
        case x > 0 => println("x > 0")
        case x < 0 => println("x < 0")  // 匹配成功
        case _ => println("x = 0")
    }
}
```

> [!NOTE]
> 这种写法的特点：
> - `match` 后面**没有**括号和待匹配的值
> - `case` 后面直接跟 `Bool` 表达式（而不是模式）
> - 从上到下依次判断，遇到第一个 `true` 就执行对应分支

---

## 3.2.6 match 表达式的值与类型

和 `if` 一样，`match` 也是一个**表达式**，可以有值。

规则和 `if` 类似：

| 场景 | 规则 |
|------|------|
| match 被赋值给变量 | 类型由各分支的公共父类型决定 |
| match 独立使用（没被赋值） | 类型为 `Unit` |
| 各分支类型相同 | match 的类型就是该类型 |
| 各分支类型不兼容 | ❌ 编译报错 |

### 作为表达式使用

```Cangjie
main() {
    let x = 2
    let s = match (x) {
        case 0 => "x = 0"
        case 1 => "x = 1"
        case _ => "x 不是 0 也不是 1"  // 匹配成功
    }
    println(s)    // 输出：x 不是 0 也不是 1
}
```

这里每个分支都返回 `String`，所以 `match` 的类型是 `String`，`s` 的类型也是 `String`。

### 分支类型不兼容的情况

如果各分支返回的类型不兼容，编译会报错：

```Cangjie
main() {
    let x = 2
    let s = match (x) {
        case 0 => "x = 0"       // String
        case 1 => 1             // Int64
        case _ => "其他"         // String
    }  // ❌ 编译报错：String 和 Int64 不兼容
}
```

> [!NOTE]
> 和 `if` 一样，`match` 的所有分支必须返回**相同类型**或**有公共父类型**的值。`String` 和 `Int64` 之间没有公共父类型，所以编译报错。

### 独立使用（不关心返回值）

```Cangjie
main() {
    let x = 2
    match (x) {          // 这里的 match 类型是 Unit
        case 0 =>  
            println("零") 
        case 1 => 
            println("一") 
        case _ => 
            println("其他") 
    }
}
```

## 3.2.7 小结

| 概念 | 要点 |
|------|------|
| 基本语法 | `match (值) { case 模式 => ... }` |
| 穷尽性 | 必须覆盖所有可能的取值，否则编译报错 |
| 兜底分支 | 用 `case _ =>` 覆盖剩余所有情况 |
| 多模式 | 用 `|` 连接多个模式，如 `case 1 \| 2 =>` |
| 无匹配值 | `match { case 条件 => ... }` 用于判断多个 Bool 表达式 |
| 作为表达式 | 有值，类型由各分支的公共父类型决定 |

## 常见问题

### Q1：`match` 和 `if-else if` 怎么选？

| 场景 | 推荐 |
|------|------|
| 只有两个分支 | `if-else` |
| 判断一个值的多种具体取值 | `match` |
| 判断多个不同的条件 | `if-else if` 或无匹配值的 `match` |
| 需要模式匹配（如提取 enum 内部的值） | `match` |

简单原则：**分支多、判断的是同一个值，用 `match`；分支少、条件各不相同，用 `if`。**

### Q2：`_` 通配符只能放在最后吗？

**通常是的。** 因为 `match` 是按顺序匹配的，一旦匹配成功就结束。如果把 `_` 放在前面，它会匹配所有值，后面的分支就永远没机会执行了：

```Cangjie
match (x) {
    case _ => println("通配符")   // ❌ 会匹配所有值
    case 0 => println("零")        // 永远不会执行
}
```

编译器通常会对这种"不可达分支"给出警告。

### Q3：`match` 的分支里可以有多个语句吗？

可以。直接在 `=>` 后面写多行代码就行：

```Cangjie
match (x) {
    case 0 =>
        println("恭喜！")
        println("你选中了零")
    case _ =>
        println("其他")
}
```

### Q4：`match` 能匹配哪些类型？

`match` 可以匹配**任何类型**的值，包括：
- 基本类型（`Int64`、`String`、`Bool` 等）
- 枚举类型（`enum`）
- 自定义类型的值

## 3.2.1 什么是 match？

上一节我们学了 `if`，它可以处理"如果……否则……"的逻辑。但有时候条件分支多了，`if-else if` 一长串写起来很繁琐：

```Cangjie
let day = 3
if (day == 1) {
    println("周一")
} else if (day == 2) {
    println("周二")
} else if (day == 3) {
    println("周三")
} // ... 还要继续写下去
```

`match` 就是解决这个问题的——它让你能**根据一个值的不同情况，清晰地执行不同代码**。

> [!TIP]
> 如果你学过其他语言，`match` 相当于：
> - **C/Java** 的 `switch`
>
> 但仓颉的 `match` 更强大、更安全。

---

## 3.2.2 含匹配值的 match：基本用法

### 语法结构

```Cangjie

    case 模式1 => 
        // 匹配模式1时执行的代码
    case 模式2 => 
        // 匹配模式2时执行的代码
    case _ => 
        // 以上都不匹配时执行（类似 if 的 else）
}
```

### 一个简单的例子

```Cangjie

        case 1 => 
            println("x = 1")
        case 0 => 
            println("x = 0")   // 匹配成功，输出这一行

            println("x 不是 0 也不是 1")

> 可以这样理解：
> - `match (x)` = "来检查一下 x 是什么"
> - `case 值 =>` = "如果 x 是这个值，就……"
> - `case _ =>` = "如果以上都不是（通配符），就……"（就像 `else`）

`match` 的一个关键特性：**一旦某个 `case` 匹配成功，执行完它的代码后，整个 `match` 就结束了**，不会继续检查后面的 `case`。你不需要写 `break`。

        case _ => 
            println("默认")
    }

## 3.2.3 穷尽性检查：必须覆盖所有可能

`match` 有一个重要的安全机制：**你必须覆盖所有可能的取值**，否则编译会报错。

```Cangjie
func nonExhaustive(x: Int64) {
    match (x) {
        case 0 => 
            print("x = 0") 
        case 1 => 
            print("x = 1") 
        case 2 => 
            print("x = 2") 
    }  // ❌ 编译报错：x 是 Int64，还有无数种可能没覆盖

```

> [!WARNING]
> `Int64` 的取值范围是 -2⁶³ ~ 2⁶³-1，你不可能把每个值都写出来。所以 `match` **要求你必须有一个兜底分支**。

`_`（下划线）是**通配符**，可以匹配任何值，用来确保穷尽：

```Cangjie

            print("x = 0")

            print("x = 1")

            print("x = 2") 
        case _ => 
            print("其他值")   // ✅ 覆盖所有剩余情况
    }
}
```
