## 3.4.1 do-while 和 while 的区别

上一节我们学了 `while`——它**先判断条件**，条件成立才执行循环体。

`do-while` 正好反过来：**先执行一次循环体，再判断条件**。

> [!TIP]
> 就像两个不同的"吃饭"策略：
> - `while`：先看饿不饿 → 饿了才吃
> - `do-while`：先吃一口 → 吃完再看饿不饿，饿了继续吃

- **`while`**：可能一次都不执行（条件一开始就为 `false`）
- **`do-while`**：**至少执行一次**（先做再说）

---

## 3.4.2 do-while 的基本用法

```Cangjie
do {
    // 循环体（至少执行一次）
} while (条件)
```

执行流程：

1. 执行循环体
2. 判断条件 → 为 `true` 回到第 1 步，为 `false` 结束循环

### 最简单的例子

```Cangjie
main() {
    var count = 0

    do {
        println("第 ${count} 次循环")
        count++
    } while (count < 5)
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

### 和 while 对比：至少执行一次

下面的例子能明显看出区别：

```Cangjie
main() {
    var i = 10

    // while：先判断，条件为 false，一次都不执行
    while (i < 5) {
        println("while 执行了")     // 不输出
    }

    // do-while：先执行，再判断
    do {
        println("do-while 执行了")   // 输出一次
    } while (i < 5)
}
```

运行结果：

```Bash
do-while 执行了
```

> [!TIP]
> `while` 像是"先看菜单再决定吃不吃"，菜单不合口味就不吃；
> `do-while` 像是"先尝一口，不好吃再放下"——无论如何都会尝第一口。

---

## 3.4.3 实际例子：蒙特卡洛法求圆周率

下面这个程序用**蒙特卡洛方法**（随机采样）来近似计算圆周率：

```Cangjie
import std.random.Random

main() {
    let random = Random()
    var totalPoints = 0
    var hitPoints = 0

    do {
        // 在 ((0, 0), (1, 1)) 这个正方形中随机取点
        let x = random.nextFloat64()
        let y = random.nextFloat64()
        // 判断是否落在正方形内接圆里
        if ((x - 0.5) ** 2 + (y - 0.5) ** 2 < 0.25) {
            hitPoints++
        }
        totalPoints++
    } while (totalPoints < 1000000)

    let pi = 4.0 * Float64(hitPoints) / Float64(totalPoints)
    println("圆周率近似值为：${pi}")
}
```

运行结果（每次可能不同）：

```Bash
圆周率近似值为：3.141872
```

> [!NOTE]
> 因为用到了随机数，每次运行的结果可能不同，但都会约等于 3.14。采样次数越多（`totalPoints` 越大），结果越精确。
> 随机数方法此处不要求掌握，大致了解一下即可

这里用 `do-while` 很自然——循环体**至少需要执行一次**来采样，然后不断重复直到采集到足够多的样本。

---

## 3.4.4 条件必须是布尔值

和 `while`、`if` 一样，`do-while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x)         // ❌ 编译错误：不能把 Int64 当作条件
}
```

正确写法：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x > 0)     // ✅ 正确
}
```

---

## 3.4.5 do-while 的类型是 Unit

和 `while` 一样，`do-while` 表达式的类型也是 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = do {
        println(i)
        i++
    } while (i < 3)
    println(result)     // 输出：()
}
```

因为我们通常不关心循环的"返回值"，所以大多数时候直接独立使用。

---

## 3.4.6 while vs do-while 对比

| 对比项 | `while` | `do-while` |
|--------|---------|-----------|
| 执行时机 | 先判断，再执行 | 先执行，再判断 |
| 最少执行次数 | 0 次（条件可能一开始就 false） | **1 次**（先做再说） |
| 适用场景 | 可能不需要执行的情况 | **至少需要执行一次**的情况 |
| 语法 | `while (条件) { }` | `do { } while (条件)` |
| 类型 | `Unit` | `Unit` |

### 什么时候用 do-while？

`do-while` 最常见的场景是：**先执行一次初始化操作，然后根据结果决定是否继续**。比如：

- 游戏循环：先渲染一帧画面，再判断是否退出
- 用户输入：先让用户输入一次，再判断输入是否合法
- 蒙特卡洛模拟：先采样一次，再判断样本量是否足够

---

## 3.4.7 小结

| 概念 | 要点 |
|------|------|
| 基本语法 | `do { 循环体 } while (条件)` |
| 和 while 的区别 | `do-while` **至少执行一次**，`while` 可能零次 |
| 条件类型 | 必须是 `Bool` |
| 类型 | `Unit` |

## 常见问题

### Q1：`do-while` 能嵌套使用吗？

可以，和 `while` 一样支持嵌套：

```Cangjie
main() {
    var i = 0
    do {
        var j = 0
        do {
            print("(${i},${j}) ")
            j++
        } while (j < 3)
        println()
        i++
    } while (i < 3)
}
```

### Q2：既然 `while` 就能实现所有功能，为什么还需要 `do-while`？

`do-while` 可以让代码**更自然地表达"至少执行一次"的意图**。

如果不使用 `do-while`，你需要这样写：

```Cangjie
// 用 while 模拟 do-while
var i = 0
println("第一次执行")     // 手动先执行一次
i++
while (i < 5) {
    println("后续执行")
    i++
}
```

而用 `do-while` 更简洁：

```Cangjie
// 直接用 do-while
var i = 0
do {
    println("执行")
    i++
} while (i < 5)
```

**阅读代码时，看到 `do-while` 你立刻知道"这至少会执行一次"，这就是它的价值。**

### Q3：`do-while` 会进入死循环吗？

会。如果条件永远为 `true`，`do-while` 也会无限循环：

```Cangjie
do {
    println("停不下来")
} while (true)      // 死循环
```

## 3.4.1 do-while 和 while 的区别

上一节我们学了 `while`——它**先判断条件**，条件成立才执行循环体。

`do-while` 正好反过来：**先执行一次循环体，再判断条件**。

> [!TIP]
> 就像两个不同的"吃饭"策略：
> - `while`：先看饿不饿 → 饿了才吃
> - `do-while`：先吃一口 → 吃完再看饿不饿，饿了继续吃

- **`while`**：可能一次都不执行（条件一开始就为 `false`）
- **`do-while`**：**至少执行一次**（先做再说）

---

## 3.4.2 do-while 的基本用法

```Cangjie
do {
    // 循环体（至少执行一次）
} while (条件)
```

执行流程：

1. 执行循环体
2. 判断条件 → 为 `true` 回到第 1 步，为 `false` 结束循环

### 最简单的例子

```Cangjie
main() {
    var count = 0

    do {
        println("第 ${count} 次循环")
        count++
    } while (count < 5)
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

### 和 while 对比：至少执行一次

下面的例子能明显看出区别：

```Cangjie
main() {
    var i = 10

    // while：先判断，条件为 false，一次都不执行
    while (i < 5) {
        println("while 执行了")     // 不输出
    }

    // do-while：先执行，再判断
    do {
        println("do-while 执行了")   // 输出一次
    } while (i < 5)
}
```

运行结果：

```Bash
do-while 执行了
```

> [!TIP]
> `while` 像是"先看菜单再决定吃不吃"，菜单不合口味就不吃；
> `do-while` 像是"先尝一口，不好吃再放下"——无论如何都会尝第一口。

## 3.4.3 实际例子：蒙特卡洛法求圆周率

下面这个程序用**蒙特卡洛方法**（随机采样）来近似计算圆周率：

```Cangjie
import std.random.Random

main() {
    let random = Random()
    var totalPoints = 0
    var hitPoints = 0

    do {
        // 在 ((0, 0), (1, 1)) 这个正方形中随机取点
        let x = random.nextFloat64()
        let y = random.nextFloat64()
        // 判断是否落在正方形内接圆里
        if ((x - 0.5) ** 2 + (y - 0.5) ** 2 < 0.25) {
            hitPoints++
        }
        totalPoints++
    } while (totalPoints < 1000000)

    let pi = 4.0 * Float64(hitPoints) / Float64(totalPoints)
    println("圆周率近似值为：${pi}")
}
```

运行结果（每次可能不同）：

```Bash
圆周率近似值为：3.141872
```

> [!NOTE]
> 因为用到了随机数，每次运行的结果可能不同，但都会约等于 3.14。采样次数越多（`totalPoints` 越大），结果越精确。
> 随机数方法此处不要求掌握，大致了解一下即可

这里用 `do-while` 很自然——循环体**至少需要执行一次**来采样，然后不断重复直到采集到足够多的样本。

## 3.4.4 条件必须是布尔值

和 `while`、`if` 一样，`do-while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    var x = 5

        println(x)
        x--
    } while (x)         // ❌ 编译错误：不能把 Int64 当作条件

```

正确写法：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x > 0)     // ✅ 正确
}
```
