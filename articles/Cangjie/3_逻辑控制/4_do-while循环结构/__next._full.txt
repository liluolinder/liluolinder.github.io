## 3.4.1 do-while 和 while 的区别

上一节我们学了 `while`——它**先判断条件**，条件成立才执行循环体。

`do-while` 正好反过来：**先执行一次循环体，再判断条件**。

> [!TIP]
> 就像两个不同的"吃饭"策略：
> - `while`：先看饿不饿 → 饿了才吃（**可能一次都不吃**）
> - `do-while`：先吃一口 → 吃完再看饿不饿，饿了继续吃（**至少吃一口**）

### 核心区别

| 循环 | 最小执行次数 | 执行顺序 |
|------|------------|---------|
| `while` | **0 次**（条件一开始为 false 就不执行） | 先判断，后执行 |
| `do-while` | **1 次**（无论如何都会执行一次） | 先执行，后判断 |

### 直观对比

```Cangjie
main() {
    var i = 10

    // while：先判断条件，10 < 5 为 false，一次都不执行
    while (i < 5) {
        println("while 执行了")     // 不输出
    }

    // do-while：先执行一次，再判断条件
    do {
        println("do-while 执行了")   // 输出一次
    } while (i < 5)                 // 10 < 5 为 false，结束
}
```

运行结果：

```Bash
do-while 执行了
```

> [!NOTE]
> 这个例子完美展示了区别：`i = 10` 一开始就 `> 5`，`while` 直接跳过，而 `do-while` 仍然会执行一次。

---

## 3.4.2 do-while 的基本用法

```Cangjie
do {
    // 循环体（至少执行一次）
} while (条件)
```

执行流程：

```
      ┌─ 执行循环体（第一次）
      │
      ▼
  判断条件 ── true ──→ 再次执行循环体
      │
     false
      ▼
   退出循环
```

> [!TIP]
> 注意 `do-while` 末尾的 `while (条件)` 后面需要**加分号** `;`！这是仓颉的语法要求，和 `while` 不一样。

### 最简单的例子

```Cangjie
main() {
    var count = 0

    do {
        println("第 ${count} 次循环")
        count++
    } while (count < 5)
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

**逐轮过程**（注意和 while 的区别——先执行再判断）：

```
第 1 轮：执行循环体（count=0，输出）→ 判断 0 < 5？→ true，继续
第 2 轮：执行循环体（count=1，输出）→ 判断 1 < 5？→ true，继续
第 3 轮：执行循环体（count=2，输出）→ 判断 2 < 5？→ true，继续
第 4 轮：执行循环体（count=3，输出）→ 判断 3 < 5？→ true，继续
第 5 轮：执行循环体（count=4，输出）→ 判断 4 < 5？→ true，继续
第 6 轮：执行循环体（count=5，输出）→ 判断 5 < 5？→ false，结束
```

### 常见的陷阱：忘记写 `;`

```Cangjie
main() {
    var i = 0
    do {
        println(i)
        i++
    } while (i < 5)    // ❌ 这里没写分号会编译报错！
}
```

> [!CAUTION]
> 和 `while` 不同，`do-while` 的 `while (条件)` 后面必须跟一个**分号 `;`**。
>
> 初学者最容易忘记这个分号！

## 3.4.3 实际例子：蒙特卡洛法求圆周率

下面这个程序用**蒙特卡洛方法**（随机采样）来近似计算圆周率：

```Cangjie
import std.random.Random

main() {
    let random = Random()
    var totalPoints = 0
    var hitPoints = 0

    do {
        // 在 ((0, 0), (1, 1)) 这个正方形中随机取点
        let x = random.nextFloat64()
        let y = random.nextFloat64()
        // 判断是否落在正方形内接圆里
        if ((x - 0.5) ** 2 + (y - 0.5) ** 2 < 0.25) {
            hitPoints++
        }
        totalPoints++
    } while (totalPoints < 1000000)

    let pi = 4.0 * Float64(hitPoints) / Float64(totalPoints)
    println("圆周率近似值为：${pi}")
}
```

运行结果（每次可能不同）：

```Bash
圆周率近似值为：3.141872
```

> [!NOTE]
> 因为用到了随机数，每次运行的结果可能不同，但都会约等于 3.14。采样次数越多（`totalPoints` 越大），结果越精确。
> 随机数方法此处不要求掌握，大致了解一下即可。

这里用 `do-while` 很自然——循环体**至少需要执行一次**来采样，然后不断重复直到采集到足够多的样本。

## 3.4.4 条件必须是布尔值

和 `while`、`if` 一样，`do-while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x)         // ❌ 编译错误：不能把 Int64 当作条件
}
```

正确写法：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x > 0)     // ✅ 正确
}
```

---

## 3.4.5 do-while 的类型是 Unit

和 `while` 一样，`do-while` 表达式的类型也是 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = do {
        println(i)
        i++
    } while (i < 3)
    println(result)     // 输出：()
}
```

因为我们通常不关心循环的"返回值"，所以大多数时候直接独立使用。

---

## 3.4.6 while vs do-while 对比

| 对比项 | `while` | `do-while` |
|--------|---------|-----------|
| 执行时机 | 先判断，再执行 | 先执行，再判断 |
| 最少执行次数 | 0 次（条件可能一开始就 false） | **1 次**（先做再说） |
| 语法 | `while (条件) { }` | `do { } while (条件)` |
| 末尾分号 | 不需要 | **需要** `;` |
| 类型 | `Unit` | `Unit` |
| 直观理解 | "先看菜单再决定吃不吃" | "先尝一口，不好吃再放下" |

### 什么时候用 do-while？

`do-while` 最常见的场景是：**先执行一次初始化操作，然后根据结果决定是否继续**。

| 场景 | 说明 |
|------|------|
| 游戏循环 | 先渲染一帧画面，再判断是否退出 |
| 用户输入 | 先让用户输入一次，再判断输入是否合法 |
| 蒙特卡洛模拟 | 先采样一次，再判断样本量是否足够 |
| 菜单系统 | 先显示菜单，让用户选择一次，再决定是否退出 |

```Cangjie
// 一个简单的"猜数字"游戏框架
let secret = 42
var guess: Int64
do {
    println("请输入你猜的数字：")
    guess = getInput()      // 先让用户输入一次
} while (guess != secret)   // 猜对了才退出
println("恭喜你猜对了！")
```

这里用 `do-while` 很优雅——**无论如何先让用户猜一次**，如果猜对了就结束，猜错了继续猜。

## 3.4.7 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本语法 | `do { 循环体 } while (条件)` | **别忘了末尾分号** |
| 和 while 的区别 | `do-while` **至少执行一次**，`while` 可能零次 | 先做再说 |
| 适用场景 | 至少需要执行一次的情况 | 先初始化，再判断是否继续 |
| 条件类型 | 必须是 `Bool` | 和其他分支条件一样 |
| 类型 | `Unit` | 一般不赋值使用 |

## 常见问题

### Q1：`do-while` 能嵌套使用吗？

可以，和 `while` 一样支持嵌套：

```Cangjie
main() {
    var i = 0
    do {
        var j = 0
        do {
            print("(${i},${j}) ")
            j++
        } while (j < 3)
        println()
        i++
    } while (i < 3)
}
```

### Q2：既然 `while` 就能实现所有功能，为什么还需要 `do-while`？

`do-while` 让代码**更自然地表达"至少执行一次"的意图**。

如果不使用 `do-while`，你需要这样写：

```Cangjie
// 用 while 模拟 do-while：需要手动先执行一次
var i = 0
println("第一次执行")     // 手动先执行一次
i++
while (i < 5) {
    println("后续执行")
    i++
}
```

而用 `do-while` 更简洁：

```Cangjie
// 直接用 do-while
var i = 0
do {
    println("执行")
    i++
} while (i < 5)
```

**阅读代码时，看到 `do-while` 你立刻知道"这至少会执行一次"，这就是它的价值。**

### Q3：`do-while` 会进入死循环吗？

会。如果条件永远为 `true`，`do-while` 也会无限循环：

```Cangjie
do {
    println("停不下来")
} while (true)      // 死循环
```

### Q4：`do-while` 和 `while` 怎么选？

- 如果逻辑是"先判断，有必要才做" → `while`（比如：先检查余额，够才取钱）
- 如果逻辑是"先做一次，然后看要不要继续" → `do-while`（比如：先吃一口，再看饿不饿）

## 3.4.1 do-while 和 while 的区别

上一节我们学了 `while`——它**先判断条件**，条件成立才执行循环体。

`do-while` 正好反过来：**先执行一次循环体，再判断条件**。

> [!TIP]
> 就像两个不同的"吃饭"策略：
> - `while`：先看饿不饿 → 饿了才吃（**可能一次都不吃**）
> - `do-while`：先吃一口 → 吃完再看饿不饿，饿了继续吃（**至少吃一口**）

### 核心区别

| 循环 | 最小执行次数 | 执行顺序 |
|------|------------|---------|
| `while` | **0 次**（条件一开始为 false 就不执行） | 先判断，后执行 |
| `do-while` | **1 次**（无论如何都会执行一次） | 先执行，后判断 |

### 直观对比

```Cangjie
main() {
    var i = 10

    // while：先判断条件，10 < 5 为 false，一次都不执行
    while (i < 5) {
        println("while 执行了")     // 不输出
    }

    // do-while：先执行一次，再判断条件
    do {
        println("do-while 执行了")   // 输出一次
    } while (i < 5)                 // 10 < 5 为 false，结束
}
```

运行结果：

```Bash
do-while 执行了
```

> [!NOTE]
> 这个例子完美展示了区别：`i = 10` 一开始就 `> 5`，`while` 直接跳过，而 `do-while` 仍然会执行一次。

---

## 3.4.2 do-while 的基本用法

```Cangjie
do {
    // 循环体（至少执行一次）
} while (条件)
```

执行流程：

```
      ┌─ 执行循环体（第一次）
      │
      ▼
  判断条件 ── true ──→ 再次执行循环体
      │
     false
      ▼
   退出循环
```

> [!TIP]
> 注意 `do-while` 末尾的 `while (条件)` 后面需要**加分号** `;`！这是仓颉的语法要求，和 `while` 不一样。

### 最简单的例子

```Cangjie
main() {
    var count = 0

    do {
        println("第 ${count} 次循环")
        count++
    } while (count < 5)
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

**逐轮过程**（注意和 while 的区别——先执行再判断）：

```
第 1 轮：执行循环体（count=0，输出）→ 判断 0 < 5？→ true，继续
第 2 轮：执行循环体（count=1，输出）→ 判断 1 < 5？→ true，继续
第 3 轮：执行循环体（count=2，输出）→ 判断 2 < 5？→ true，继续
第 4 轮：执行循环体（count=3，输出）→ 判断 3 < 5？→ true，继续
第 5 轮：执行循环体（count=4，输出）→ 判断 4 < 5？→ true，继续
第 6 轮：执行循环体（count=5，输出）→ 判断 5 < 5？→ false，结束
```

### 常见的陷阱：忘记写 `;`

```Cangjie
main() {
    var i = 0
    do {
        println(i)
        i++
    } while (i < 5)    // ❌ 这里没写分号会编译报错！
}
```

> [!CAUTION]
> 和 `while` 不同，`do-while` 的 `while (条件)` 后面必须跟一个**分号 `;`**。
>
> 初学者最容易忘记这个分号！

## 3.4.3 实际例子：蒙特卡洛法求圆周率

下面这个程序用**蒙特卡洛方法**（随机采样）来近似计算圆周率：

```Cangjie
import std.random.Random

main() {
    let random = Random()
    var totalPoints = 0
    var hitPoints = 0

    do {
        // 在 ((0, 0), (1, 1)) 这个正方形中随机取点
        let x = random.nextFloat64()
        let y = random.nextFloat64()
        // 判断是否落在正方形内接圆里
        if ((x - 0.5) ** 2 + (y - 0.5) ** 2 < 0.25) {
            hitPoints++
        }
        totalPoints++
    } while (totalPoints < 1000000)

    let pi = 4.0 * Float64(hitPoints) / Float64(totalPoints)
    println("圆周率近似值为：${pi}")
}
```

运行结果（每次可能不同）：

```Bash
圆周率近似值为：3.141872
```

> [!NOTE]
> 因为用到了随机数，每次运行的结果可能不同，但都会约等于 3.14。采样次数越多（`totalPoints` 越大），结果越精确。
> 随机数方法此处不要求掌握，大致了解一下即可。

这里用 `do-while` 很自然——循环体**至少需要执行一次**来采样，然后不断重复直到采集到足够多的样本。

## 3.4.4 条件必须是布尔值

和 `while`、`if` 一样，`do-while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x)         // ❌ 编译错误：不能把 Int64 当作条件
}
```

正确写法：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x > 0)     // ✅ 正确
}
```

---

## 3.4.5 do-while 的类型是 Unit

和 `while` 一样，`do-while` 表达式的类型也是 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = do {
        println(i)
        i++
    } while (i < 3)
    println(result)     // 输出：()
}
```

因为我们通常不关心循环的"返回值"，所以大多数时候直接独立使用。

## 3.4.6 while vs do-while 对比

| 对比项 | `while` | `do-while` |
|--------|---------|-----------|
| 执行时机 | 先判断，再执行 | 先执行，再判断 |
| 最少执行次数 | 0 次（条件可能一开始就 false） | **1 次**（先做再说） |
| 语法 | `while (条件) { }` | `do { } while (条件)` |
| 末尾分号 | 不需要 | **需要** `;` |
| 类型 | `Unit` | `Unit` |
| 直观理解 | "先看菜单再决定吃不吃" | "先尝一口，不好吃再放下" |

### 什么时候用 do-while？

`do-while` 最常见的场景是：**先执行一次初始化操作，然后根据结果决定是否继续**。

| 场景 | 说明 |
|------|------|
| 游戏循环 | 先渲染一帧画面，再判断是否退出 |
| 用户输入 | 先让用户输入一次，再判断输入是否合法 |
| 蒙特卡洛模拟 | 先采样一次，再判断样本量是否足够 |
| 菜单系统 | 先显示菜单，让用户选择一次，再决定是否退出 |

```Cangjie
// 一个简单的"猜数字"游戏框架
let secret = 42
var guess: Int64
do {
    println("请输入你猜的数字：")
    guess = getInput()      // 先让用户输入一次
} while (guess != secret)   // 猜对了才退出
println("恭喜你猜对了！")
```

这里用 `do-while` 很优雅——**无论如何先让用户猜一次**，如果猜对了就结束，猜错了继续猜。

## 3.4.7 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本语法 | `do { 循环体 } while (条件)` | **别忘了末尾分号** |
| 和 while 的区别 | `do-while` **至少执行一次**，`while` 可能零次 | 先做再说 |
| 适用场景 | 至少需要执行一次的情况 | 先初始化，再判断是否继续 |
| 条件类型 | 必须是 `Bool` | 和其他分支条件一样 |
| 类型 | `Unit` | 一般不赋值使用 |

## 常见问题

### Q1：`do-while` 能嵌套使用吗？

可以，和 `while` 一样支持嵌套：

```Cangjie
main() {
    var i = 0
    do {
        var j = 0
        do {
            print("(${i},${j}) ")
            j++
        } while (j < 3)
        println()
        i++
    } while (i < 3)
}
```

### Q2：既然 `while` 就能实现所有功能，为什么还需要 `do-while`？

`do-while` 让代码**更自然地表达"至少执行一次"的意图**。

如果不使用 `do-while`，你需要这样写：

```Cangjie
// 用 while 模拟 do-while：需要手动先执行一次
var i = 0
println("第一次执行")     // 手动先执行一次
i++
while (i < 5) {
    println("后续执行")
    i++
}
```

而用 `do-while` 更简洁：

```Cangjie
// 直接用 do-while
var i = 0
do {
    println("执行")
    i++
} while (i < 5)
```

**阅读代码时，看到 `do-while` 你立刻知道"这至少会执行一次"，这就是它的价值。**

### Q3：`do-while` 会进入死循环吗？

会。如果条件永远为 `true`，`do-while` 也会无限循环：

```Cangjie
do {
    println("停不下来")
} while (true)      // 死循环
```

### Q4：`do-while` 和 `while` 怎么选？

- 如果逻辑是"先判断，有必要才做" → `while`（比如：先检查余额，够才取钱）
- 如果逻辑是"先做一次，然后看要不要继续" → `do-while`（比如：先吃一口，再看饿不饿）

## 3.4.1 do-while 和 while 的区别

上一节我们学了 `while`——它**先判断条件**，条件成立才执行循环体。

`do-while` 正好反过来：**先执行一次循环体，再判断条件**。

> [!TIP]
> 就像两个不同的"吃饭"策略：
> - `while`：先看饿不饿 → 饿了才吃（**可能一次都不吃**）
> - `do-while`：先吃一口 → 吃完再看饿不饿，饿了继续吃（**至少吃一口**）

### 核心区别

| 循环 | 最小执行次数 | 执行顺序 |
|------|------------|---------|
| `while` | **0 次**（条件一开始为 false 就不执行） | 先判断，后执行 |
| `do-while` | **1 次**（无论如何都会执行一次） | 先执行，后判断 |

### 直观对比

```Cangjie
main() {
    var i = 10

    // while：先判断条件，10 < 5 为 false，一次都不执行
    while (i < 5) {
        println("while 执行了")     // 不输出
    }

    // do-while：先执行一次，再判断条件
    do {
        println("do-while 执行了")   // 输出一次
    } while (i < 5)                 // 10 < 5 为 false，结束
}
```

运行结果：

```Bash
do-while 执行了
```

> [!NOTE]
> 这个例子完美展示了区别：`i = 10` 一开始就 `> 5`，`while` 直接跳过，而 `do-while` 仍然会执行一次。

---

## 3.4.2 do-while 的基本用法

```Cangjie
do {
    // 循环体（至少执行一次）
} while (条件)
```

执行流程：

```
      ┌─ 执行循环体（第一次）
      │
      ▼
  判断条件 ── true ──→ 再次执行循环体
      │
     false
      ▼
   退出循环
```

> [!TIP]
> 注意 `do-while` 末尾的 `while (条件)` 后面需要**加分号** `;`！这是仓颉的语法要求，和 `while` 不一样。

### 最简单的例子

```Cangjie
main() {
    var count = 0

    do {
        println("第 ${count} 次循环")
        count++
    } while (count < 5)
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

**逐轮过程**（注意和 while 的区别——先执行再判断）：

```
第 1 轮：执行循环体（count=0，输出）→ 判断 0 < 5？→ true，继续
第 2 轮：执行循环体（count=1，输出）→ 判断 1 < 5？→ true，继续
第 3 轮：执行循环体（count=2，输出）→ 判断 2 < 5？→ true，继续
第 4 轮：执行循环体（count=3，输出）→ 判断 3 < 5？→ true，继续
第 5 轮：执行循环体（count=4，输出）→ 判断 4 < 5？→ true，继续
第 6 轮：执行循环体（count=5，输出）→ 判断 5 < 5？→ false，结束
```

### 常见的陷阱：忘记写 `;`

```Cangjie
main() {
    var i = 0
    do {
        println(i)
        i++
    } while (i < 5)    // ❌ 这里没写分号会编译报错！
}
```

> [!CAUTION]
> 和 `while` 不同，`do-while` 的 `while (条件)` 后面必须跟一个**分号 `;`**。
>
> 初学者最容易忘记这个分号！

## 3.4.3 实际例子：蒙特卡洛法求圆周率

下面这个程序用**蒙特卡洛方法**（随机采样）来近似计算圆周率：

```Cangjie
import std.random.Random

main() {
    let random = Random()
    var totalPoints = 0
    var hitPoints = 0

    do {
        // 在 ((0, 0), (1, 1)) 这个正方形中随机取点
        let x = random.nextFloat64()
        let y = random.nextFloat64()
        // 判断是否落在正方形内接圆里
        if ((x - 0.5) ** 2 + (y - 0.5) ** 2 < 0.25) {
            hitPoints++
        }
        totalPoints++
    } while (totalPoints < 1000000)

    let pi = 4.0 * Float64(hitPoints) / Float64(totalPoints)
    println("圆周率近似值为：${pi}")
}
```

运行结果（每次可能不同）：

```Bash
圆周率近似值为：3.141872
```

> [!NOTE]
> 因为用到了随机数，每次运行的结果可能不同，但都会约等于 3.14。采样次数越多（`totalPoints` 越大），结果越精确。
> 随机数方法此处不要求掌握，大致了解一下即可。

这里用 `do-while` 很自然——循环体**至少需要执行一次**来采样，然后不断重复直到采集到足够多的样本。

---

## 3.4.4 条件必须是布尔值

和 `while`、`if` 一样，`do-while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x)         // ❌ 编译错误：不能把 Int64 当作条件
}
```

正确写法：

```Cangjie
main() {
    var x = 5
    do {
        println(x)
        x--
    } while (x > 0)     // ✅ 正确
}
```

---

## 3.4.5 do-while 的类型是 Unit

和 `while` 一样，`do-while` 表达式的类型也是 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = do {
        println(i)
        i++
    } while (i < 3)
    println(result)     // 输出：()
}
```

因为我们通常不关心循环的"返回值"，所以大多数时候直接独立使用。

---

## 3.4.6 while vs do-while 对比

| 对比项 | `while` | `do-while` |
|--------|---------|-----------|
| 执行时机 | 先判断，再执行 | 先执行，再判断 |
| 最少执行次数 | 0 次（条件可能一开始就 false） | **1 次**（先做再说） |
| 语法 | `while (条件) { }` | `do { } while (条件)` |
| 末尾分号 | 不需要 | **需要** `;` |
| 类型 | `Unit` | `Unit` |
| 直观理解 | "先看菜单再决定吃不吃" | "先尝一口，不好吃再放下" |

### 什么时候用 do-while？

`do-while` 最常见的场景是：**先执行一次初始化操作，然后根据结果决定是否继续**。

| 场景 | 说明 |
|------|------|
| 游戏循环 | 先渲染一帧画面，再判断是否退出 |
| 用户输入 | 先让用户输入一次，再判断输入是否合法 |
| 蒙特卡洛模拟 | 先采样一次，再判断样本量是否足够 |
| 菜单系统 | 先显示菜单，让用户选择一次，再决定是否退出 |

```Cangjie
// 一个简单的"猜数字"游戏框架
let secret = 42
var guess: Int64
do {
    println("请输入你猜的数字：")
    guess = getInput()      // 先让用户输入一次
} while (guess != secret)   // 猜对了才退出
println("恭喜你猜对了！")
```

这里用 `do-while` 很优雅——**无论如何先让用户猜一次**，如果猜对了就结束，猜错了继续猜。

## 3.4.7 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本语法 | `do { 循环体 } while (条件)` | **别忘了末尾分号** |
| 和 while 的区别 | `do-while` **至少执行一次**，`while` 可能零次 | 先做再说 |
| 适用场景 | 至少需要执行一次的情况 | 先初始化，再判断是否继续 |
| 条件类型 | 必须是 `Bool` | 和其他分支条件一样 |
| 类型 | `Unit` | 一般不赋值使用 |

## 常见问题

### Q1：`do-while` 能嵌套使用吗？

可以，和 `while` 一样支持嵌套：

```Cangjie
main() {
    var i = 0
    do {
        var j = 0
        do {
            print("(${i},${j}) ")
            j++
        } while (j < 3)
        println()
        i++
    } while (i < 3)
}
```

### Q2：既然 `while` 就能实现所有功能，为什么还需要 `do-while`？

`do-while` 让代码**更自然地表达"至少执行一次"的意图**。

如果不使用 `do-while`，你需要这样写：

```Cangjie
// 用 while 模拟 do-while：需要手动先执行一次
var i = 0
println("第一次执行")     // 手动先执行一次
i++
while (i < 5) {
    println("后续执行")
    i++
}
```

而用 `do-while` 更简洁：

```Cangjie
// 直接用 do-while
var i = 0
do {
    println("执行")
    i++
} while (i < 5)
```

**阅读代码时，看到 `do-while` 你立刻知道"这至少会执行一次"，这就是它的价值。**

### Q3：`do-while` 会进入死循环吗？

会。如果条件永远为 `true`，`do-while` 也会无限循环：

```Cangjie
do {
    println("停不下来")
} while (true)      // 死循环
```

### Q4：`do-while` 和 `while` 怎么选？

- 如果逻辑是"先判断，有必要才做" → `while`（比如：先检查余额，够才取钱）
- 如果逻辑是"先做一次，然后看要不要继续" → `do-while`（比如：先吃一口，再看饿不饿）
