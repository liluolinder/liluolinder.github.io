## 3.1.1 什么是分支结构？

程序并不总是按顺序一路执行到底——很多时候，我们需要根据**不同的条件**执行**不同的代码**。就像生活中：

> 如果今天下雨，就带伞；否则就不带。

在编程中，这种"如果……否则……"的逻辑就叫做**分支结构**，而 `if` 就是实现它的工具。

---

## 3.1.2 if 的基本用法

`if` 表达式的结构很简单：

```Cangjie
if (条件1) {
    // 条件1成立时执行的代码
} else if(条件2){
    // 条件1不满足 条件2满足时候执行
}else {
    // 条件不成立时执行的代码
}
```

> [!TIP]
> 可以这样理解：
> - `if` = "如果……"
> - `else if` = "再如果"
> - `else` = "否则……"
> - 条件用圆括号 `()` 包裹
> - 要执行的代码用花括号 `{}` 包裹

### 最简单的例子

```Cangjie
main() {
    let score = 85

    if (score >= 60) {
        println("及格了！")
    } else {
        println("不及格，继续加油💪")
    }
}
```

运行结果：

```Bash
及格了！
```

### else 可以省略

如果只关心"条件成立时"做什么，`else` 可以不写：

```Cangjie
main() {
    let isRaining = true

    if (isRaining) {
        println("记得带伞！")
    }
    // 没下雨就什么都不做
}
```

### 多级判断：else if

当有多个条件需要依次判断时，可以在 `else` 后面再接一个 `if`：

```Cangjie
import std.random.Random

main() {
    let speed = Random().nextFloat64() * 20.0
    println("${speed} km/s")

    if (speed > 16.7) {
        println("第三宇宙速度，鹊桥相会")
    } else if (speed > 11.2) {
        println("第二宇宙速度，嫦娥奔月")
    } else if (speed > 7.9) {
        println("第一宇宙速度，腾云驾雾")
    } else {
        println("脚踏实地，仰望星空")
    }
}
```

> [!TIP]
> 这样一层层判断，就像流水线筛选：
> 1️⃣ 速度 > 16.7？→ 第三宇宙速度
> 2️⃣ 否则，速度 > 11.2？→ 第二宇宙速度
> 3️⃣ 否则，速度 > 7.9？→ 第一宇宙速度
> 4️⃣ 否则 → 达不到宇宙速度

---

## 3.1.3 条件必须是布尔值

仓颉是**强类型**语言——`if` 的条件**必须**是 `Bool` 类型（`true` 或 `false`），不能拿数字代替。

```Cangjie
main() {
    let number = 1
    if (number) {       // ❌ 编译错误：不能把 Int64 当作条件
        println("非零数")
    }
}
```

> [!NOTE]
> 有些语言（如 C 语言）允许把 `0` 当作 `false`、非 `0` 当作 `true`，但仓颉**不允许**。你必须写出明确的布尔表达式，比如 `number != 0`。

```Cangjie
main() {
    let number = 1
    if (number != 0) {  // ✅ 正确：用比较运算得到 Bool 值
        println("非零数")
    }
}
```

---

## 3.1.4 if 表达式的值与类型

`if` 不仅仅是一个"判断语句"，它本身也是一个**表达式**——可以有值。这一点和很多其他语言不同，需要特别注意。

### 情况一：if 作为表达式被使用

当你把 `if` 的结果赋值给变量时，`if` 就是**有值**的——它的值是所执行分支中**最后一行表达式**的值，类型也是那个表达式的类型：

```Cangjie
main() {
    let voltage = 5.0
    let bit = if (voltage < 2.5) {
        0
    } else {
        1
    }
    println(bit)        // 输出：1
}
```

这里的 `if` 表达式会返回一个值：条件成立时返回 `0`，不成立时返回 `1`。

**类型推断规则：**

| 场景 | 规则 | 示例 |
|------|------|------|
| 两个分支类型相同 | `if` 的类型就是该类型 | `{ 0 }` / `{ 1 }` → `Int64` |
| 两个分支有公共父类型 | `if` 的类型取公共父类型 | 见下文 |
| 两个分支类型不兼容 | ❌ 编译报错 | `{ "a" }` / `{ 1 }` → 报错 |

两个分支类型不兼容的情况：

```Cangjie
main() {
    let a = 10
    let b = if (a == 10) {   // ❌ 编译报错：String 和 Int64 不兼容
        "this is 10"
    } else {
        20
    }
}
```

### 情况二：if 没有被赋值

当 `if` 只是独立使用、没有被赋值时，它的类型是 `Unit`，值为 `()`：

```Cangjie
main() {
    let a = 10
    if (a == 10) {
        println("是10")
    } else {
        println("不是10")
    }
    // 这里 if 的类型是 Unit，我们不关心它的值
}
```

绝大多数初学者遇到的都是这种情况——只是让 `if` 执行不同的操作，不关心返回值。

### 情况三：没有 else 分支

如果 `if` 没有 `else` 分支，那它的类型一定是 `Unit`：

```Cangjie
main() {
    let score = 85
    let result = if (score >= 60) {   // ❌ 编译报错？其实不会
        "及格"
    }
    // 但 result 的类型是 Unit，值是 ()
}
```

> [!WARNING]
> 没有 `else` 的 `if` 类型固定为 `Unit`！如果你想把 `if` 的结果存到变量里，**必须**有 `else` 分支。

---

## 3.1.5 小结

| 概念 | 要点 |
|------|------|
| 基本结构 | `if (条件) { ... } else { ... }` |
| 条件类型 | 必须是 `Bool`（不能是数字） |
| else 可省略 | 是，如果只关心"成立时" |
| 多级判断 | `else if` 可以连续使用 |
| if 作为表达式 | 有值，类型由分支共同决定 |
| 没有 else 的 if | 类型固定为 `Unit` |

---

## 常见问题

### Q1：`if` 的条件可以不加括号吗？

**不可以。** 仓颉要求 `if` 的条件必须用圆括号 `()` 包裹：

```Cangjie
if (score >= 60) {    // ✅ 正确
    println("及格")
}

if score >= 60 {      // ❌ 报错：缺少括号
    println("及格")
}
```

### Q2：多级 `if` 嵌套太深怎么办？

当 `if-else if` 层级太多时，代码会变得难以阅读。这种情况可以考虑后续章节会讲到的 `match` 表达式（类似其他语言的 `switch`）。

### Q3：`if` 分支里可以有多个语句吗？

可以。花括号 `{}` 里可以写任意多行代码，按照顺序依次执行：

```Cangjie
if (score >= 60) {
    println("恭喜！")
    println("你及格了！")
    let bonus = score - 60
    println("超出及格线 ${bonus} 分")
}
```

### Q4：`else if` 和 `else { if }` 有区别吗？

**效果一样**，但 `else if` 写法更简洁。下面两种写法等价：

```Cangjie
// 写法一：else if（推荐）
if (a > 0) {
    println("正数")
} else if (a < 0) {
    println("负数")
} else {
    println("零")
}

// 写法二：else 里面再套 if（不推荐）
if (a > 0) {
    println("正数")
} else {
    if (a < 0) {
        println("负数")
    } else {
        println("零")
    }
}
```

推荐使用 `else if`，代码更清晰。

## 3.1.2 if 的基本用法

```Cangjie

    // 条件1成立时执行的代码
} else if(条件2){
    // 条件1不满足 条件2满足时候执行
}else {
    // 条件不成立时执行的代码
}
```

> 可以这样理解：
> - `if` = "如果……"
> - `else if` = "再如果"
> - `else` = "否则……"
> - 条件用圆括号 `()` 包裹

```Cangjie
main() {

    if (score >= 60) {

    }
}
```

```Bash

```

## 3.1.3 条件必须是布尔值

仓颉是**强类型**语言——`if` 的条件**必须**是 `Bool` 类型（`true` 或 `false`），不能拿数字代替。

```Cangjie
main() {
    let number = 1
    if (number) {       // ❌ 编译错误：不能把 Int64 当作条件
        println("非零数")
    }
}
```

> [!NOTE]
> 有些语言（如 C 语言）允许把 `0` 当作 `false`、非 `0` 当作 `true`，但仓颉**不允许**。你必须写出明确的布尔表达式，比如 `number != 0`。

```Cangjie
main() {
    let number = 1
    if (number != 0) {  // ✅ 正确：用比较运算得到 Bool 值
        println("非零数")
    }
}
```

## 3.1.4 if 表达式的值与类型

`if` 不仅仅是一个"判断语句"，它本身也是一个**表达式**——可以有值。这一点和很多其他语言不同，需要特别注意。

### 情况一：if 作为表达式被使用

当你把 `if` 的结果赋值给变量时，`if` 就是**有值**的——它的值是所执行分支中**最后一行表达式**的值，类型也是那个表达式的类型：

```Cangjie
main() {
    let voltage = 5.0
    let bit = if (voltage < 2.5) {
        0
    } else {
        1
    }
    println(bit)        // 输出：1
}
```

这里的 `if` 表达式会返回一个值：条件成立时返回 `0`，不成立时返回 `1`。

**类型推断规则：**

| 场景 | 规则 | 示例 |
|------|------|------|
| 两个分支类型相同 | `if` 的类型就是该类型 | `{ 0 }` / `{ 1 }` → `Int64` |
| 两个分支有公共父类型 | `if` 的类型取公共父类型 | 见下文 |
| 两个分支类型不兼容 | ❌ 编译报错 | `{ "a" }` / `{ 1 }` → 报错 |

两个分支类型不兼容的情况：

```Cangjie
main() {
    let a = 10
    let b = if (a == 10) {   // ❌ 编译报错：String 和 Int64 不兼容
        "this is 10"
    } else {
        20
    }
}
```

### 情况二：if 没有被赋值

当 `if` 只是独立使用、没有被赋值时，它的类型是 `Unit`，值为 `()`：

```Cangjie
main() {
    let a = 10
    if (a == 10) {
        println("是10")
    } else {
        println("不是10")
    }
    // 这里 if 的类型是 Unit，我们不关心它的值
}
```

绝大多数初学者遇到的都是这种情况——只是让 `if` 执行不同的操作，不关心返回值。

### 情况三：没有 else 分支

如果 `if` 没有 `else` 分支，那它的类型一定是 `Unit`：

```Cangjie
main() {
    let score = 85
    let result = if (score >= 60) {   // ❌ 编译报错？其实不会
        "及格"
    }
    // 但 result 的类型是 Unit，值是 ()
}
```

> [!WARNING]
> 没有 `else` 的 `if` 类型固定为 `Unit`！如果你想把 `if` 的结果存到变量里，**必须**有 `else` 分支。

## 常见问题

### Q1：`if` 的条件可以不加括号吗？

**不可以。** 仓颉要求 `if` 的条件必须用圆括号 `()` 包裹：

```Cangjie
if (score >= 60) {    // ✅ 正确
    println("及格")

if score >= 60 {      // ❌ 报错：缺少括号
    println("及格")
}
```

### Q2：多级 `if` 嵌套太深怎么办？

当 `if-else if` 层级太多时，代码会变得难以阅读。这种情况可以考虑后续章节会讲到的 `match` 表达式（类似其他语言的 `switch`）。

### Q3：`if` 分支里可以有多个语句吗？

```Cangjie
if (score >= 60) {
    println("恭喜！")
    println("你及格了！")
    let bonus = score - 60
    println("超出及格线 ${bonus} 分")
}
```

### Q4：`else if` 和 `else { if }` 有区别吗？

**效果一样**，但 `else if` 写法更简洁。下面两种写法等价：

```Cangjie
// 写法一：else if（推荐）
if (a > 0) {
    println("正数")
} else if (a < 0) {
    println("负数")
} else {
    println("零")
}

// 写法二：else 里面再套 if（不推荐）
if (a > 0) {
    println("正数")
} else {
    if (a < 0) {
        println("负数")
    } else {
        println("零")
    }
}
```

推荐使用 `else if`，代码更清晰。

## 3.1.1 什么是分支结构？

程序并不总是按顺序一路执行到底——很多时候，我们需要根据**不同的条件**执行**不同的代码**。就像生活中：

> 如果今天下雨，就带伞；否则就不带。

在编程中，这种"如果……否则……"的逻辑就叫做**分支结构**，而 `if` 就是实现它的工具。

---

## 3.1.2 if 的基本用法

`if` 表达式的结构很简单：

```Cangjie
if (条件1) {
    // 条件1成立时执行的代码
} else if(条件2){
    // 条件1不满足 条件2满足时候执行
}else {
    // 条件不成立时执行的代码
}
```

> [!TIP]
> 可以这样理解：
> - `if` = "如果……"
> - `else if` = "再如果"
> - `else` = "否则……"
> - 条件用圆括号 `()` 包裹
> - 要执行的代码用花括号 `{}` 包裹

### 最简单的例子

```Cangjie
main() {
    let score = 85

    if (score >= 60) {
        println("及格了！")
    } else {
        println("不及格，继续加油💪")
    }
}
```

运行结果：

```Bash
及格了！
```

### else 可以省略

如果只关心"条件成立时"做什么，`else` 可以不写：

```Cangjie
main() {
    let isRaining = true

    if (isRaining) {
        println("记得带伞！")
    }
    // 没下雨就什么都不做
}
```

### 多级判断：else if

当有多个条件需要依次判断时，可以在 `else` 后面再接一个 `if`：

```Cangjie
import std.random.Random

main() {
    let speed = Random().nextFloat64() * 20.0
    println("${speed} km/s")

    if (speed > 16.7) {
        println("第三宇宙速度，鹊桥相会")
    } else if (speed > 11.2) {
        println("第二宇宙速度，嫦娥奔月")
    } else if (speed > 7.9) {
        println("第一宇宙速度，腾云驾雾")
    } else {
        println("脚踏实地，仰望星空")
    }
}
```

> [!TIP]
> 这样一层层判断，就像流水线筛选：
> 1️⃣ 速度 > 16.7？→ 第三宇宙速度
> 2️⃣ 否则，速度 > 11.2？→ 第二宇宙速度
> 3️⃣ 否则，速度 > 7.9？→ 第一宇宙速度
> 4️⃣ 否则 → 达不到宇宙速度

---

## 3.1.3 条件必须是布尔值

仓颉是**强类型**语言——`if` 的条件**必须**是 `Bool` 类型（`true` 或 `false`），不能拿数字代替。

```Cangjie
main() {
    let number = 1
    if (number) {       // ❌ 编译错误：不能把 Int64 当作条件
        println("非零数")
    }
}
```

> [!NOTE]
> 有些语言（如 C 语言）允许把 `0` 当作 `false`、非 `0` 当作 `true`，但仓颉**不允许**。你必须写出明确的布尔表达式，比如 `number != 0`。

```Cangjie
main() {
    let number = 1
    if (number != 0) {  // ✅ 正确：用比较运算得到 Bool 值
        println("非零数")
    }
}
```
