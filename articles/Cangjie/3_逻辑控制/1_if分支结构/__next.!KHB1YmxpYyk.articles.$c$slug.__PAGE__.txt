## 3.1.1 什么是分支结构？

程序并不总是按顺序一路执行到底——很多时候，我们需要根据**不同的条件**执行**不同的代码**。就像生活中：

> 如果今天下雨，就带伞；否则就不带。
>
> 如果分数及格，就说"通过了"；否则就说"继续加油"。

在编程中，这种"如果……否则……"的逻辑就叫做**分支结构**，而 `if` 就是实现它的工具。

### 你可以把 if 想象成一个"岔路口"

```
                ┌─ 条件成立 → 走这条路（if 分支）
 走到路口判断 ──┤
                └─ 条件不成立 → 走那条路（else 分支）
```

每个条件判断都是一次"二选一"。

---

## 3.1.2 if 的基本用法

`if` 表达式的结构很简单：

```Cangjie
if (条件) {
    // 条件为 true 时执行的代码
} else {
    // 条件为 false 时执行的代码
}
```

> [!TIP]
> 可以这样理解：
> - `if` = "如果……"
> - `else` = "否则……"
> - 条件用圆括号 `()` 包裹
> - 要执行的代码用花括号 `{}` 包裹

### 最简单的例子

```Cangjie
main() {
    let score = 85

    if (score >= 60) {
        println("及格了！")
    } else {
        println("不及格，继续加油💪")
    }
}
```

运行结果：

```Bash
及格了！
```

**逐行执行过程**：

1. `score = 85`
2. 判断 `score >= 60` → `85 >= 60` → `true`
3. 进入 `if` 分支 → 输出"及格了！"
4. `else` 分支被跳过，不会执行

### else 可以省略

如果只关心"条件成立时"做什么，`else` 可以不写：

```Cangjie
main() {
    let isRaining = true

    if (isRaining) {
        println("记得带伞！")
    }
    // 没下雨就什么都不做
}
```

> [!NOTE]
> 没有 `else` 时，如果条件为 `false`，程序什么也不做，直接跳过 `if` 继续往下执行。

### 多级判断：else if

当有多个条件需要依次判断时，可以在 `else` 后面再接一个 `if`：

```Cangjie
import std.random.Random

main() {
    let speed = Random().nextFloat64() * 20.0
    println("${speed} km/s")

    if (speed > 16.7) {
        println("第三宇宙速度，鹊桥相会")
    } else if (speed > 11.2) {
        println("第二宇宙速度，嫦娥奔月")
    } else if (speed > 7.9) {
        println("第一宇宙速度，腾云驾雾")
    } else {
        println("脚踏实地，仰望星空")
    }
}
```

> [!TIP]
> 这样一层层判断，就像流水线筛选：
> ```
> 速度 > 16.7？ → 第三宇宙速度 ✅ 后面的分支跳过
> 否则 → 速度 > 11.2？ → 第二宇宙速度 ✅ 后面的分支跳过
> 否则 → 速度 > 7.9？ → 第一宇宙速度 ✅ 后面的分支跳过
> 否则 → 脚踏实地
> ```
>
> **关键点**：一旦某个分支匹配成功，后面的 `else if` 和 `else` 都不会再执行了。

### else if 不是必须的——可以无限嵌套 if-else

实际上 `else if` 只是"`else` 里面再写一个 `if`"的简写。但推荐用 `else if` 写法，更清晰：

```Cangjie
// ✅ 推荐：else if 写法
if (a > 0) {
    println("正数")
} else if (a < 0) {
    println("负数")
} else {
    println("零")
}

// ⚠️ 不推荐：else 里面再套 if（缩进越来越深）
if (a > 0) {
    println("正数")
} else {
    if (a < 0) {
        println("负数")
    } else {
        println("零")
    }
}
```

## 3.1.3 条件必须是布尔值

仓颉是**强类型**语言——`if` 的条件**必须**是 `Bool` 类型（`true` 或 `false`），不能拿数字代替。

```Cangjie
main() {
    let number = 1
    if (number) {       // ❌ 编译错误：不能把 Int64 当作条件
        println("非零数")
    }
}
```

> [!CAUTION]
> **这是新手最容易犯的错之一！**
>
> 有些语言（如 C 语言）允许把 `0` 当作 `false`、非 `0` 当作 `true`，比如 `if (1)` 是成立的。
>
> 但仓颉**不允许**这样写。你必须写出明确的布尔表达式：

```Cangjie
main() {
    let number = 1
    if (number != 0) {  // ✅ 正确：用比较运算得到 Bool 值
        println("非零数")
    }
}
```

### 常见的布尔判断写法

| 你想表达的意思 | ✅ 正确写法 | ❌ 错误写法 |
|--------------|-----------|-----------|
| number 非零 | `number != 0` | `if (number)` |
| 字符串非空 | `name.size > 0` | `if (name)` |
| 布尔值为真 | `flag == true` 或直接 `flag` | — |

## 3.1.4 if 表达式的值与类型

`if` 不仅仅是一个"判断语句"，它本身也是一个**表达式**——可以有值。这一点和很多其他语言不同，需要特别注意。

> [!TIP]
> 如果学过其他语言（如 Java、C++），它们的 `if` 是"语句"（statement），没有值。仓颉的 `if` 是"表达式"（expression），**可有值**。

### 情况一：if 作为表达式被使用（有值）

当你把 `if` 的结果赋值给变量时，`if` 就是**有值**的——它的值是所执行分支中**最后一行表达式**的值，类型也是那个表达式的类型：

```Cangjie
main() {
    let voltage = 5.0
    let bit = if (voltage < 2.5) {
        0
    } else {
        1
    }
    println(bit)        // 输出：1
}
```

这里的 `if` 表达式会返回一个值：
- `voltage < 2.5` 为 `true` → 返回 `0`
- `voltage < 2.5` 为 `false` → 返回 `1`

因为 `voltage = 5.0`，所以条件不成立，走 `else` 分支，返回 `1`。

**类型推断规则（重要！）：**

| 场景 | 规则 | 示例 |
|------|------|------|
| 两个分支类型相同 | `if` 的类型就是该类型 | `{ 0 }` / `{ 1 }` → `Int64` |
| 两个分支有公共父类型 | `if` 的类型取公共父类型 | 后续章节讲继承时会涉及 |
| 两个分支类型不兼容 | ❌ 编译报错 | `{ "a" }` / `{ 1 }` → 报错 |

两个分支类型不兼容的情况：

```Cangjie
main() {
    let a = 10
    let b = if (a == 10) {   // ❌ 编译报错：String 和 Int64 不兼容
        "this is 10"
    } else {
        20
    }
}
```

> [!CAUTION]
> 想让 `if` 作为表达式使用，**两个分支的类型必须兼容**。一个返回 `String`、一个返回 `Int64` 是不行的。

### 情况二：if 没有被赋值（最常用）

当 `if` 只是独立使用、没有被赋值时，它的类型是 `Unit`，值为 `()`：

```Cangjie
main() {
    let a = 10
    if (a == 10) {
        println("是10")
    } else {
        println("不是10")
    }
    // 这里 if 的类型是 Unit，我们不关心它的值
}
```

绝大多数初学者遇到的都是这种情况——只是让 `if` 执行不同的操作，不关心返回值。

### 情况三：没有 else 分支

如果 `if` 没有 `else` 分支，那它的类型一定是 `Unit`：

```Cangjie
main() {
    let result = if (score >= 60) {
        "及格"      // ❓ result 是什么类型？
    }
    // result 的类型是 Unit，值是 ()
    // 因为 if 没有 else，编译器认为"条件不成立时没有值"
}
```

> [!WARNING]
> **这是新手最容易踩的坑！**
>
> 没有 `else` 的 `if` 类型固定为 `Unit`。如果你想把 `if` 的结果存到变量里，**必须**有 `else` 分支，且所有分支的类型要一致。

---

## 3.1.5 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本结构 | `if (条件) { ... } else { ... }` | 如果……否则…… |
| 条件类型 | 必须是 `Bool`（不能是数字） | 仓颉不要 `if(1)` |
| else 可省略 | 是，如果只关心"成立时" | 不写 else 就是"不做" |
| 多级判断 | `else if` 可以连续使用 | 像个漏斗，从上往下筛 |
| if 作为表达式 | 有值，类型由分支共同决定 | 可以赋值给变量 |
| 没有 else 的 if | 类型固定为 `Unit` | 不能赋值使用 |

## 常见问题

### Q1：`if` 的条件可以不加括号吗？

**不可以。** 仓颉要求 `if` 的条件必须用圆括号 `()` 包裹：

```Cangjie
if (score >= 60) {    // ✅ 正确
    println("及格")
}

if score >= 60 {      // ❌ 报错：缺少括号
    println("及格")
}
```

### Q2：多级 `if-else if` 嵌套太深怎么办？

当 `if-else if` 层级太多时，代码会变得难以阅读。一种常见的替代方案是用后续要讲的 `match` 表达式。

如果实在要用 `if`，也可以用 **提前返回** 的技巧：

```Cangjie
// 每层缩进太深了
func getType(value: Int64): String {
    if (value > 100) {
        return "大"
    } else {
        if (value > 50) {
            return "中"
        } else {
            return "小"
        }
    }
}

// 用提前返回，减少缩进
func getType(value: Int64): String {
    if (value > 100) {
        return "大"
    }
    if (value > 50) {
        return "中"
    }
    return "小"
}
```

第二种写法更清晰——**每层判断完就返回，不需要层层嵌套**。

### Q3：`if` 分支里可以有多个语句吗？

可以。花括号 `{}` 里可以写任意多行代码，按照顺序依次执行：

```Cangjie
if (score >= 60) {
    println("恭喜！")
    println("你及格了！")
    let bonus = score - 60
    println("超出及格线 ${bonus} 分")
}
```

但要注意：当 `if` 作为表达式使用时（赋值给变量），每个分支的**最后一行**决定返回值。

```Cangjie
let result = if (score >= 60) {
    println("恭喜！")
    score               // ← 最后一行，决定 if 的返回值
} else {
    println("加油！")
    0                   // ← 最后一行
}
```

### Q4：为什么仓颉的 `if` 是表达式而不是语句？

这是仓颉的设计特点——让更多东西成为"表达式"，可以组合使用。比如：

```Cangjie
// 不用 if 表达式时：需要先声明变量，再 if 赋值
var message = ""
if (age >= 18) {
    message = "成人"
} else {
    message = "未成年"
}

// 用 if 表达式时：一行搞定声明+赋值
let message = if (age >= 18) { "成人" } else { "未成年" }
```

后者更简洁，而且 `message` 可以用 `let`（不可变），更安全。

## 3.1.1 什么是分支结构？

程序并不总是按顺序一路执行到底——很多时候，我们需要根据**不同的条件**执行**不同的代码**。就像生活中：

> 如果今天下雨，就带伞；否则就不带。
>
> 如果分数及格，就说"通过了"；否则就说"继续加油"。

在编程中，这种"如果……否则……"的逻辑就叫做**分支结构**，而 `if` 就是实现它的工具。

### 你可以把 if 想象成一个"岔路口"

```
                ┌─ 条件成立 → 走这条路（if 分支）
 走到路口判断 ──┤
                └─ 条件不成立 → 走那条路（else 分支）
```

每个条件判断都是一次"二选一"。

---

## 3.1.2 if 的基本用法

`if` 表达式的结构很简单：

```Cangjie
if (条件) {
    // 条件为 true 时执行的代码
} else {
    // 条件为 false 时执行的代码
}
```

> [!TIP]
> 可以这样理解：
> - `if` = "如果……"
> - `else` = "否则……"
> - 条件用圆括号 `()` 包裹
> - 要执行的代码用花括号 `{}` 包裹

### 最简单的例子

```Cangjie
main() {
    let score = 85

    if (score >= 60) {
        println("及格了！")
    } else {
        println("不及格，继续加油💪")
    }
}
```

运行结果：

```Bash
及格了！
```

**逐行执行过程**：

1. `score = 85`
2. 判断 `score >= 60` → `85 >= 60` → `true`
3. 进入 `if` 分支 → 输出"及格了！"
4. `else` 分支被跳过，不会执行

### else 可以省略

如果只关心"条件成立时"做什么，`else` 可以不写：

```Cangjie
main() {
    let isRaining = true

    if (isRaining) {
        println("记得带伞！")
    }
    // 没下雨就什么都不做
}
```

> [!NOTE]
> 没有 `else` 时，如果条件为 `false`，程序什么也不做，直接跳过 `if` 继续往下执行。

### 多级判断：else if

当有多个条件需要依次判断时，可以在 `else` 后面再接一个 `if`：

```Cangjie
import std.random.Random

main() {
    let speed = Random().nextFloat64() * 20.0
    println("${speed} km/s")

    if (speed > 16.7) {
        println("第三宇宙速度，鹊桥相会")
    } else if (speed > 11.2) {
        println("第二宇宙速度，嫦娥奔月")
    } else if (speed > 7.9) {
        println("第一宇宙速度，腾云驾雾")
    } else {
        println("脚踏实地，仰望星空")
    }
}
```

> [!TIP]
> 这样一层层判断，就像流水线筛选：
> ```
> 速度 > 16.7？ → 第三宇宙速度 ✅ 后面的分支跳过
> 否则 → 速度 > 11.2？ → 第二宇宙速度 ✅ 后面的分支跳过
> 否则 → 速度 > 7.9？ → 第一宇宙速度 ✅ 后面的分支跳过
> 否则 → 脚踏实地
> ```
>
> **关键点**：一旦某个分支匹配成功，后面的 `else if` 和 `else` 都不会再执行了。

### else if 不是必须的——可以无限嵌套 if-else

实际上 `else if` 只是"`else` 里面再写一个 `if`"的简写。但推荐用 `else if` 写法，更清晰：

```Cangjie
// ✅ 推荐：else if 写法
if (a > 0) {
    println("正数")
} else if (a < 0) {
    println("负数")
} else {
    println("零")
}

// ⚠️ 不推荐：else 里面再套 if（缩进越来越深）
if (a > 0) {
    println("正数")
} else {
    if (a < 0) {
        println("负数")
    } else {
        println("零")
    }
}
```

## 3.1.3 条件必须是布尔值

仓颉是**强类型**语言——`if` 的条件**必须**是 `Bool` 类型（`true` 或 `false`），不能拿数字代替。

```Cangjie
main() {
    let number = 1
    if (number) {       // ❌ 编译错误：不能把 Int64 当作条件
        println("非零数")
    }
}
```

> [!CAUTION]
> **这是新手最容易犯的错之一！**
>
> 有些语言（如 C 语言）允许把 `0` 当作 `false`、非 `0` 当作 `true`，比如 `if (1)` 是成立的。
>
> 但仓颉**不允许**这样写。你必须写出明确的布尔表达式：

```Cangjie
main() {
    let number = 1
    if (number != 0) {  // ✅ 正确：用比较运算得到 Bool 值
        println("非零数")
    }
}
```

### 常见的布尔判断写法

| 你想表达的意思 | ✅ 正确写法 | ❌ 错误写法 |
|--------------|-----------|-----------|
| number 非零 | `number != 0` | `if (number)` |
| 字符串非空 | `name.size > 0` | `if (name)` |
| 布尔值为真 | `flag == true` 或直接 `flag` | — |

## 3.1.4 if 表达式的值与类型

`if` 不仅仅是一个"判断语句"，它本身也是一个**表达式**——可以有值。这一点和很多其他语言不同，需要特别注意。

> [!TIP]
> 如果学过其他语言（如 Java、C++），它们的 `if` 是"语句"（statement），没有值。仓颉的 `if` 是"表达式"（expression），**可有值**。

### 情况一：if 作为表达式被使用（有值）

当你把 `if` 的结果赋值给变量时，`if` 就是**有值**的——它的值是所执行分支中**最后一行表达式**的值，类型也是那个表达式的类型：

```Cangjie
main() {
    let voltage = 5.0
    let bit = if (voltage < 2.5) {
        0
    } else {
        1
    }
    println(bit)        // 输出：1
}
```

这里的 `if` 表达式会返回一个值：
- `voltage < 2.5` 为 `true` → 返回 `0`
- `voltage < 2.5` 为 `false` → 返回 `1`

因为 `voltage = 5.0`，所以条件不成立，走 `else` 分支，返回 `1`。

**类型推断规则（重要！）：**

| 场景 | 规则 | 示例 |
|------|------|------|
| 两个分支类型相同 | `if` 的类型就是该类型 | `{ 0 }` / `{ 1 }` → `Int64` |
| 两个分支有公共父类型 | `if` 的类型取公共父类型 | 后续章节讲继承时会涉及 |
| 两个分支类型不兼容 | ❌ 编译报错 | `{ "a" }` / `{ 1 }` → 报错 |

两个分支类型不兼容的情况：

```Cangjie
main() {
    let a = 10
    let b = if (a == 10) {   // ❌ 编译报错：String 和 Int64 不兼容
        "this is 10"
    } else {
        20
    }
}
```

> [!CAUTION]
> 想让 `if` 作为表达式使用，**两个分支的类型必须兼容**。一个返回 `String`、一个返回 `Int64` 是不行的。

### 情况二：if 没有被赋值（最常用）

当 `if` 只是独立使用、没有被赋值时，它的类型是 `Unit`，值为 `()`：

```Cangjie
main() {
    let a = 10
    if (a == 10) {
        println("是10")
    } else {
        println("不是10")
    }
    // 这里 if 的类型是 Unit，我们不关心它的值
}
```

绝大多数初学者遇到的都是这种情况——只是让 `if` 执行不同的操作，不关心返回值。

### 情况三：没有 else 分支

如果 `if` 没有 `else` 分支，那它的类型一定是 `Unit`：

```Cangjie
main() {
    let result = if (score >= 60) {
        "及格"      // ❓ result 是什么类型？
    }
    // result 的类型是 Unit，值是 ()
    // 因为 if 没有 else，编译器认为"条件不成立时没有值"
}
```

> [!WARNING]
> **这是新手最容易踩的坑！**
>
> 没有 `else` 的 `if` 类型固定为 `Unit`。如果你想把 `if` 的结果存到变量里，**必须**有 `else` 分支，且所有分支的类型要一致。

## 3.1.5 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本结构 | `if (条件) { ... } else { ... }` | 如果……否则…… |
| 条件类型 | 必须是 `Bool`（不能是数字） | 仓颉不要 `if(1)` |
| else 可省略 | 是，如果只关心"成立时" | 不写 else 就是"不做" |
| 多级判断 | `else if` 可以连续使用 | 像个漏斗，从上往下筛 |
| if 作为表达式 | 有值，类型由分支共同决定 | 可以赋值给变量 |
| 没有 else 的 if | 类型固定为 `Unit` | 不能赋值使用 |

## 常见问题

### Q1：`if` 的条件可以不加括号吗？

**不可以。** 仓颉要求 `if` 的条件必须用圆括号 `()` 包裹：

```Cangjie
if (score >= 60) {    // ✅ 正确
    println("及格")
}

if score >= 60 {      // ❌ 报错：缺少括号
    println("及格")
}
```

### Q2：多级 `if-else if` 嵌套太深怎么办？

当 `if-else if` 层级太多时，代码会变得难以阅读。一种常见的替代方案是用后续要讲的 `match` 表达式。

如果实在要用 `if`，也可以用 **提前返回** 的技巧：

```Cangjie
// 每层缩进太深了
func getType(value: Int64): String {
    if (value > 100) {
        return "大"
    } else {
        if (value > 50) {
            return "中"
        } else {
            return "小"
        }
    }
}

// 用提前返回，减少缩进
func getType(value: Int64): String {
    if (value > 100) {
        return "大"
    }
    if (value > 50) {
        return "中"
    }
    return "小"
}
```

第二种写法更清晰——**每层判断完就返回，不需要层层嵌套**。

### Q3：`if` 分支里可以有多个语句吗？

可以。花括号 `{}` 里可以写任意多行代码，按照顺序依次执行：

```Cangjie
if (score >= 60) {
    println("恭喜！")
    println("你及格了！")
    let bonus = score - 60
    println("超出及格线 ${bonus} 分")
}
```

但要注意：当 `if` 作为表达式使用时（赋值给变量），每个分支的**最后一行**决定返回值。

```Cangjie
let result = if (score >= 60) {
    println("恭喜！")
    score               // ← 最后一行，决定 if 的返回值
} else {
    println("加油！")
    0                   // ← 最后一行
}
```

### Q4：为什么仓颉的 `if` 是表达式而不是语句？

这是仓颉的设计特点——让更多东西成为"表达式"，可以组合使用。比如：

```Cangjie
// 不用 if 表达式时：需要先声明变量，再 if 赋值
var message = ""
if (age >= 18) {
    message = "成人"
} else {
    message = "未成年"
}

// 用 if 表达式时：一行搞定声明+赋值
let message = if (age >= 18) { "成人" } else { "未成年" }
```

后者更简洁，而且 `message` 可以用 `let`（不可变），更安全。

## 3.1.1 什么是分支结构？

程序并不总是按顺序一路执行到底——很多时候，我们需要根据**不同的条件**执行**不同的代码**。就像生活中：

> 如果今天下雨，就带伞；否则就不带。
>
> 如果分数及格，就说"通过了"；否则就说"继续加油"。

在编程中，这种"如果……否则……"的逻辑就叫做**分支结构**，而 `if` 就是实现它的工具。

### 你可以把 if 想象成一个"岔路口"

```
                ┌─ 条件成立 → 走这条路（if 分支）
 走到路口判断 ──┤
                └─ 条件不成立 → 走那条路（else 分支）
```

每个条件判断都是一次"二选一"。

---

## 3.1.2 if 的基本用法

`if` 表达式的结构很简单：

```Cangjie
if (条件) {
    // 条件为 true 时执行的代码
} else {
    // 条件为 false 时执行的代码
}
```

> [!TIP]
> 可以这样理解：
> - `if` = "如果……"
> - `else` = "否则……"
> - 条件用圆括号 `()` 包裹
> - 要执行的代码用花括号 `{}` 包裹

### 最简单的例子

```Cangjie
main() {
    let score = 85

    if (score >= 60) {
        println("及格了！")
    } else {
        println("不及格，继续加油💪")
    }
}
```

运行结果：

```Bash
及格了！
```

**逐行执行过程**：

1. `score = 85`
2. 判断 `score >= 60` → `85 >= 60` → `true`
3. 进入 `if` 分支 → 输出"及格了！"
4. `else` 分支被跳过，不会执行

### else 可以省略

如果只关心"条件成立时"做什么，`else` 可以不写：

```Cangjie
main() {
    let isRaining = true

    if (isRaining) {
        println("记得带伞！")
    }
    // 没下雨就什么都不做
}
```

> [!NOTE]
> 没有 `else` 时，如果条件为 `false`，程序什么也不做，直接跳过 `if` 继续往下执行。

### 多级判断：else if

当有多个条件需要依次判断时，可以在 `else` 后面再接一个 `if`：

```Cangjie
import std.random.Random

main() {
    let speed = Random().nextFloat64() * 20.0
    println("${speed} km/s")

    if (speed > 16.7) {
        println("第三宇宙速度，鹊桥相会")
    } else if (speed > 11.2) {
        println("第二宇宙速度，嫦娥奔月")
    } else if (speed > 7.9) {
        println("第一宇宙速度，腾云驾雾")
    } else {
        println("脚踏实地，仰望星空")
    }
}
```

> [!TIP]
> 这样一层层判断，就像流水线筛选：
> ```
> 速度 > 16.7？ → 第三宇宙速度 ✅ 后面的分支跳过
> 否则 → 速度 > 11.2？ → 第二宇宙速度 ✅ 后面的分支跳过
> 否则 → 速度 > 7.9？ → 第一宇宙速度 ✅ 后面的分支跳过
> 否则 → 脚踏实地
> ```
>
> **关键点**：一旦某个分支匹配成功，后面的 `else if` 和 `else` 都不会再执行了。

### else if 不是必须的——可以无限嵌套 if-else

实际上 `else if` 只是"`else` 里面再写一个 `if`"的简写。但推荐用 `else if` 写法，更清晰：

```Cangjie
// ✅ 推荐：else if 写法
if (a > 0) {
    println("正数")
} else if (a < 0) {
    println("负数")
} else {
    println("零")
}

// ⚠️ 不推荐：else 里面再套 if（缩进越来越深）
if (a > 0) {
    println("正数")
} else {
    if (a < 0) {
        println("负数")
    } else {
        println("零")
    }
}
```

## 3.1.3 条件必须是布尔值

仓颉是**强类型**语言——`if` 的条件**必须**是 `Bool` 类型（`true` 或 `false`），不能拿数字代替。

```Cangjie
main() {
    let number = 1
    if (number) {       // ❌ 编译错误：不能把 Int64 当作条件
        println("非零数")
    }
}
```

> [!CAUTION]
> **这是新手最容易犯的错之一！**
>
> 有些语言（如 C 语言）允许把 `0` 当作 `false`、非 `0` 当作 `true`，比如 `if (1)` 是成立的。
>
> 但仓颉**不允许**这样写。你必须写出明确的布尔表达式：

```Cangjie
main() {
    let number = 1
    if (number != 0) {  // ✅ 正确：用比较运算得到 Bool 值
        println("非零数")
    }
}
```

### 常见的布尔判断写法

| 你想表达的意思 | ✅ 正确写法 | ❌ 错误写法 |
|--------------|-----------|-----------|
| number 非零 | `number != 0` | `if (number)` |
| 字符串非空 | `name.size > 0` | `if (name)` |
| 布尔值为真 | `flag == true` 或直接 `flag` | — |

---

## 3.1.4 if 表达式的值与类型

`if` 不仅仅是一个"判断语句"，它本身也是一个**表达式**——可以有值。这一点和很多其他语言不同，需要特别注意。

> [!TIP]
> 如果学过其他语言（如 Java、C++），它们的 `if` 是"语句"（statement），没有值。仓颉的 `if` 是"表达式"（expression），**可有值**。

### 情况一：if 作为表达式被使用（有值）

当你把 `if` 的结果赋值给变量时，`if` 就是**有值**的——它的值是所执行分支中**最后一行表达式**的值，类型也是那个表达式的类型：

```Cangjie
main() {
    let voltage = 5.0
    let bit = if (voltage < 2.5) {
        0
    } else {
        1
    }
    println(bit)        // 输出：1
}
```

这里的 `if` 表达式会返回一个值：
- `voltage < 2.5` 为 `true` → 返回 `0`
- `voltage < 2.5` 为 `false` → 返回 `1`

因为 `voltage = 5.0`，所以条件不成立，走 `else` 分支，返回 `1`。

**类型推断规则（重要！）：**

| 场景 | 规则 | 示例 |
|------|------|------|
| 两个分支类型相同 | `if` 的类型就是该类型 | `{ 0 }` / `{ 1 }` → `Int64` |
| 两个分支有公共父类型 | `if` 的类型取公共父类型 | 后续章节讲继承时会涉及 |
| 两个分支类型不兼容 | ❌ 编译报错 | `{ "a" }` / `{ 1 }` → 报错 |

两个分支类型不兼容的情况：

```Cangjie
main() {
    let a = 10
    let b = if (a == 10) {   // ❌ 编译报错：String 和 Int64 不兼容
        "this is 10"
    } else {
        20
    }
}
```

> [!CAUTION]
> 想让 `if` 作为表达式使用，**两个分支的类型必须兼容**。一个返回 `String`、一个返回 `Int64` 是不行的。

### 情况二：if 没有被赋值（最常用）

当 `if` 只是独立使用、没有被赋值时，它的类型是 `Unit`，值为 `()`：

```Cangjie
main() {
    let a = 10
    if (a == 10) {
        println("是10")
    } else {
        println("不是10")
    }
    // 这里 if 的类型是 Unit，我们不关心它的值
}
```

绝大多数初学者遇到的都是这种情况——只是让 `if` 执行不同的操作，不关心返回值。

### 情况三：没有 else 分支

如果 `if` 没有 `else` 分支，那它的类型一定是 `Unit`：

```Cangjie
main() {
    let result = if (score >= 60) {
        "及格"      // ❓ result 是什么类型？
    }
    // result 的类型是 Unit，值是 ()
    // 因为 if 没有 else，编译器认为"条件不成立时没有值"
}
```

> [!WARNING]
> **这是新手最容易踩的坑！**
>
> 没有 `else` 的 `if` 类型固定为 `Unit`。如果你想把 `if` 的结果存到变量里，**必须**有 `else` 分支，且所有分支的类型要一致。

## 3.1.5 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本结构 | `if (条件) { ... } else { ... }` | 如果……否则…… |
| 条件类型 | 必须是 `Bool`（不能是数字） | 仓颉不要 `if(1)` |
| else 可省略 | 是，如果只关心"成立时" | 不写 else 就是"不做" |
| 多级判断 | `else if` 可以连续使用 | 像个漏斗，从上往下筛 |
| if 作为表达式 | 有值，类型由分支共同决定 | 可以赋值给变量 |
| 没有 else 的 if | 类型固定为 `Unit` | 不能赋值使用 |

## 常见问题

### Q1：`if` 的条件可以不加括号吗？

**不可以。** 仓颉要求 `if` 的条件必须用圆括号 `()` 包裹：

```Cangjie
if (score >= 60) {    // ✅ 正确
    println("及格")
}

if score >= 60 {      // ❌ 报错：缺少括号
    println("及格")
}
```

### Q2：多级 `if-else if` 嵌套太深怎么办？

当 `if-else if` 层级太多时，代码会变得难以阅读。一种常见的替代方案是用后续要讲的 `match` 表达式。

如果实在要用 `if`，也可以用 **提前返回** 的技巧：

```Cangjie
// 每层缩进太深了
func getType(value: Int64): String {
    if (value > 100) {
        return "大"
    } else {
        if (value > 50) {
            return "中"
        } else {
            return "小"
        }
    }
}

// 用提前返回，减少缩进
func getType(value: Int64): String {
    if (value > 100) {
        return "大"
    }
    if (value > 50) {
        return "中"
    }
    return "小"
}
```

第二种写法更清晰——**每层判断完就返回，不需要层层嵌套**。

### Q3：`if` 分支里可以有多个语句吗？

可以。花括号 `{}` 里可以写任意多行代码，按照顺序依次执行：

```Cangjie
if (score >= 60) {
    println("恭喜！")
    println("你及格了！")
    let bonus = score - 60
    println("超出及格线 ${bonus} 分")
}
```

但要注意：当 `if` 作为表达式使用时（赋值给变量），每个分支的**最后一行**决定返回值。

```Cangjie
let result = if (score >= 60) {
    println("恭喜！")
    score               // ← 最后一行，决定 if 的返回值
} else {
    println("加油！")
    0                   // ← 最后一行
}
```

### Q4：为什么仓颉的 `if` 是表达式而不是语句？

这是仓颉的设计特点——让更多东西成为"表达式"，可以组合使用。比如：

```Cangjie
// 不用 if 表达式时：需要先声明变量，再 if 赋值
var message = ""
if (age >= 18) {
    message = "成人"
} else {
    message = "未成年"
}

// 用 if 表达式时：一行搞定声明+赋值
let message = if (age >= 18) { "成人" } else { "未成年" }
```

后者更简洁，而且 `message` 可以用 `let`（不可变），更安全。
