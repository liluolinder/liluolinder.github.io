## 3.6.1 为什么要循环控制？

前面我们学了三种循环：`while`、`do-while`、`for-in`。它们都是"条件满足就一直执行"。

但有时候你需要在循环中途**提前结束**或**跳过某次**：

> 比如你在一个水果篮里找橘子：
> - 找到橘子后，后面的水果就不用看了 → **`break`**（提前终止整个循环）
> - 看到苹果，不感兴趣，直接看下一个 → **`continue`**（跳过本轮，进入下一轮）

仓颉提供了两个循环控制表达式：**`break`** 和 **`continue`**，它们的类型都是 `Nothing`。

---

## 3.6.2 break：提前终止循环

`break` 用于**立即结束整个循环**，执行循环后面的代码。

### 例子：找到第一个能被 5 整除的数

```Cangjie
main() {
    let numbers = [12, 18, 25, 36, 49, 55]
    for (number in numbers) {
        if (number % 5 == 0) {
            println(number)
            break       // 找到后立即结束循环
        }
    }
}
```

运行结果：

```Bash
25
```

> [!TIP]
> 如果没有 `break`，循环会遍历所有数字，输出 25 和 55（两个都能被 5 整除）。
> 有了 `break`，找到第一个就停，后面的 36、49、55 都不再检查了。

### break 在 while 中使用

```Cangjie
main() {
    var i = 0
    while (true) {          // 死循环
        if (i >= 5) {
            break           // 条件满足时退出
        }
        println(i)
        i++
    }
}
```

运行结果：

```Bash
0
1
2
3
4
```

> [!NOTE]
> `while (true)` + `break` 是一种常见写法——在循环体中间判断退出条件，比写在 `while` 条件里更灵活。

---

## 3.6.3 continue：跳过本轮循环

`continue` 用于**提前结束本轮循环**，直接进入下一轮。

### 例子：打印数组中的奇数

```Cangjie
main() {
    let numbers = [12, 18, 25, 36, 49, 55]
    for (number in numbers) {
        if (number % 2 == 0) {
            continue        // 偶数跳过，不打印
        }
        println(number)
    }
}
```

运行结果：

```Bash
25
49
55
```

循环过程：
1. `12` → 偶数 → `continue` → 跳过，进入下一轮
2. `18` → 偶数 → `continue` → 跳过
3. `25` → 奇数 → 打印 `25`
4. `36` → 偶数 → `continue` → 跳过
5. `49` → 奇数 → 打印 `49`
6. `55` → 奇数 → 打印 `55`

### continue 在 while 中使用

```Cangjie
main() {
    var i = 0
    while (i < 10) {
        i++
        if (i % 2 == 0) {
            continue        // 偶数跳过
        }
        println(i)           // 只打印奇数
    }
}
```

运行结果：

```Bash
1
3
5
7
9
```

> [!WARNING]
> 在 `while` 中使用 `continue` 要小心：如果 `continue` 在变量更新**之前**执行，可能导致死循环。
>
> ```Cangjie
> var i = 0
> while (i < 10) {
>     if (i % 2 == 0) {
>         continue    // ❌ i 还没增加，会一直 i=0 死循环
>     }
>     i++
> }
> ```

---

## 3.6.4 break vs continue 对比

| | `break` | `continue` |
|---|---------|-----------|
| 作用 | **终止整个循环** | **跳过本轮循环** |
| 后续执行 | 继续执行循环后面的代码 | 进入下一轮循环 |
| 适用场景 | 找到目标后就不需要继续找了 | 对当前元素不感兴趣，看下一个 |
| 类比 | 找到橘子，后面的水果不看了 | 看到苹果，跳过，看下一个 |

---

## 3.6.5 break 和 continue 的类型

`break` 和 `continue` 的类型都是 **`Nothing`**（不是 `Unit`）。

`Nothing` 是一个特殊的类型，表示"永远不会有值"——因为 `break` 和 `continue` 一旦执行，就会跳出或跳过循环，根本不会返回任何值。

> [!TIP]
> 你不需要主动使用 `Nothing` 类型，了解即可。它和 `Unit` 的区别：
> - `Unit`：有值，值是 `()`（"有一个空值"）
> - `Nothing`：永远不会有值（"根本不会执行到这里"）

---

## 3.6.6 小结

| 概念 | 要点 |
|------|------|
| `break` | 提前终止整个循环 |
| `continue` | 跳过本轮循环，进入下一轮 |
| 类型 | 都是 `Nothing` |
| 适用循环 | `while`、`do-while`、`for-in` |
| 常见用途 | `break`：找到目标后停止；`continue`：跳过不符合条件的元素 |

---

## 常见问题

### Q1：`break` 和 `continue` 能用在嵌套循环中吗？

**可以，但它们只作用于当前所在的循环层**，不会影响外层循环：

```Cangjie
main() {
    for (i in 0..3) {
        for (j in 0..3) {
            if (j == 2) {
                break       // 只跳出内层 for (j)，外层继续
            }
            print("(${i},${j}) ")
        }
        println()
    }
}
```

运行结果：

```Bash
(0,0) (0,1)
(1,0) (1,1)
(2,0) (2,1)
```

### Q2：`continue` 在 `while` 和 `for-in` 中的行为一样吗？

**不完全一样。** 在 `for-in` 中，`continue` 后会自动取下一个元素；但在 `while` 中，如果你在变量更新之前 `continue`，可能会导致死循环。

```Cangjie
// for-in：安全，continue 后自动取下一个元素
for (i in 0..5) {
    if (i % 2 == 0) { continue }
    println(i)
}

// while：需要小心，确保 continue 前变量已更新
var i = 0
while (i < 5) {
    i++                 // 在 continue 之前更新
    if (i % 2 == 0) { continue }
    println(i)
}
```

### Q3：`break` 能用在 `if` 中吗？

**不能。** `break` 和 `continue` 只能用在循环体（`while`、`do-while`、`for-in`）中，不能用在 `if` 里（除非 `if` 在循环体中）。

```Cangjie
if (true) {
    break       // ❌ 编译报错：break 不在循环中
}

for (i in 0..5) {
    if (i == 3) {
        break   // ✅ 正确：break 在 for-in 循环体中
    }
}
```

## 3.6.1 为什么要循环控制？

前面我们学了三种循环：`while`、`do-while`、`for-in`。它们都是"条件满足就一直执行"。

但有时候你需要在循环中途**提前结束**或**跳过某次**：

> 比如你在一个水果篮里找橘子：
> - 找到橘子后，后面的水果就不用看了 → **`break`**（提前终止整个循环）
> - 看到苹果，不感兴趣，直接看下一个 → **`continue`**（跳过本轮，进入下一轮）

仓颉提供了两个循环控制表达式：**`break`** 和 **`continue`**，它们的类型都是 `Nothing`。

---

## 3.6.2 break：提前终止循环

`break` 用于**立即结束整个循环**，执行循环后面的代码。

### 例子：找到第一个能被 5 整除的数

```Cangjie
main() {
    let numbers = [12, 18, 25, 36, 49, 55]
    for (number in numbers) {
        if (number % 5 == 0) {
            println(number)
            break       // 找到后立即结束循环
        }
    }
}
```

运行结果：

```Bash
25
```

> [!TIP]
> 如果没有 `break`，循环会遍历所有数字，输出 25 和 55（两个都能被 5 整除）。
> 有了 `break`，找到第一个就停，后面的 36、49、55 都不再检查了。

### break 在 while 中使用

```Cangjie
main() {
    var i = 0
    while (true) {          // 死循环
        if (i >= 5) {
            break           // 条件满足时退出
        }
        println(i)
        i++
    }
}
```

运行结果：

```Bash
0
1
2
3
4
```

> [!NOTE]
> `while (true)` + `break` 是一种常见写法——在循环体中间判断退出条件，比写在 `while` 条件里更灵活。

---

## 3.6.3 continue：跳过本轮循环

`continue` 用于**提前结束本轮循环**，直接进入下一轮。

### 例子：打印数组中的奇数

```Cangjie
main() {
    let numbers = [12, 18, 25, 36, 49, 55]
    for (number in numbers) {
        if (number % 2 == 0) {
            continue        // 偶数跳过，不打印
        }
        println(number)
    }
}
```

运行结果：

```Bash
25
49
55
```

循环过程：
1. `12` → 偶数 → `continue` → 跳过，进入下一轮
2. `18` → 偶数 → `continue` → 跳过
3. `25` → 奇数 → 打印 `25`
4. `36` → 偶数 → `continue` → 跳过
5. `49` → 奇数 → 打印 `49`
6. `55` → 奇数 → 打印 `55`

### continue 在 while 中使用

```Cangjie
main() {
    var i = 0
    while (i < 10) {
        i++
        if (i % 2 == 0) {
            continue        // 偶数跳过
        }
        println(i)           // 只打印奇数
    }
}
```

运行结果：

```Bash
1
3
5
7
9
```

> [!WARNING]
> 在 `while` 中使用 `continue` 要小心：如果 `continue` 在变量更新**之前**执行，可能导致死循环。
>
> ```Cangjie
> var i = 0
> while (i < 10) {
>     if (i % 2 == 0) {
>         continue    // ❌ i 还没增加，会一直 i=0 死循环
>     }
>     i++
> }
> ```

---

## 3.6.4 break vs continue 对比

| | `break` | `continue` |
|---|---------|-----------|
| 作用 | **终止整个循环** | **跳过本轮循环** |
| 后续执行 | 继续执行循环后面的代码 | 进入下一轮循环 |
| 适用场景 | 找到目标后就不需要继续找了 | 对当前元素不感兴趣，看下一个 |
| 类比 | 找到橘子，后面的水果不看了 | 看到苹果，跳过，看下一个 |

---

## 3.6.5 break 和 continue 的类型

`break` 和 `continue` 的类型都是 **`Nothing`**（不是 `Unit`）。

`Nothing` 是一个特殊的类型，表示"永远不会有值"——因为 `break` 和 `continue` 一旦执行，就会跳出或跳过循环，根本不会返回任何值。

> [!TIP]
> 你不需要主动使用 `Nothing` 类型，了解即可。它和 `Unit` 的区别：
> - `Unit`：有值，值是 `()`（"有一个空值"）
> - `Nothing`：永远不会有值（"根本不会执行到这里"）

---

## 3.6.6 小结

| 概念 | 要点 |
|------|------|
| `break` | 提前终止整个循环 |
| `continue` | 跳过本轮循环，进入下一轮 |
| 类型 | 都是 `Nothing` |
| 适用循环 | `while`、`do-while`、`for-in` |
| 常见用途 | `break`：找到目标后停止；`continue`：跳过不符合条件的元素 |

## 常见问题

### Q1：`break` 和 `continue` 能用在嵌套循环中吗？

**可以，但它们只作用于当前所在的循环层**，不会影响外层循环：

```Cangjie
main() {
    for (i in 0..3) {
        for (j in 0..3) {
            if (j == 2) {
                break       // 只跳出内层 for (j)，外层继续
            }
            print("(${i},${j}) ")
        }
        println()
    }
}
```

运行结果：

```Bash
(0,0) (0,1)
(1,0) (1,1)
(2,0) (2,1)
```

### Q2：`continue` 在 `while` 和 `for-in` 中的行为一样吗？

**不完全一样。** 在 `for-in` 中，`continue` 后会自动取下一个元素；但在 `while` 中，如果你在变量更新之前 `continue`，可能会导致死循环。

```Cangjie
// for-in：安全，continue 后自动取下一个元素
for (i in 0..5) {
    if (i % 2 == 0) { continue }
    println(i)
}

// while：需要小心，确保 continue 前变量已更新
var i = 0
while (i < 5) {
    i++                 // 在 continue 之前更新
    if (i % 2 == 0) { continue }
    println(i)
}
```

### Q3：`break` 能用在 `if` 中吗？

**不能。** `break` 和 `continue` 只能用在循环体（`while`、`do-while`、`for-in`）中，不能用在 `if` 里（除非 `if` 在循环体中）。

```Cangjie
if (true) {
    break       // ❌ 编译报错：break 不在循环中
}

for (i in 0..5) {
    if (i == 3) {
        break   // ✅ 正确：break 在 for-in 循环体中
    }
}

## 常见问题

### Q1：`break` 和 `continue` 能用在嵌套循环中吗？

**可以，但它们只作用于当前所在的循环层**，不会影响外层循环：

```Cangjie
main() {
    for (i in 0..3) {
        for (j in 0..3) {
            if (j == 2) {
                break       // 只跳出内层 for (j)，外层继续
            }
            print("(${i},${j}) ")
        }
        println()
    }
}
```

运行结果：

```Bash
(0,0) (0,1)
(1,0) (1,1)
(2,0) (2,1)
```

### Q2：`continue` 在 `while` 和 `for-in` 中的行为一样吗？

**不完全一样。** 在 `for-in` 中，`continue` 后会自动取下一个元素；但在 `while` 中，如果你在变量更新之前 `continue`，可能会导致死循环。

```Cangjie
// for-in：安全，continue 后自动取下一个元素
for (i in 0..5) {
    if (i % 2 == 0) { continue }
    println(i)
}

// while：需要小心，确保 continue 前变量已更新
var i = 0
while (i < 5) {
    i++                 // 在 continue 之前更新
    if (i % 2 == 0) { continue }
    println(i)
}
```

### Q3：`break` 能用在 `if` 中吗？

**不能。** `break` 和 `continue` 只能用在循环体（`while`、`do-while`、`for-in`）中，不能用在 `if` 里（除非 `if` 在循环体中）。

```Cangjie
if (true) {
    break       // ❌ 编译报错：break 不在循环中
}

for (i in 0..5) {
    if (i == 3) {
        break   // ✅ 正确：break 在 for-in 循环体中
    }
}
```
