## 3.3.1 什么是循环？

我们之前学的 `if` 和 `match` 都是"做一次判断，执行一次代码"。但有时候，我们需要**反复执行同一段代码**，直到某个条件不再满足为止。

比如：

> 你还没有吃饱 → 再吃一口 → 还没饱 → 再吃一口 → …… → 饱了 → 停止

这种"反复执行"的机制就叫做**循环**，而 `while` 就是实现它的工具。

### 生活中到处都是循环

| 场景 | 代码逻辑 |
|------|---------|
| 还没吃饱 → 再吃一口 | `while (还没饱) { 吃一口 }` |
| 还没跑完 5 圈 → 继续跑 | `while (跑完的圈数 < 5) { 跑一圈 }` |
| 还有作业没做完 → 继续做 | `while (还有作业) { 做一题 }` |

### while 的执行流程

```
           ┌─────────────────────┐
           │                     │
           ▼                     │
    ┌─ 判断条件 ──→ 条件为真？──→ 执行循环体
    │                              │
    │  条件为假                     │
    ▼                              │
  退出循环 ←───────────────────────┘
```

> [!TIP]
> `while` 就像"自动门"——你走到门前（判断条件），如果门开了（条件为真），就走进去（执行循环体）；如果门关着（条件为假），就不进去了（退出循环）。进去之后，再回头看门关了没有……

---

## 3.3.2 while 的基本用法

`while` 的结构很简单：

```Cangjie
while (条件) {
    // 条件为 true 时反复执行的代码
}
```

执行流程：
1. 检查条件 → 如果为 `true`，进入循环体
2. 执行循环体中的代码
3. 再次检查条件 → 如果还是 `true`，继续执行
4. 重复……直到条件为 `false`，退出循环

### 一个最简单的例子

```Cangjie
main() {
    var count = 0

    while (count < 5) {
        println("第 ${count} 次循环")
        count++
    }
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

> [!TIP]
> `count++` 的意思是 `count = count + 1`，每循环一次让计数器加 1。
>
> **逐轮过程**：
> - 第 1 轮：`count=0`，判断 `0 < 5` → true → 输出"第 0 次"，`count` 变为 1
> - 第 2 轮：`count=1`，判断 `1 < 5` → true → 输出"第 1 次"，`count` 变为 2
> - ……
> - 第 5 轮：`count=4`，判断 `4 < 5` → true → 输出"第 4 次"，`count` 变为 5
> - 第 6 轮：`count=5`，判断 `5 < 5` → false → 退出循环

### 实际例子：二分法求平方根

```Cangjie
main() {
    var root = 0.0
    var min = 1.0
    var max = 2.0
    var error = 1.0
    let tolerance = 0.1 ** 10

    while (error ** 2 > tolerance) {
        root = (min + max) / 2.0
        error = root ** 2 - 2.0
        if (error > 0.0) {
            max = root
        } else {
            min = root
        }
    }
    println("2 的平方根约等于：${root}")
}
```

运行结果：

```Bash
2 的平方根约等于：1.414215
```

这个程序用**二分法**不断逼近 2 的平方根——每次循环将范围缩小一半，直到误差足够小。

> [!NOTE]
> 这个例子需要一些数学基础，看不懂也没关系。关键是理解 `while` 的用法：**在条件满足时不断重复，条件不满足时结束**。

## 3.3.3 条件必须是布尔值

和 `if` 一样，`while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    let x = 1
    while (x) {         // ❌ 编译错误：不能把 Int64 当作条件
        println("无限循环？")
    }
}
```

必须写成明确的布尔表达式：

```Cangjie
main() {
    var x = 5
    while (x > 0) {     // ✅ 正确
        println(x)
        x--
    }
}
```

> [!CAUTION]
> 这是和 `if` 一样的规则。仓颉不会帮你把数字自动转成布尔值——你必须自己写比较条件。

---

## 3.3.4 死循环与终止

如果条件**永远为 `true`**，循环会一直执行下去，这就是"死循环"（无限循环）：

```Cangjie
main() {
    while (true) {
        println("停不下来啦！")
    }
}
```

> [!WARNING]
> **新手的噩梦：不小心写出死循环**
>
> 最常见的死循环原因：**忘记改变条件**。
>
> 比如上面那个 `count < 5` 的例子，如果忘记写 `count++`：
> ```Cangjie
> var count = 0
> while (count < 5) {
>     println("第 ${count} 次循环")
>     // 忘记写 count++ 了！！！count 永远是 0
> }
> // count < 5 永远为 true → 无限循环
> ```

**死循环不全是坏事！** 有些程序需要一直运行（比如服务器、游戏主循环、实时监控），这时候 `while (true)` 就是有意为之：

```Cangjie
// 服务器主循环：一直运行，等待客户端请求
while (true) {
    let request = waitForRequest()
    handleRequest(request)
}
```

### 如何终止死循环？

- 如果是在终端中运行的程序，按 `Ctrl+C` 强制终止
- 如果是 IDE 中运行，点击"停止"按钮
- 更好的方式：用 `while (true)` + `break` 组合，在循环内部判断退出条件

---

## 3.3.5 while 的类型是 Unit

和没有 `else` 的 `if` 一样，`while` 表达式的类型固定为 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = while (i < 3) {
        println(i)
        i++
    }
    println(result)     // 输出：()
}
```

因为 `while` 本身的值没有实际意义，所以我们通常不把它赋值给变量，而是直接使用。

---

## 3.3.6 while 与 for-in 怎么选？

刚开始学的时候容易混淆这两个。简单区分：

| 场景 | 推荐 | 原因 |
|------|------|------|
| 不确定要循环多少次，依赖某个条件 | `while` | 条件判断更灵活 |
| 遍历数组/区间中的每个元素 | `for-in` | 更简洁，自动取下一个 |
| 需要计数器循环 | 两者都可以 | `while` 手动控制，`for-in` 自动控制 |

```Cangjie
// while：手动控制计数器
var i = 0
while (i < 5) {
    println(i)
    i++     // 手动增加
}

// for-in：自动控制计数器（后续章节学）
for (i in 0..5) {
    println(i)  // 自动取下一个 i
}
```

## 3.3.7 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本语法 | `while (条件) { 循环体 }` | 条件满足就一直做 |
| 条件类型 | 必须是 `Bool`（和 `if` 一样） | 不要 `while(1)` |
| 执行流程 | 判断 → 执行 → 再判断 → 再执行 → …… → 结束 | 先看再做 |
| 死循环 | 条件永远为 `true` 时产生 | 忘记改变条件就会发生 |
| 退出循环 | 条件变为 `false` 时自动退出 | 或者用 `break`（后面学） |
| while 的类型 | `Unit`（值没有实际意义） | 一般不用赋值 |

## 常见问题

### Q1：`while` 的条件什么时候检查？

**每次循环开始前检查。** 如果一开始条件就是 `false`，循环体一次都不会执行：

```Cangjie
main() {
    var i = 10
    while (i < 5) {     // 条件一开始就是 false
        println("不会执行")
    }
}
```

### Q2：`while` 能嵌套使用吗？

**可以。** `while` 里面可以再套一个 `while`，实现二维循环（比如遍历表格）：

```Cangjie
main() {
    var i = 0
    while (i < 3) {
        var j = 0
        while (j < 3) {
            print("(${i},${j}) ")
            j++
        }
        println()   // 换行
        i++
    }
}
```

运行结果：

```Bash
(0,0) (0,1) (0,2)
(1,0) (1,1) (1,2)
(2,0) (2,1) (2,2)
```

> [!TIP]
> 嵌套循环的思路：外层循环控制"行"，内层循环控制"列"，就像打印一个表格。

### Q3：`while` 循环中 `i++` 写在前和写在后有区别吗？

在单独一行时没区别，但如果和别的语句混用就有区别了：

```Cangjie
var i = 0
while (i < 3) {
    println(i)    // i++ 写在这后面 → 输出 0, 1, 2
    i++
}

var i = 0
while (i < 3) {
    i++           // i++ 写在这前面 → 输出 1, 2, 3
    println(i)
}
```

两种都常见，但**建议统一风格**——惯用写法是把 `i++` 放在循环体的末尾。

## 3.3.1 什么是循环？

我们之前学的 `if` 和 `match` 都是"做一次判断，执行一次代码"。但有时候，我们需要**反复执行同一段代码**，直到某个条件不再满足为止。

比如：

> 你还没有吃饱 → 再吃一口 → 还没饱 → 再吃一口 → …… → 饱了 → 停止

这种"反复执行"的机制就叫做**循环**，而 `while` 就是实现它的工具。

### 生活中到处都是循环

| 场景 | 代码逻辑 |
|------|---------|
| 还没吃饱 → 再吃一口 | `while (还没饱) { 吃一口 }` |
| 还没跑完 5 圈 → 继续跑 | `while (跑完的圈数 < 5) { 跑一圈 }` |
| 还有作业没做完 → 继续做 | `while (还有作业) { 做一题 }` |

### while 的执行流程

```
           ┌─────────────────────┐
           │                     │
           ▼                     │
    ┌─ 判断条件 ──→ 条件为真？──→ 执行循环体
    │                              │
    │  条件为假                     │
    ▼                              │
  退出循环 ←───────────────────────┘
```

> [!TIP]
> `while` 就像"自动门"——你走到门前（判断条件），如果门开了（条件为真），就走进去（执行循环体）；如果门关着（条件为假），就不进去了（退出循环）。进去之后，再回头看门关了没有……

---

## 3.3.2 while 的基本用法

`while` 的结构很简单：

```Cangjie
while (条件) {
    // 条件为 true 时反复执行的代码
}
```

执行流程：
1. 检查条件 → 如果为 `true`，进入循环体
2. 执行循环体中的代码
3. 再次检查条件 → 如果还是 `true`，继续执行
4. 重复……直到条件为 `false`，退出循环

### 一个最简单的例子

```Cangjie
main() {
    var count = 0

    while (count < 5) {
        println("第 ${count} 次循环")
        count++
    }
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

> [!TIP]
> `count++` 的意思是 `count = count + 1`，每循环一次让计数器加 1。
>
> **逐轮过程**：
> - 第 1 轮：`count=0`，判断 `0 < 5` → true → 输出"第 0 次"，`count` 变为 1
> - 第 2 轮：`count=1`，判断 `1 < 5` → true → 输出"第 1 次"，`count` 变为 2
> - ……
> - 第 5 轮：`count=4`，判断 `4 < 5` → true → 输出"第 4 次"，`count` 变为 5
> - 第 6 轮：`count=5`，判断 `5 < 5` → false → 退出循环

### 实际例子：二分法求平方根

```Cangjie
main() {
    var root = 0.0
    var min = 1.0
    var max = 2.0
    var error = 1.0
    let tolerance = 0.1 ** 10

    while (error ** 2 > tolerance) {
        root = (min + max) / 2.0
        error = root ** 2 - 2.0
        if (error > 0.0) {
            max = root
        } else {
            min = root
        }
    }
    println("2 的平方根约等于：${root}")
}
```

运行结果：

```Bash
2 的平方根约等于：1.414215
```

这个程序用**二分法**不断逼近 2 的平方根——每次循环将范围缩小一半，直到误差足够小。

> [!NOTE]
> 这个例子需要一些数学基础，看不懂也没关系。关键是理解 `while` 的用法：**在条件满足时不断重复，条件不满足时结束**。

## 3.3.3 条件必须是布尔值

和 `if` 一样，`while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    let x = 1
    while (x) {         // ❌ 编译错误：不能把 Int64 当作条件
        println("无限循环？")
    }
}
```

必须写成明确的布尔表达式：

```Cangjie
main() {
    var x = 5
    while (x > 0) {     // ✅ 正确
        println(x)
        x--
    }
}
```

> [!CAUTION]
> 这是和 `if` 一样的规则。仓颉不会帮你把数字自动转成布尔值——你必须自己写比较条件。

## 3.3.4 死循环与终止

如果条件**永远为 `true`**，循环会一直执行下去，这就是"死循环"（无限循环）：

```Cangjie
main() {
    while (true) {
        println("停不下来啦！")
    }
}
```

> [!WARNING]
> **新手的噩梦：不小心写出死循环**
>
> 最常见的死循环原因：**忘记改变条件**。
>
> 比如上面那个 `count < 5` 的例子，如果忘记写 `count++`：
> ```Cangjie
> var count = 0
> while (count < 5) {
>     println("第 ${count} 次循环")
>     // 忘记写 count++ 了！！！count 永远是 0
> }
> // count < 5 永远为 true → 无限循环
> ```

**死循环不全是坏事！** 有些程序需要一直运行（比如服务器、游戏主循环、实时监控），这时候 `while (true)` 就是有意为之：

```Cangjie
// 服务器主循环：一直运行，等待客户端请求
while (true) {
    let request = waitForRequest()
    handleRequest(request)
}
```

### 如何终止死循环？

- 如果是在终端中运行的程序，按 `Ctrl+C` 强制终止
- 如果是 IDE 中运行，点击"停止"按钮
- 更好的方式：用 `while (true)` + `break` 组合，在循环内部判断退出条件

## 3.3.5 while 的类型是 Unit

和没有 `else` 的 `if` 一样，`while` 表达式的类型固定为 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = while (i < 3) {
        println(i)
        i++
    }
    println(result)     // 输出：()
}
```

因为 `while` 本身的值没有实际意义，所以我们通常不把它赋值给变量，而是直接使用。

## 3.3.6 while 与 for-in 怎么选？

刚开始学的时候容易混淆这两个。简单区分：

| 场景 | 推荐 | 原因 |
|------|------|------|
| 不确定要循环多少次，依赖某个条件 | `while` | 条件判断更灵活 |
| 遍历数组/区间中的每个元素 | `for-in` | 更简洁，自动取下一个 |
| 需要计数器循环 | 两者都可以 | `while` 手动控制，`for-in` 自动控制 |

```Cangjie
// while：手动控制计数器
var i = 0
while (i < 5) {
    println(i)
    i++     // 手动增加
}

// for-in：自动控制计数器（后续章节学）
for (i in 0..5) {
    println(i)  // 自动取下一个 i
}
```

## 3.3.7 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本语法 | `while (条件) { 循环体 }` | 条件满足就一直做 |
| 条件类型 | 必须是 `Bool`（和 `if` 一样） | 不要 `while(1)` |
| 执行流程 | 判断 → 执行 → 再判断 → 再执行 → …… → 结束 | 先看再做 |
| 死循环 | 条件永远为 `true` 时产生 | 忘记改变条件就会发生 |
| 退出循环 | 条件变为 `false` 时自动退出 | 或者用 `break`（后面学） |
| while 的类型 | `Unit`（值没有实际意义） | 一般不用赋值 |

## 常见问题

### Q1：`while` 的条件什么时候检查？

**每次循环开始前检查。** 如果一开始条件就是 `false`，循环体一次都不会执行：

```Cangjie
main() {
    var i = 10
    while (i < 5) {     // 条件一开始就是 false
        println("不会执行")
    }
}
```

### Q2：`while` 能嵌套使用吗？

**可以。** `while` 里面可以再套一个 `while`，实现二维循环（比如遍历表格）：

```Cangjie
main() {
    var i = 0
    while (i < 3) {
        var j = 0
        while (j < 3) {
            print("(${i},${j}) ")
            j++
        }
        println()   // 换行
        i++
    }
}
```

运行结果：

```Bash
(0,0) (0,1) (0,2)
(1,0) (1,1) (1,2)
(2,0) (2,1) (2,2)
```

> [!TIP]
> 嵌套循环的思路：外层循环控制"行"，内层循环控制"列"，就像打印一个表格。

### Q3：`while` 循环中 `i++` 写在前和写在后有区别吗？

在单独一行时没区别，但如果和别的语句混用就有区别了：

```Cangjie
var i = 0
while (i < 3) {
    println(i)    // i++ 写在这后面 → 输出 0, 1, 2
    i++
}

var i = 0
while (i < 3) {
    i++           // i++ 写在这前面 → 输出 1, 2, 3
    println(i)
}
```

两种都常见，但**建议统一风格**——惯用写法是把 `i++` 放在循环体的末尾。

## 3.3.1 什么是循环？

我们之前学的 `if` 和 `match` 都是"做一次判断，执行一次代码"。但有时候，我们需要**反复执行同一段代码**，直到某个条件不再满足为止。

比如：

> 你还没有吃饱 → 再吃一口 → 还没饱 → 再吃一口 → …… → 饱了 → 停止

这种"反复执行"的机制就叫做**循环**，而 `while` 就是实现它的工具。

### 生活中到处都是循环

| 场景 | 代码逻辑 |
|------|---------|
| 还没吃饱 → 再吃一口 | `while (还没饱) { 吃一口 }` |
| 还没跑完 5 圈 → 继续跑 | `while (跑完的圈数 < 5) { 跑一圈 }` |
| 还有作业没做完 → 继续做 | `while (还有作业) { 做一题 }` |

### while 的执行流程

```
           ┌─────────────────────┐
           │                     │
           ▼                     │
    ┌─ 判断条件 ──→ 条件为真？──→ 执行循环体
    │                              │
    │  条件为假                     │
    ▼                              │
  退出循环 ←───────────────────────┘
```

> [!TIP]
> `while` 就像"自动门"——你走到门前（判断条件），如果门开了（条件为真），就走进去（执行循环体）；如果门关着（条件为假），就不进去了（退出循环）。进去之后，再回头看门关了没有……

---

## 3.3.2 while 的基本用法

`while` 的结构很简单：

```Cangjie
while (条件) {
    // 条件为 true 时反复执行的代码
}
```

执行流程：
1. 检查条件 → 如果为 `true`，进入循环体
2. 执行循环体中的代码
3. 再次检查条件 → 如果还是 `true`，继续执行
4. 重复……直到条件为 `false`，退出循环

### 一个最简单的例子

```Cangjie
main() {
    var count = 0

    while (count < 5) {
        println("第 ${count} 次循环")
        count++
    }
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

> [!TIP]
> `count++` 的意思是 `count = count + 1`，每循环一次让计数器加 1。
>
> **逐轮过程**：
> - 第 1 轮：`count=0`，判断 `0 < 5` → true → 输出"第 0 次"，`count` 变为 1
> - 第 2 轮：`count=1`，判断 `1 < 5` → true → 输出"第 1 次"，`count` 变为 2
> - ……
> - 第 5 轮：`count=4`，判断 `4 < 5` → true → 输出"第 4 次"，`count` 变为 5
> - 第 6 轮：`count=5`，判断 `5 < 5` → false → 退出循环

### 实际例子：二分法求平方根

```Cangjie
main() {
    var root = 0.0
    var min = 1.0
    var max = 2.0
    var error = 1.0
    let tolerance = 0.1 ** 10

    while (error ** 2 > tolerance) {
        root = (min + max) / 2.0
        error = root ** 2 - 2.0
        if (error > 0.0) {
            max = root
        } else {
            min = root
        }
    }
    println("2 的平方根约等于：${root}")
}
```

运行结果：

```Bash
2 的平方根约等于：1.414215
```

这个程序用**二分法**不断逼近 2 的平方根——每次循环将范围缩小一半，直到误差足够小。

> [!NOTE]
> 这个例子需要一些数学基础，看不懂也没关系。关键是理解 `while` 的用法：**在条件满足时不断重复，条件不满足时结束**。

---

## 3.3.3 条件必须是布尔值

和 `if` 一样，`while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    let x = 1
    while (x) {         // ❌ 编译错误：不能把 Int64 当作条件
        println("无限循环？")
    }
}
```

必须写成明确的布尔表达式：

```Cangjie
main() {
    var x = 5
    while (x > 0) {     // ✅ 正确
        println(x)
        x--
    }
}
```

> [!CAUTION]
> 这是和 `if` 一样的规则。仓颉不会帮你把数字自动转成布尔值——你必须自己写比较条件。

---

## 3.3.4 死循环与终止

如果条件**永远为 `true`**，循环会一直执行下去，这就是"死循环"（无限循环）：

```Cangjie
main() {
    while (true) {
        println("停不下来啦！")
    }
}
```

> [!WARNING]
> **新手的噩梦：不小心写出死循环**
>
> 最常见的死循环原因：**忘记改变条件**。
>
> 比如上面那个 `count < 5` 的例子，如果忘记写 `count++`：
> ```Cangjie
> var count = 0
> while (count < 5) {
>     println("第 ${count} 次循环")
>     // 忘记写 count++ 了！！！count 永远是 0
> }
> // count < 5 永远为 true → 无限循环
> ```

**死循环不全是坏事！** 有些程序需要一直运行（比如服务器、游戏主循环、实时监控），这时候 `while (true)` 就是有意为之：

```Cangjie
// 服务器主循环：一直运行，等待客户端请求
while (true) {
    let request = waitForRequest()
    handleRequest(request)
}
```

### 如何终止死循环？

- 如果是在终端中运行的程序，按 `Ctrl+C` 强制终止
- 如果是 IDE 中运行，点击"停止"按钮
- 更好的方式：用 `while (true)` + `break` 组合，在循环内部判断退出条件

---

## 3.3.5 while 的类型是 Unit

和没有 `else` 的 `if` 一样，`while` 表达式的类型固定为 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = while (i < 3) {
        println(i)
        i++
    }
    println(result)     // 输出：()
}
```

因为 `while` 本身的值没有实际意义，所以我们通常不把它赋值给变量，而是直接使用。

---

## 3.3.6 while 与 for-in 怎么选？

刚开始学的时候容易混淆这两个。简单区分：

| 场景 | 推荐 | 原因 |
|------|------|------|
| 不确定要循环多少次，依赖某个条件 | `while` | 条件判断更灵活 |
| 遍历数组/区间中的每个元素 | `for-in` | 更简洁，自动取下一个 |
| 需要计数器循环 | 两者都可以 | `while` 手动控制，`for-in` 自动控制 |

```Cangjie
// while：手动控制计数器
var i = 0
while (i < 5) {
    println(i)
    i++     // 手动增加
}

// for-in：自动控制计数器（后续章节学）
for (i in 0..5) {
    println(i)  // 自动取下一个 i
}
```

---

## 3.3.7 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| 基本语法 | `while (条件) { 循环体 }` | 条件满足就一直做 |
| 条件类型 | 必须是 `Bool`（和 `if` 一样） | 不要 `while(1)` |
| 执行流程 | 判断 → 执行 → 再判断 → 再执行 → …… → 结束 | 先看再做 |
| 死循环 | 条件永远为 `true` 时产生 | 忘记改变条件就会发生 |
| 退出循环 | 条件变为 `false` 时自动退出 | 或者用 `break`（后面学） |
| while 的类型 | `Unit`（值没有实际意义） | 一般不用赋值 |

## 常见问题

### Q1：`while` 的条件什么时候检查？

**每次循环开始前检查。** 如果一开始条件就是 `false`，循环体一次都不会执行：

```Cangjie
main() {
    var i = 10
    while (i < 5) {     // 条件一开始就是 false
        println("不会执行")
    }
}
```

### Q2：`while` 能嵌套使用吗？

**可以。** `while` 里面可以再套一个 `while`，实现二维循环（比如遍历表格）：

```Cangjie
main() {
    var i = 0
    while (i < 3) {
        var j = 0
        while (j < 3) {
            print("(${i},${j}) ")
            j++
        }
        println()   // 换行
        i++
    }
}
```

运行结果：

```Bash
(0,0) (0,1) (0,2)
(1,0) (1,1) (1,2)
(2,0) (2,1) (2,2)
```

> [!TIP]
> 嵌套循环的思路：外层循环控制"行"，内层循环控制"列"，就像打印一个表格。

    println(i)
