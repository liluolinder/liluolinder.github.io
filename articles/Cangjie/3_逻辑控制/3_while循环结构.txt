## 3.3.1 什么是循环？

我们之前学的 `if` 和 `match` 都是"做一次判断，执行一次代码"。但有时候，我们需要**反复执行同一段代码**，直到某个条件不再满足为止。

比如：

> 你还没有吃饱 → 再吃一口 → 还没饱 → 再吃一口 → …… → 饱了 → 停止

这种"反复执行"的机制就叫做**循环**，而 `while` 就是实现它的工具。

---

## 3.3.2 while 的基本用法

`while` 的结构很简单：

```Cangjie
while (条件) {
    // 条件为 true 时反复执行的代码
}
```

执行流程就像一个"原地打转"的循环：

1. 检查条件 → 如果为 `true`，进入循环体
2. 执行循环体中的代码
3. 再次检查条件 → 如果还是 `true`，继续执行
4. 重复……直到条件为 `false`，退出循环

### 一个最简单的例子

```Cangjie
main() {
    var count = 0

    while (count < 5) {
        println("第 ${count} 次循环")  //这里是插值字符串哦 还记得吗 不记得到前面去看字符串类型哦
        count++
    }
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

> [!TIP]
> `count++` 的意思是 `count = count + 1`，每循环一次让计数器加 1，最终 `count` 变成 5，条件 `count < 5` 为 `false`，循环结束。

### 实际例子：二分法求平方根

```Cangjie
main() {
    var root = 0.0
    var min = 1.0
    var max = 2.0
    var error = 1.0
    let tolerance = 0.1 ** 10

    while (error ** 2 > tolerance) {
        root = (min + max) / 2.0
        error = root ** 2 - 2.0
        if (error > 0.0) {
            max = root
        } else {
            min = root
        }
    }
    println("2 的平方根约等于：${root}")
}
```

运行结果：

```Bash
2 的平方根约等于：1.414215
```

这个程序用**二分法**不断逼近 2 的平方根——每次循环将范围缩小一半，直到误差足够小。

---

## 3.3.3 条件必须是布尔值

和 `if` 一样，`while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    let x = 1
    while (x) {         // ❌ 编译错误：不能把 Int64 当作条件
        println("无限循环？")
    }
}
```

必须写成明确的布尔表达式：

```Cangjie
main() {
    var x = 5
    while (x > 0) {     // ✅ 正确
        println(x)
        x--
    }
}
```

---

## 3.3.4 死循环与终止

如果条件**永远为 `true`**，循环会一直执行下去，这就是"死循环"（无限循环）：

```Cangjie
main() {
    while (true) {
        println("停不下来啦！")
    }
}
```

> [!WARNING]
> 死循环通常是因为**忘记改变条件**导致的。比如上面那个 `count < 5` 的例子，如果忘记写 `count++`，`count` 永远为 0，循环永远不会结束。

死循环不全是坏事——有些程序需要一直运行（比如服务器、游戏主循环），这时候 `while (true)` 就是有意为之。

---

## 3.3.5 while 的类型是 Unit

和没有 `else` 的 `if` 一样，`while` 表达式的类型固定为 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = while (i < 3) {
        println(i)
        i++
    }
    println(result)     // 输出：()
}
```

因为 `while` 本身的值没有实际意义，所以我们通常不把它赋值给变量，而是直接使用。

---

## 3.3.6 小结

| 概念 | 要点 |
|------|------|
| 基本语法 | `while (条件) { 循环体 }` |
| 条件类型 | 必须是 `Bool`（和 `if` 一样） |
| 执行流程 | 判断 → 执行 → 再判断 → 再执行 → …… → 结束 |
| 死循环 | 条件永远为 `true` 时产生 |
| 退出循环 | 条件变为 `false` 时自动退出 |
| while 的类型 | `Unit`（值没有实际意义） |

## 常见问题

### Q1：`while` 的条件什么时候检查？

**每次循环开始前检查。** 如果一开始条件就是 `false`，循环体一次都不会执行：

```Cangjie
main() {
    var i = 10
    while (i < 5) {     // 条件一开始就是 false
        println("不会执行")
    }
}
```

### Q2：`while` 能嵌套使用吗？

**可以。** `while` 里面可以再套一个 `while`，实现二维循环（比如遍历表格）：

```Cangjie
main() {
    var i = 0
    while (i < 3) {
        var j = 0
        while (j < 3) {
            print("(${i},${j}) ")
            j++
        }
        println()   // 换行
        i++
    }
}
```

运行结果：

```Bash
(0,0) (0,1) (0,2)
(1,0) (1,1) (1,2)
(2,0) (2,1) (2,2)
```

## 3.3.1 什么是循环？

我们之前学的 `if` 和 `match` 都是"做一次判断，执行一次代码"。但有时候，我们需要**反复执行同一段代码**，直到某个条件不再满足为止。

比如：

> 你还没有吃饱 → 再吃一口 → 还没饱 → 再吃一口 → …… → 饱了 → 停止

这种"反复执行"的机制就叫做**循环**，而 `while` 就是实现它的工具。

---

## 3.3.2 while 的基本用法

`while` 的结构很简单：

```Cangjie
while (条件) {
    // 条件为 true 时反复执行的代码
}
```

执行流程就像一个"原地打转"的循环：

1. 检查条件 → 如果为 `true`，进入循环体
2. 执行循环体中的代码
3. 再次检查条件 → 如果还是 `true`，继续执行
4. 重复……直到条件为 `false`，退出循环

### 一个最简单的例子

```Cangjie
main() {
    var count = 0

    while (count < 5) {
        println("第 ${count} 次循环")  //这里是插值字符串哦 还记得吗 不记得到前面去看字符串类型哦
        count++
    }
}
```

运行结果：

```Bash
第 0 次循环
第 1 次循环
第 2 次循环
第 3 次循环
第 4 次循环
```

> [!TIP]
> `count++` 的意思是 `count = count + 1`，每循环一次让计数器加 1，最终 `count` 变成 5，条件 `count < 5` 为 `false`，循环结束。

### 实际例子：二分法求平方根

```Cangjie
main() {
    var root = 0.0
    var min = 1.0
    var max = 2.0
    var error = 1.0
    let tolerance = 0.1 ** 10

    while (error ** 2 > tolerance) {
        root = (min + max) / 2.0
        error = root ** 2 - 2.0
        if (error > 0.0) {
            max = root
        } else {
            min = root
        }
    }
    println("2 的平方根约等于：${root}")
}
```

运行结果：

```Bash
2 的平方根约等于：1.414215
```

这个程序用**二分法**不断逼近 2 的平方根——每次循环将范围缩小一半，直到误差足够小。

---

## 3.3.3 条件必须是布尔值

和 `if` 一样，`while` 的条件也**必须**是 `Bool` 类型：

```Cangjie
main() {
    let x = 1
    while (x) {         // ❌ 编译错误：不能把 Int64 当作条件
        println("无限循环？")
    }
}
```

必须写成明确的布尔表达式：

```Cangjie
main() {
    var x = 5
    while (x > 0) {     // ✅ 正确
        println(x)
        x--
    }
}
```

---

## 3.3.4 死循环与终止

如果条件**永远为 `true`**，循环会一直执行下去，这就是"死循环"（无限循环）：

```Cangjie
main() {
    while (true) {
        println("停不下来啦！")
    }
}
```

> [!WARNING]
> 死循环通常是因为**忘记改变条件**导致的。比如上面那个 `count < 5` 的例子，如果忘记写 `count++`，`count` 永远为 0，循环永远不会结束。

死循环不全是坏事——有些程序需要一直运行（比如服务器、游戏主循环），这时候 `while (true)` 就是有意为之。

---

## 3.3.5 while 的类型是 Unit

和没有 `else` 的 `if` 一样，`while` 表达式的类型固定为 **`Unit`**，值为 `()`：

```Cangjie
main() {
    var i = 0
    let result = while (i < 3) {
        println(i)
        i++
    }
    println(result)     // 输出：()
}
```

因为 `while` 本身的值没有实际意义，所以我们通常不把它赋值给变量，而是直接使用。

---

## 3.3.6 小结

| 概念 | 要点 |
|------|------|
| 基本语法 | `while (条件) { 循环体 }` |
| 条件类型 | 必须是 `Bool`（和 `if` 一样） |
| 执行流程 | 判断 → 执行 → 再判断 → 再执行 → …… → 结束 |
| 死循环 | 条件永远为 `true` 时产生 |
| 退出循环 | 条件变为 `false` 时自动退出 |
| while 的类型 | `Unit`（值没有实际意义） |

---

## 常见问题

### Q1：`while` 的条件什么时候检查？

**每次循环开始前检查。** 如果一开始条件就是 `false`，循环体一次都不会执行：

```Cangjie
main() {
    var i = 10
    while (i < 5) {     // 条件一开始就是 false
        println("不会执行")
    }
}
```

### Q2：`while` 能嵌套使用吗？

**可以。** `while` 里面可以再套一个 `while`，实现二维循环（比如遍历表格）：

```Cangjie
main() {
    var i = 0
    while (i < 3) {
        var j = 0
        while (j < 3) {
            print("(${i},${j}) ")
            j++
        }
        println()   // 换行
        i++
    }
}
```

运行结果：

```Bash
(0,0) (0,1) (0,2)
(1,0) (1,1) (1,2)
(2,0) (2,1) (2,2)
