## 3.6.1 为什么要循环控制？

前面我们学了三种循环：`while`、`do-while`、`for-in`。它们都是"条件满足就一直执行"。

但有时候你需要在循环中途**提前结束**或**跳过某次**：

> 比如你在一个水果篮里找橘子：
> - 找到橘子后，后面的水果就不用看了 → **`break`**
> - 看到苹果，不感兴趣，直接看下一个 → **`continue`**

仓颉提供了两个循环控制表达式：**`break`** 和 **`continue`**。

### 一句话区分

| 关键字 | 作用 | 生活中的类比 |
|--------|------|-------------|
| `break` | **终止整个循环**，不再执行 | 找到橘子了，后面的水果不看了 |
| `continue` | **跳过本轮循环**，进入下一轮 | 看到苹果，跳过，看下一个 |

> [!NOTE]
> `break` 和 `continue` 的类型都是 **`Nothing`**（一个特殊的类型，表示"永远不会有值"）。

---

## 3.6.2 break：提前终止循环

`break` 用于**立即结束整个循环**，执行循环后面的代码。

### 例子：找到第一个能被 5 整除的数

```Cangjie
main() {
    let numbers = [12, 18, 25, 36, 49, 55]
    for (number in numbers) {
        if (number % 5 == 0) {
            println(number)
            break       // 找到后立即结束循环
        }
    }
}
```

运行结果：

```Bash
25
```

> [!TIP]
> 如果没有 `break`，循环会遍历所有数字，输出 25 和 55（两个都能被 5 整除）。
>
> 有了 `break`，找到第一个（25）就停，后面的 36、49、55 都不再检查了。这在处理大量数据时能**节省时间**。

### 执行过程演示

```Cangjie
let numbers = [12, 18, 25, 36, 49, 55]
```

| 轮次 | number | number % 5 == 0? | 是否 break? | 输出 |
|------|--------|-----------------|-------------|------|
| 1 | 12 | 12 % 5 = 2，false | 否 | — |
| 2 | 18 | 18 % 5 = 3，false | 否 | — |
| 3 | **25** | **25 % 5 = 0，true** | **是** | **25** |
| 4~6 | — | — | 已结束，不执行 | — |

### break 在 while 中使用

```Cangjie
main() {
    var i = 0
    while (true) {          // 死循环
        if (i >= 5) {
            break           // 条件满足时退出
        }
        println(i)
        i++
    }
}
```

运行结果：

```Bash
0
1
2
3
4
```

> [!NOTE]
> `while (true)` + `break` 是一种常见写法——在循环体中间判断退出条件，比写在 `while` 条件里更灵活。
>
> 比如：如果你需要在循环体**中间**检查退出条件，用 `while(true) + break` 比把条件写得复杂更清晰。

---

## 3.6.3 continue：跳过本轮循环

`continue` 用于**提前结束本轮循环**，直接进入下一轮。

### 例子：打印数组中的奇数

```Cangjie
main() {
    let numbers = [12, 18, 25, 36, 49, 55]
    for (number in numbers) {
        if (number % 2 == 0) {
            continue        // 偶数跳过，不打印
        }
        println(number)
    }
}
```

运行结果：

```Bash
25
49
55
```

### 执行过程演示

| 轮次 | number | number % 2 == 0? | 是否 continue? | 输出 |
|------|--------|-----------------|---------------|------|
| 1 | 12 | true（偶数） | **是，跳过** | — |
| 2 | 18 | true（偶数） | **是，跳过** | — |
| 3 | 25 | false（奇数） | 否 | **25** |
| 4 | 36 | true（偶数） | **是，跳过** | — |
| 5 | 49 | false（奇数） | 否 | **49** |
| 6 | 55 | false（奇数） | 否 | **55** |

### continue 在 while 中使用

```Cangjie
main() {
    var i = 0
    while (i < 10) {
        i++
        if (i % 2 == 0) {
            continue        // 偶数跳过
        }
        println(i)           // 只打印奇数
    }
}
```

运行结果：

```Bash
1
3
5
7
9
```

> [!WARNING]
> **在 `while` 中使用 `continue` 要特别小心！**
>
> 如果 `continue` 在变量更新**之前**执行，可能导致死循环：
>
> ```Cangjie
> var i = 0
> while (i < 10) {
>     if (i % 2 == 0) {
>         continue    // ❌ 危险！i 还没增加就 continue 了，i 永远是 0
>     }
>     i++             // 这行被 continue 跳过了！
> }
> ```
>
> **解决方法**：把变量更新放在 `continue` 之前，或者用 `for-in` 代替 `while`（for-in 会自动取下一个值，不受 continue 影响）。

---

## 3.6.4 break vs continue 对比

| 对比项 | `break` | `continue` |
|--------|---------|-----------|
| 作用 | **终止整个循环** | **跳过本轮循环** |
| 循环是否继续 | ❌ 不再继续 | ✅ 继续下一轮 |
| 后续执行 | 执行循环后面的代码 | 进入下一轮循环 |
| 适用场景 | 找到目标后就不需要继续找了 | 对当前元素不感兴趣，看下一个 |
| 类比 | 找到橘子，后面的水果不看了 | 看到苹果，跳过，看下一个 |

### 对比示意图

```
遍历 [12, 18, 25, 36, 49, 55]:

break（找第一个能被 5 整除的）：
  [12] → [18] → [25] → ⛔ 找到了，不看了

continue（打印所有奇数）：
  [12] → 跳过 → [18] → 跳过 → [25] → 打印 → [36]
   → 跳过 → [49] → 打印 → [55] → 打印 → ✅ 遍历完
```

---

## 3.6.5 break 和 continue 的类型

`break` 和 `continue` 的类型都是 **`Nothing`**（不是 `Unit`）。

`Nothing` 是一个特殊的类型，表示"永远不会有值"——因为 `break` 和 `continue` 一旦执行，就会跳出或跳过循环，根本不会返回任何值。

> [!TIP]
> 你不需要主动使用 `Nothing` 类型，了解即可。它和 `Unit` 的区别：
> - `Unit`：有值，值是 `()`（"有一个空值"）
> - `Nothing`：永远不会有值（"根本不会执行到这里"）

---

## 3.6.6 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| `break` | 提前**终止**整个循环 | 找到了，不用再找了 |
| `continue` | **跳过**本轮循环，进入下一轮 | 不感兴趣，看下一个 |
| 类型 | 都是 `Nothing` | 永远不会有值 |
| 适用循环 | `while`、`do-while`、`for-in` | 所有循环都能用 |
| while 注意 | `continue` 前确保变量已更新 | 否则可能死循环 |

## 常见问题

### Q1：`break` 和 `continue` 能用在嵌套循环中吗？

**可以，但它们只作用于当前所在的循环层**，不会影响外层循环：

```Cangjie
main() {
    for (i in 0..3) {
        for (j in 0..3) {
            if (j == 2) {
                break       // 只跳出内层 for (j)，外层继续
            }
            print("(${i},${j}) ")
        }
        println()
    }
}
```

运行结果：

```Bash
(0,0) (0,1)
(1,0) (1,1)
(2,0) (2,1)
```

**逐轮解释**：
- `i=0`：内层 j 从 0 开始，j=0 输出，j=1 输出，j=2 → **break 内层**，外层换行，i=1
- `i=1`：同上
- `i=2`：同上

> [!TIP]
> 如果想在嵌套循环中**跳出外层循环**，可以借助标志变量：
>
> ```Cangjie
> main() {
>     var found = false
>     for (i in 0..3) {
>         for (j in 0..3) {
>             if (i == 1 && j == 1) {
>                 found = true
>                 break       // 跳出内层
>             }
>             print("(${i},${j}) ")
>         }
>         if (found) {
>             break           // 跳出外层
>         }
>     }
> }
> ```

### Q2：`continue` 在 `while` 和 `for-in` 中的行为一样吗？

**不完全一样。** 这是一个重要区别：

- 在 `for-in` 中，`continue` 后会自动取下一个元素——**安全**
- 在 `while` 中，`continue` 后不会自动更新变量——**需要小心**

```Cangjie
// for-in：安全，continue 后自动取下一个元素
for (i in 0..5) {
    if (i % 2 == 0) { continue }
    println(i)
}

// while：需要确保 continue 前变量已更新
var i = 0
while (i < 5) {
    i++                 // ✅ 在 continue 之前更新
    if (i % 2 == 0) { continue }
    println(i)
}
```

### Q3：`break` 能用在 `if` 中吗？

**不能。** `break` 和 `continue` 只能用在循环体（`while`、`do-while`、`for-in`）中，不能用在 `if` 里（除非 `if` 在循环体中）。

```Cangjie
if (true) {
    break       // ❌ 编译报错：break 不在循环中
}

for (i in 0..5) {
    if (i == 3) {
        break   // ✅ 正确：break 在 for-in 循环体中
    }
}
```

### Q4：`break` 和 `return` 有什么区别？

- `break`：**退出当前循环**，继续执行循环后面的代码
- `return`：**退出整个函数**，回到函数的调用处

```Cangjie
func search() {
    for (i in 0..10) {
        if (i == 5) {
            break       // 只退出 for 循环，然后执行 println("完成")
        }
    }
    println("完成")     // break 后执行到这里 ✅
}

func search() {
    for (i in 0..10) {
        if (i == 5) {
            return      // 退出整个函数，不会执行 println("完成")
        }
    }
    println("完成")     // return 后不会执行到这里 ❌
}

## 3.6.1 为什么要循环控制？

前面我们学了三种循环：`while`、`do-while`、`for-in`。它们都是"条件满足就一直执行"。

但有时候你需要在循环中途**提前结束**或**跳过某次**：

> 比如你在一个水果篮里找橘子：
> - 找到橘子后，后面的水果就不用看了 → **`break`**
> - 看到苹果，不感兴趣，直接看下一个 → **`continue`**

仓颉提供了两个循环控制表达式：**`break`** 和 **`continue`**。

### 一句话区分

| 关键字 | 作用 | 生活中的类比 |
|--------|------|-------------|
| `break` | **终止整个循环**，不再执行 | 找到橘子了，后面的水果不看了 |
| `continue` | **跳过本轮循环**，进入下一轮 | 看到苹果，跳过，看下一个 |

> [!NOTE]
> `break` 和 `continue` 的类型都是 **`Nothing`**（一个特殊的类型，表示"永远不会有值"）。

---

## 3.6.2 break：提前终止循环

`break` 用于**立即结束整个循环**，执行循环后面的代码。

### 例子：找到第一个能被 5 整除的数

```Cangjie
main() {
    let numbers = [12, 18, 25, 36, 49, 55]
    for (number in numbers) {
        if (number % 5 == 0) {
            println(number)
            break       // 找到后立即结束循环
        }
    }
}
```

运行结果：

```Bash
25
```

> [!TIP]
> 如果没有 `break`，循环会遍历所有数字，输出 25 和 55（两个都能被 5 整除）。
>
> 有了 `break`，找到第一个（25）就停，后面的 36、49、55 都不再检查了。这在处理大量数据时能**节省时间**。

### 执行过程演示

```Cangjie
let numbers = [12, 18, 25, 36, 49, 55]
```

| 轮次 | number | number % 5 == 0? | 是否 break? | 输出 |
|------|--------|-----------------|-------------|------|
| 1 | 12 | 12 % 5 = 2，false | 否 | — |
| 2 | 18 | 18 % 5 = 3，false | 否 | — |
| 3 | **25** | **25 % 5 = 0，true** | **是** | **25** |
| 4~6 | — | — | 已结束，不执行 | — |

### break 在 while 中使用

```Cangjie
main() {
    var i = 0
    while (true) {          // 死循环
        if (i >= 5) {
            break           // 条件满足时退出
        }
        println(i)
        i++
    }
}
```

运行结果：

```Bash
0
1
2
3
4
```

> [!NOTE]
> `while (true)` + `break` 是一种常见写法——在循环体中间判断退出条件，比写在 `while` 条件里更灵活。
>
> 比如：如果你需要在循环体**中间**检查退出条件，用 `while(true) + break` 比把条件写得复杂更清晰。

---

## 3.6.3 continue：跳过本轮循环

`continue` 用于**提前结束本轮循环**，直接进入下一轮。

### 例子：打印数组中的奇数

```Cangjie
main() {
    let numbers = [12, 18, 25, 36, 49, 55]
    for (number in numbers) {
        if (number % 2 == 0) {
            continue        // 偶数跳过，不打印
        }
        println(number)
    }
}
```

运行结果：

```Bash
25
49
55
```

### 执行过程演示

| 轮次 | number | number % 2 == 0? | 是否 continue? | 输出 |
|------|--------|-----------------|---------------|------|
| 1 | 12 | true（偶数） | **是，跳过** | — |
| 2 | 18 | true（偶数） | **是，跳过** | — |
| 3 | 25 | false（奇数） | 否 | **25** |
| 4 | 36 | true（偶数） | **是，跳过** | — |
| 5 | 49 | false（奇数） | 否 | **49** |
| 6 | 55 | false（奇数） | 否 | **55** |

### continue 在 while 中使用

```Cangjie
main() {
    var i = 0
    while (i < 10) {
        i++
        if (i % 2 == 0) {
            continue        // 偶数跳过
        }
        println(i)           // 只打印奇数
    }
}
```

运行结果：

```Bash
1
3
5
7
9
```

> [!WARNING]
> **在 `while` 中使用 `continue` 要特别小心！**
>
> 如果 `continue` 在变量更新**之前**执行，可能导致死循环：
>
> ```Cangjie
> var i = 0
> while (i < 10) {
>     if (i % 2 == 0) {
>         continue    // ❌ 危险！i 还没增加就 continue 了，i 永远是 0
>     }
>     i++             // 这行被 continue 跳过了！
> }
> ```
>
> **解决方法**：把变量更新放在 `continue` 之前，或者用 `for-in` 代替 `while`（for-in 会自动取下一个值，不受 continue 影响）。

## 3.6.4 break vs continue 对比

| 对比项 | `break` | `continue` |
|--------|---------|-----------|
| 作用 | **终止整个循环** | **跳过本轮循环** |
| 循环是否继续 | ❌ 不再继续 | ✅ 继续下一轮 |
| 后续执行 | 执行循环后面的代码 | 进入下一轮循环 |
| 适用场景 | 找到目标后就不需要继续找了 | 对当前元素不感兴趣，看下一个 |
| 类比 | 找到橘子，后面的水果不看了 | 看到苹果，跳过，看下一个 |

### 对比示意图

```
遍历 [12, 18, 25, 36, 49, 55]:

break（找第一个能被 5 整除的）：
  [12] → [18] → [25] → ⛔ 找到了，不看了

continue（打印所有奇数）：
  [12] → 跳过 → [18] → 跳过 → [25] → 打印 → [36]
   → 跳过 → [49] → 打印 → [55] → 打印 → ✅ 遍历完
```

## 3.6.5 break 和 continue 的类型

`break` 和 `continue` 的类型都是 **`Nothing`**（不是 `Unit`）。

`Nothing` 是一个特殊的类型，表示"永远不会有值"——因为 `break` 和 `continue` 一旦执行，就会跳出或跳过循环，根本不会返回任何值。

> [!TIP]
> 你不需要主动使用 `Nothing` 类型，了解即可。它和 `Unit` 的区别：
> - `Unit`：有值，值是 `()`（"有一个空值"）
> - `Nothing`：永远不会有值（"根本不会执行到这里"）

## 3.6.6 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| `break` | 提前**终止**整个循环 | 找到了，不用再找了 |
| `continue` | **跳过**本轮循环，进入下一轮 | 不感兴趣，看下一个 |
| 类型 | 都是 `Nothing` | 永远不会有值 |
| 适用循环 | `while`、`do-while`、`for-in` | 所有循环都能用 |
| while 注意 | `continue` 前确保变量已更新 | 否则可能死循环 |

## 常见问题

### Q1：`break` 和 `continue` 能用在嵌套循环中吗？

**可以，但它们只作用于当前所在的循环层**，不会影响外层循环：

```Cangjie
main() {
    for (i in 0..3) {
        for (j in 0..3) {
            if (j == 2) {
                break       // 只跳出内层 for (j)，外层继续
            }
            print("(${i},${j}) ")
        }
        println()
    }
}
```

运行结果：

```Bash
(0,0) (0,1)
(1,0) (1,1)
(2,0) (2,1)
```

**逐轮解释**：
- `i=0`：内层 j 从 0 开始，j=0 输出，j=1 输出，j=2 → **break 内层**，外层换行，i=1
- `i=1`：同上
- `i=2`：同上

> [!TIP]
> 如果想在嵌套循环中**跳出外层循环**，可以借助标志变量：
>
> ```Cangjie
> main() {
>     var found = false
>     for (i in 0..3) {
>         for (j in 0..3) {
>             if (i == 1 && j == 1) {
>                 found = true
>                 break       // 跳出内层
>             }
>             print("(${i},${j}) ")
>         }
>         if (found) {
>             break           // 跳出外层
>         }
>     }
> }
> ```

### Q2：`continue` 在 `while` 和 `for-in` 中的行为一样吗？

**不完全一样。** 这是一个重要区别：

- 在 `for-in` 中，`continue` 后会自动取下一个元素——**安全**
- 在 `while` 中，`continue` 后不会自动更新变量——**需要小心**

```Cangjie
// for-in：安全，continue 后自动取下一个元素
for (i in 0..5) {
    if (i % 2 == 0) { continue }
    println(i)
}

// while：需要确保 continue 前变量已更新
var i = 0
while (i < 5) {
    i++                 // ✅ 在 continue 之前更新
    if (i % 2 == 0) { continue }
    println(i)
}
```

### Q3：`break` 能用在 `if` 中吗？

**不能。** `break` 和 `continue` 只能用在循环体（`while`、`do-while`、`for-in`）中，不能用在 `if` 里（除非 `if` 在循环体中）。

```Cangjie
if (true) {
    break       // ❌ 编译报错：break 不在循环中
}

for (i in 0..5) {
    if (i == 3) {
        break   // ✅ 正确：break 在 for-in 循环体中
    }
}
```

### Q4：`break` 和 `return` 有什么区别？

- `break`：**退出当前循环**，继续执行循环后面的代码
- `return`：**退出整个函数**，回到函数的调用处

```Cangjie
func search() {
    for (i in 0..10) {
        if (i == 5) {
            break       // 只退出 for 循环，然后执行 println("完成")
        }
    }
    println("完成")     // break 后执行到这里 ✅
}

func search() {
    for (i in 0..10) {
        if (i == 5) {
            return      // 退出整个函数，不会执行 println("完成")
        }
    }
    println("完成")     // return 后不会执行到这里 ❌
}

遍历 [12, 18, 25, 36, 49, 55]:

break（找第一个能被 5 整除的）：
  [12] → [18] → [25] → ⛔ 找到了，不看了

continue（打印所有奇数）：
  [12] → 跳过 → [18] → 跳过 → [25] → 打印 → [36]
   → 跳过 → [49] → 打印 → [55] → 打印 → ✅ 遍历完
```

---

## 3.6.5 break 和 continue 的类型

`break` 和 `continue` 的类型都是 **`Nothing`**（不是 `Unit`）。

`Nothing` 是一个特殊的类型，表示"永远不会有值"——因为 `break` 和 `continue` 一旦执行，就会跳出或跳过循环，根本不会返回任何值。

> [!TIP]
> 你不需要主动使用 `Nothing` 类型，了解即可。它和 `Unit` 的区别：
> - `Unit`：有值，值是 `()`（"有一个空值"）
> - `Nothing`：永远不会有值（"根本不会执行到这里"）

---

## 3.6.6 小结

| 概念 | 要点 | 一句话记住 |
|------|------|-----------|
| `break` | 提前**终止**整个循环 | 找到了，不用再找了 |
| `continue` | **跳过**本轮循环，进入下一轮 | 不感兴趣，看下一个 |
| 类型 | 都是 `Nothing` | 永远不会有值 |
| 适用循环 | `while`、`do-while`、`for-in` | 所有循环都能用 |
| while 注意 | `continue` 前确保变量已更新 | 否则可能死循环 |

## 常见问题

### Q1：`break` 和 `continue` 能用在嵌套循环中吗？

**可以，但它们只作用于当前所在的循环层**，不会影响外层循环：

```Cangjie
main() {
    for (i in 0..3) {
        for (j in 0..3) {
            if (j == 2) {
                break       // 只跳出内层 for (j)，外层继续
            }
            print("(${i},${j}) ")
        }
        println()
    }
}
```

运行结果：

```Bash
(0,0) (0,1)
(1,0) (1,1)
(2,0) (2,1)
```

**逐轮解释**：
- `i=0`：内层 j 从 0 开始，j=0 输出，j=1 输出，j=2 → **break 内层**，外层换行，i=1
- `i=1`：同上
- `i=2`：同上

> [!TIP]
> 如果想在嵌套循环中**跳出外层循环**，可以借助标志变量：
>
> ```Cangjie
> main() {
>     var found = false
>     for (i in 0..3) {
>         for (j in 0..3) {
>             if (i == 1 && j == 1) {
>                 found = true
>                 break       // 跳出内层
>             }
>             print("(${i},${j}) ")
>         }
>         if (found) {
>             break           // 跳出外层
>         }
>     }
> }
> ```

### Q2：`continue` 在 `while` 和 `for-in` 中的行为一样吗？

**不完全一样。** 这是一个重要区别：

- 在 `for-in` 中，`continue` 后会自动取下一个元素——**安全**
- 在 `while` 中，`continue` 后不会自动更新变量——**需要小心**

```Cangjie
// for-in：安全，continue 后自动取下一个元素
for (i in 0..5) {
    if (i % 2 == 0) { continue }
    println(i)
}

// while：需要确保 continue 前变量已更新
var i = 0
while (i < 5) {
    i++                 // ✅ 在 continue 之前更新
    if (i % 2 == 0) { continue }
    println(i)
}
```

### Q3：`break` 能用在 `if` 中吗？

**不能。** `break` 和 `continue` 只能用在循环体（`while`、`do-while`、`for-in`）中，不能用在 `if` 里（除非 `if` 在循环体中）。

```Cangjie
if (true) {
    break       // ❌ 编译报错：break 不在循环中
}

for (i in 0..5) {
    if (i == 3) {
        break   // ✅ 正确：break 在 for-in 循环体中
    }
}
```

### Q4：`break` 和 `return` 有什么区别？

- `break`：**退出当前循环**，继续执行循环后面的代码
- `return`：**退出整个函数**，回到函数的调用处

```Cangjie
func search() {
    for (i in 0..10) {
        if (i == 5) {
            break       // 只退出 for 循环，然后执行 println("完成")
        }
    }
    println("完成")     // break 后执行到这里 ✅
}

func search() {
    for (i in 0..10) {
        if (i == 5) {
            return      // 退出整个函数，不会执行 println("完成")
        }
    }
    println("完成")     // return 后不会执行到这里 ❌
}
