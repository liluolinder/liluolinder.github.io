## 2.9.1 什么是 Unit 类型？

**Unit 类型**是一个特殊的类型，它表示"**没有实际的值**"。你可以把它理解为一个"占位符"——告诉编译器"这里有一个返回值，但这个返回值没有意义"。

> [!TIP]
> 在其他编程语言中，Unit 类型有各种不同的名字：
> - **Java / C++** → `void`（空）
> - **TypeScript** → `void`（空）
> - **仓颉** → **`Unit`**（单位）

Unit 类型**只有一个值**，就是一对空括号 `()`：

```Cangjie
let u: Unit = ()    // () 是 Unit 类型唯一的值
println(u)         // 输出：()
```

---

## 2.9.2 哪些表达式是 Unit 类型？

在实际编程中，你**不需要主动声明 Unit 类型**，它更多时候是作为"函数的返回值"或"某些表达式的类型"自动出现的。

### 1. 没有返回值的函数

当一个函数不需要返回任何有意义的值时，它的返回类型就是 `Unit`。比如 `println` 函数：

```Cangjie
main() {
    let result = println("你好")  //输出你好
    println(result)         // 输出：()
}
```

运行结果
```Bash
你好
()
```

如果你写 `main()` 而不写返回类型，它默认也是 `Unit`：

```Cangjie
main() {                // 等同于 main(): Unit
    println("hello")
                        // 不需要写 return，程序结束后自动 "返回" ()
}
```

### 2. 赋值表达式

赋值操作 `=` 本身也是一个表达式，它的值就是 `()`：

```Cangjie
main() {
    var x = 10
    let result = (x = 20)   // 赋值表达式的值是 ()
    println(result)          // 输出：()
}
```

### 3. 复合赋值表达式

`+=`、`-=`、`*=` 等复合赋值表达式也是 Unit 类型：

```Cangjie
main() {
    var x = 10
    let result = (x += 5)   // 复合赋值的值是 ()
    println(result)          // 输出：()
}
```

### 4. 循环表达式

`while` 和 `for-in` 循环整个表达式的类型也是 `Unit`：

```Cangjie
main() {
    let result = while (false) {
        println("不会执行")
    }
    println(result)          // 输出：()
}
```

---

## 2.9.3 Unit 类型能做什么？

Unit 类型虽然"没有实际的值"，但它有几个重要的作用：

### 1. 区分"有返回值"和"没有返回值"

在仓颉中，**每个表达式都有类型**，即使是"没有返回值"的操作，也得有一个类型来表示"没有值"——这个类型就是 `Unit`。

```Cangjie
let a = 10          // Int64    —— 有意义的整数
let b = println("")  // Unit     —— 没有意义，只是打印
```


### 2. 在 `main()` 函数中省略返回类型

当你看下面两个写法时，它们的效果是一样的：

```Cangjie
main() {            // 返回类型是 Unit
    println("hello")
}
```

```Cangjie
main(): Unit {      // 显式写出 Unit（但通常省略）
    println("hello")
}
```
当然如果在某些情况返回类型声明为了Unit类型，编译器或报错 或者 你只是想调皮一下也可以这么干

```Cangjie
main(): Unit {      // 显式写出 Unit（但通常省略）
    println("hello")
    return ()
}
```

---

## 2.9.4 Unit 支持的操作

Unit 类型支持的操作**非常有限**，只有三种：

| 操作 | 示例 | 说明 |
|------|------|------|
| 赋值 | `let a = ()` | 把 Unit 值赋给变量 |
| 判等 | `() == ()` | 判断两个 Unit 是否相等 |
| 判不等 | `() != ()` | 判断两个 Unit 是否不等 |

```Cangjie
main() {
    let a = ()
    let b = ()

    println(a == b)     // 输出：true（所有 () 都相等）
    println(a != b)     // 输出：false
}
```

除此之外，`+`、`-`、`*`、`/` 等运算符对 Unit 都无效。

---

## 2.9.5 小结

| 概念 | 要点 |
|------|------|
| 类型名 | `Unit` |
| 唯一值 | `()`（一对空括号） |
| 相当于其他语言 | `void`（Java/C++）、`None`（Python） |
| 哪些表达式是 Unit | `println()`、赋值、复合赋值、循环等 |
| 支持的操作 | 赋值、判等、判不等（不支持其他运算） |
| 常见用法 | 表示函数没有返回值、`main()` 省略返回类型 |

## 常见问题

### Q1：`Unit` 和 `void` 完全一样吗？

**不完全一样**。虽然它们都表示"没有值"，但有一个重要区别：

- **C++/Java 的 `void`** —— 不能声明变量（不能写 `void v;`）
- **仓颉的 `Unit`** —— 可以声明变量，因为它有一个实际的值 `()`（可以写 `let u: Unit = ()`）

> [!TIP]
> 简单记：`Unit` 就像有一个"空盒子"`()`，而 `void` 连盒子都没有。

### Q2：什么时候需要显式写 `: Unit`？

**几乎不需要**。通常只有两种情况：
1. 你**特别想强调**某个函数没有返回值（比如教学演示时）
2. 泛型编程中需要明确指定类型参数

日常写代码时，直接省略就好：

```Cangjie
// ✅ 常见写法
main() {
    println("不用写 :Unit")
}

// ✅ 也可以显式写（效果一样，但通常没必要）
main(): Unit {
    println("显式写了 :Unit")
}
```

### Q3：如果一个函数`return 0`，返回类型还能是 `Unit` 吗？

**不行！** 返回 `Unit` 的函数不能 `return` 任何有意义的数值：

```Cangjie
main(): Unit {
    println("hello")
    return 0        // ❌ 报错：期望 Unit，但得到 Int64
}
```

这也就是为什么你在[第一个仓颉程序](../1_初识仓颉/1_第一个仓颉程序.md)中看到 `main(): Int64` 时要搭配 `return 0` 的原因——两者类型必须匹配。

### Q4：`()` 既是 Unit 的字面量，又是空元组吗？

在仓颉中，`()` **就是 Unit 类型唯一的值**，不要和元组混淆。元组至少要有两个元素才能算元组，`()` 不是元组。

## 2.9.1 什么是 Unit 类型？

**Unit 类型**是一个特殊的类型，它表示"**没有实际的值**"。你可以把它理解为一个"占位符"——告诉编译器"这里有一个返回值，但这个返回值没有意义"。

> [!TIP]
> 在其他编程语言中，Unit 类型有各种不同的名字：
> - **Java / C++** → `void`（空）
> - **TypeScript** → `void`（空）
> - **仓颉** → **`Unit`**（单位）

Unit 类型**只有一个值**，就是一对空括号 `()`：

```Cangjie
let u: Unit = ()    // () 是 Unit 类型唯一的值
println(u)         // 输出：()
```

---

## 2.9.2 哪些表达式是 Unit 类型？

在实际编程中，你**不需要主动声明 Unit 类型**，它更多时候是作为"函数的返回值"或"某些表达式的类型"自动出现的。

### 1. 没有返回值的函数

当一个函数不需要返回任何有意义的值时，它的返回类型就是 `Unit`。比如 `println` 函数：

```Cangjie
main() {
    let result = println("你好")  //输出你好
    println(result)         // 输出：()
}
```

运行结果
```Bash
你好
()
```

如果你写 `main()` 而不写返回类型，它默认也是 `Unit`：

```Cangjie
main() {                // 等同于 main(): Unit
    println("hello")
                        // 不需要写 return，程序结束后自动 "返回" ()
}
```

### 2. 赋值表达式

赋值操作 `=` 本身也是一个表达式，它的值就是 `()`：

```Cangjie
main() {
    var x = 10
    let result = (x = 20)   // 赋值表达式的值是 ()
    println(result)          // 输出：()
}
```

### 3. 复合赋值表达式

`+=`、`-=`、`*=` 等复合赋值表达式也是 Unit 类型：

```Cangjie
main() {
    var x = 10
    let result = (x += 5)   // 复合赋值的值是 ()
    println(result)          // 输出：()
}
```

### 4. 循环表达式

`while` 和 `for-in` 循环整个表达式的类型也是 `Unit`：

```Cangjie
main() {
    let result = while (false) {
        println("不会执行")
    }
    println(result)          // 输出：()
}
```

---

## 2.9.3 Unit 类型能做什么？

Unit 类型虽然"没有实际的值"，但它有几个重要的作用：

### 1. 区分"有返回值"和"没有返回值"

在仓颉中，**每个表达式都有类型**，即使是"没有返回值"的操作，也得有一个类型来表示"没有值"——这个类型就是 `Unit`。

```Cangjie
let a = 10          // Int64    —— 有意义的整数
let b = println("")  // Unit     —— 没有意义，只是打印
```


### 2. 在 `main()` 函数中省略返回类型

当你看下面两个写法时，它们的效果是一样的：

```Cangjie
main() {            // 返回类型是 Unit
    println("hello")
}
```

```Cangjie
main(): Unit {      // 显式写出 Unit（但通常省略）
    println("hello")
}
```
当然如果在某些情况返回类型声明为了Unit类型，编译器或报错 或者 你只是想调皮一下也可以这么干

```Cangjie
main(): Unit {      // 显式写出 Unit（但通常省略）
    println("hello")
    return ()
}
```

---

## 2.9.4 Unit 支持的操作

Unit 类型支持的操作**非常有限**，只有三种：

| 操作 | 示例 | 说明 |
|------|------|------|
| 赋值 | `let a = ()` | 把 Unit 值赋给变量 |
| 判等 | `() == ()` | 判断两个 Unit 是否相等 |
| 判不等 | `() != ()` | 判断两个 Unit 是否不等 |

```Cangjie
main() {
    let a = ()
    let b = ()

    println(a == b)     // 输出：true（所有 () 都相等）
    println(a != b)     // 输出：false
}
```

除此之外，`+`、`-`、`*`、`/` 等运算符对 Unit 都无效。

|------|------|
| 类型名 | `Unit` |
| 唯一值 | `()`（一对空括号） |
| 相当于其他语言 | `void`（Java/C++）、`None`（Python） |
| 哪些表达式是 Unit | `println()`、赋值、复合赋值、循环等 |
| 支持的操作 | 赋值、判等、判不等（不支持其他运算） |
| 常见用法 | 表示函数没有返回值、`main()` 省略返回类型 |

> 简单记：`Unit` 就像有一个"空盒子"`()`，而 `void` 连盒子都没有。

1. 你**特别想强调**某个函数没有返回值（比如教学演示时）
2. 泛型编程中需要明确指定类型参数

日常写代码时，直接省略就好：

```Cangjie
// ✅ 常见写法

    println("不用写 :Unit")
}

// ✅ 也可以显式写（效果一样，但通常没必要）

    println("显式写了 :Unit")
}
```

### Q3：如果一个函数`return 0`，返回类型还能是 `Unit` 吗？

**不行！** 返回 `Unit` 的函数不能 `return` 任何有意义的数值：

```Cangjie
main(): Unit {
    println("hello")
    return 0        // ❌ 报错：期望 Unit，但得到 Int64
}

    println("不用写 :Unit")
}

// ✅ 也可以显式写（效果一样，但通常没必要）
main(): Unit {
    println("显式写了 :Unit")
}
```

### Q3：如果一个函数`return 0`，返回类型还能是 `Unit` 吗？

**不行！** 返回 `Unit` 的函数不能 `return` 任何有意义的数值：

```Cangjie
main(): Unit {
    println("hello")
    return 0        // ❌ 报错：期望 Unit，但得到 Int64
}
```

这也就是为什么你在[第一个仓颉程序](../1_初识仓颉/1_第一个仓颉程序.md)中看到 `main(): Int64` 时要搭配 `return 0` 的原因——两者类型必须匹配。

### Q4：`()` 既是 Unit 的字面量，又是空元组吗？

在仓颉中，`()` **就是 Unit 类型唯一的值**，不要和元组混淆。元组至少要有两个元素才能算元组，`()` 不是元组。
