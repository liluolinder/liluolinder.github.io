Nothing类型

仓颉的基本数据类型——Nothing类型

2.10.1 什么是 Nothing 类型？

Nothing 类型是一种特殊的类型，它不包含任何值，并且 Nothing 类型是所有类型的子类型（这当中也包括 Unit 类型）。

ℹ️NOTE

Nothing类型了解即可，基本用不着

2.10.2 哪些表达式是 Nothing 类型？

break、continue、return 和 throw 表达式的类型是 Nothing。程序执行到这些表达式时，它们之后的代码将不会被执行。

  • return 只能在函数体中使用
  • break、continue 只能在循环体中使用
Cangjie 
while (true) {
    func f() {
        break // Error, break 必须直接在循环体中使用
    }
    let g = { =>
        continue // Error, continue 必须直接在循环体中使用
    }
}
❗IMPORTANT

break 和 continue 必须直接在循环体中使用。在上例中，break 被放在了嵌套函数 f 的函数体内，continue 被放在了 lambda 表达式 g 的函数体内——它们都不再是"直接在循环体中"，因此编译报错。

由于函数的形参和其默认值不属于该函数的函数体，所以下面例子中的 return 表达式缺少包围它的函数体——它既不属于外层函数 f（因为内层函数定义 g 已经开始），也不在内层函数 g 的函数体中：

Cangjie 
func f() {
    func g(x!: Int64 = return) { // Error, return 必须在函数体中使用
        0
    }
    1
}
ℹ️NOTE

上例中，return 出现在了函数 g 的形参默认值中。形参及其默认值不属于函数体，因此 return 找不到包围它的函数体，编译报错。

2.10.3 使用限制

🚨CAUTION

目前编译器还不允许在使用类型的地方显式地使用 Nothing 类型。也就是说，你不能这样写：

Cangjie 
let x: Nothing = ???       // ❌ 不允许
func foo(): Nothing { ... } // ❌ 不允许

Nothing 类型更多是编译器内部使用，用于表达 break、continue、return、throw 等控制流表达式的类型。
