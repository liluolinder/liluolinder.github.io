## 2.12.1 为什么要类型转换？

之前我们学了各种数值类型：`Int8`、`Int16`、`Int32`、`Int64`、`Float32`、`Float64`……

它们表示不同范围和精度的数值。但在实际编程中，你经常会遇到**需要把一种类型转成另一种类型**的情况：

> 就像一个 2 升的水壶（`Int16`）装满了水，你想倒进一个 10 升的水桶（`Int64`）里——没问题，水壶里的水倒进去完全装得下。
>
> 但如果反过来，10 升水桶（`Int64`）想倒进 2 升水壶（`Int16`），就可能**溢出来**。

不同类型之间不能直接赋值或运算，需要显式转换。

---

## 2.12.2 基本语法：`T(e)`

仓颉使用 `T(e)` 的语法进行类型转换——把表达式 `e` 的值转换为类型 `T`：

```Cangjie
let a: Int8 = 10
let b = Int16(a)    // 把 Int8 的 a 转换成 Int16
```

> [!TIP]
> 可以这样记：`Int16(a)` = "把 `a` 变成 `Int16` 类型"。

---

## 2.12.3 整数之间的转换

### 小类型 → 大类型（安全）

```Cangjie
main() {
    let a: Int8 = 10
    let b: Int16 = 20

    let r1 = Int16(a)          // Int8 → Int16，安全
    println("r1 = ${r1}")      // 输出：r1 = 10

    let r2 = Int8(b)           // Int16 → Int8，可能溢出
    println("r2 = ${r2}")      // 输出：r2 = 20（20 在 Int8 范围内，没问题）
}
```

### 大类型 → 小类型（可能溢出）

```Cangjie
main() {
    let big: Int16 = 200
    let small = Int8(big)      // 200 超出了 Int8 的范围（-128~127）
    println(small)             // 编译可能报错或运行时抛出异常
}
```

> [!WARNING]
> 把大范围的整数转成小范围时，如果值超出了目标类型的范围，会发生**溢出**。编译器如果能提前检测到会直接报错，否则运行时抛出异常。

---

## 2.12.4 浮点数之间的转换

```Cangjie
main() {
    let c: Float32 = 1.0
    let d: Float64 = 1.123456789

    let r3 = Float64(c)         // Float32 → Float64，精度提升
    println("r3 = ${r3}")       // 输出：r3 = 1.000000

    let r4 = Float32(d)         // Float64 → Float32，可能损失精度
    println("r4 = ${r4}")       // 输出：r4 = 1.123457（精度丢失）
}
```

> [!NOTE]
> `Float64` 的 `1.123456789` 转成 `Float32` 后变成了 `1.123457`（精度损失了）。浮点数转换时要注意精度问题。

---

## 2.12.5 整数和浮点数之间的转换

### 整数 → 浮点数

```Cangjie
main() {
    let e: Int64 = 1024
    let r5 = Float64(e)         // Int64 → Float64
    println("r5 = ${r5}")       // 输出：r5 = 1024.000000
}
```

### 浮点数 → 整数

浮点数转整数时，**小数部分会被舍弃**（向 0 取整），不会四舍五入：

```Cangjie
main() {
    let f: Float64 = 1024.1024
    let r6 = Int64(f)           // Float64 → Int64
    println("r6 = ${r6}")       // 输出：r6 = 1024（小数部分被舍弃）
}
```

```Cangjie
main() {
    let pi: Float64 = 3.14159
    let intPi = Int64(pi)
    println(intPi)               // 输出：3（不是 3.14，也不是 4，直接舍弃小数）
}
```

> [!TIP]
> 如果你想**四舍五入**而不是直接截断，可以先加 0.5 再转：
> ```Cangjie
> let pi: Float64 = 3.6
> let rounded = Int64(pi + 0.5)   // 4
> ```

---

## 2.12.6 转换规则总结

| 转换方向 | 是否安全 | 注意事项 |
|---------|---------|---------|
| 小整数 → 大整数 | ✅ 安全 | 如 `Int8` → `Int64` |
| 大整数 → 小整数 | ⚠️ 可能溢出 | 值超出目标范围会报错或抛异常 |
| 低精度浮点 → 高精度浮点 | ✅ 安全 | 如 `Float32` → `Float64` |
| 高精度浮点 → 低精度浮点 | ⚠️ 精度损失 | 小数位可能被截断 |
| 整数 → 浮点数 | ✅ 安全 | 如 `Int64` → `Float64` |
| 浮点数 → 整数 | ⚠️ 小数舍弃 | **不会四舍五入**，直接截断 |

---

## 2.12.7 什么时候需要类型转换？

### 1. 不同类型之间运算

```Cangjie
let x: Int8 = 10
let y: Int64 = 20
// let z = x + y     // ❌ 类型不匹配
let z = Int64(x) + y   // ✅ 先转成相同类型再运算
```

### 2. 函数参数类型不匹配

```Cangjie
func foo(a: Float64) {
    println(a)
}

let val: Int64 = 42
// foo(val)           // ❌ 期望 Float64，得到 Int64
foo(Float64(val))     // ✅ 显式转换
```

### 3. 需要不同精度的数值

```Cangjie
let precise: Float64 = 3.1415926535
let rough = Float32(precise)   // 主动降低精度
```

---

## 2.12.8 小结

| 概念 | 要点 |
|------|------|
| 转换语法 | `T(e)`，把表达式 `e` 转换为类型 `T` |
| 适用范围 | 所有数值类型（整数 + 浮点数） |
| 小转大 | ✅ 安全 |
| 大转小 | ⚠️ 可能溢出 |
| 浮点转整数 | 舍弃小数，不四舍五入 |
| 不同类型运算 | 需要先转成相同类型 |

## 常见问题

### Q1：`Int64` 转 `Float64` 会损失精度吗？

对于很大很大的整数（超过 2⁵³ ≈ 9007199254740992），`Float64` 无法精确表示所有整数，会有精度损失。日常使用的小整数不会。

### Q2：浮点数转整数时，负数怎么处理？

**向 0 取整**（舍弃小数部分）：

```Cangjie
let a = Int64(3.99)      // 3  （正数向下取整）
let b = Int64(-3.99)     // -3 （负数向上取整，也是向 0 取整）
```

### Q3：为什么仓颉不提供隐式类型转换？

仓颉是**强类型语言**，所有类型转换都必须是**显式的**。这是为了避免隐藏的 bug。

比如在其他语言中（如 C），写 `int x = 3.14` 会自动把 `3.14` 截断为 `3`，你可能不知不觉就丢了精度。仓颉要求你写 `Int64(3.14)`，让你清楚地知道这里发生了转换。

> [!TIP]
> 显式转换虽然多写了几个字，但**代码的意图更清晰**，也更容易发现潜在的问题。

    println("r3 = ${r3}")       // 输出：r3 = 1.000000

    let r4 = Float32(d)         // Float64 → Float32，可能损失精度
    println("r4 = ${r4}")       // 输出：r4 = 1.123457（精度丢失）
}
```

> [!NOTE]
> `Float64` 的 `1.123456789` 转成 `Float32` 后变成了 `1.123457`（精度损失了）。浮点数转换时要注意精度问题。

## 2.12.1 为什么要类型转换？

之前我们学了各种数值类型：`Int8`、`Int16`、`Int32`、`Int64`、`Float32`、`Float64`……

它们表示不同范围和精度的数值。但在实际编程中，你经常会遇到**需要把一种类型转成另一种类型**的情况：

> 就像一个 2 升的水壶（`Int16`）装满了水，你想倒进一个 10 升的水桶（`Int64`）里——没问题，水壶里的水倒进去完全装得下。
>
> 但如果反过来，10 升水桶（`Int64`）想倒进 2 升水壶（`Int16`），就可能**溢出来**。

不同类型之间不能直接赋值或运算，需要显式转换。

---

## 2.12.2 基本语法：`T(e)`

仓颉使用 `T(e)` 的语法进行类型转换——把表达式 `e` 的值转换为类型 `T`：

```Cangjie
let a: Int8 = 10
let b = Int16(a)    // 把 Int8 的 a 转换成 Int16
```

> [!TIP]
> 可以这样记：`Int16(a)` = "把 `a` 变成 `Int16` 类型"。

## 2.12.3 整数之间的转换

### 小类型 → 大类型（安全）

```Cangjie

    let b: Int16 = 20

    let r1 = Int16(a)          // Int8 → Int16，安全
    println("r1 = ${r1}")      // 输出：r1 = 10

    let r2 = Int8(b)           // Int16 → Int8，可能溢出
    println("r2 = ${r2}")      // 输出：r2 = 20（20 在 Int8 范围内，没问题）
}
```

### 大类型 → 小类型（可能溢出）

```Cangjie

    let small = Int8(big)      // 200 超出了 Int8 的范围（-128~127）
    println(small)             // 编译可能报错或运行时抛出异常
}
```

> [!WARNING]
