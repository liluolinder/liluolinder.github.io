运算符

仓颉的基本运算操作符

2.11.1 什么是运算符？

运算符是用来执行特定运算的符号。比如数学中的 + 号可以把两个数相加，逻辑中的 && 可以组合多个条件。

运算符就像是计算器上的按钮——按哪个按钮就执行什么操作。

仓颉的运算符分为以下几大类：

类别 运算符 作用
算术 + - * / % ** 数学计算
赋值 = += -= *= /= 等 给变量赋值
关系 == != < > <= >= 比较大小、判断相等
逻辑 ! && || 组合布尔条件
位运算 ! << >> & ^ | 二进制位操作
区间 .. ..= 创建区间
自增自减 ++ -- 加 1 / 减 1
coalescing ?? Option 类型解构
💡TIP

这一节内容比较多，不需要一次性记住所有运算符。先了解常用的（算术、关系、逻辑、赋值），其他用到时再回来查。

2.11.2 算术运算符

基本运算

Cangjie 
let a = 2 + 3    // a = 5
let b = 3 - 1    // b = 2
let c = 3 * 4    // c = 12
let d = 7 / 3    // d = 2（整数除法向 0 取整）
let e = 4 % 3    // e = 1（取余数）
⚠️WARNING

两个操作数的类型必须相同！ 仓颉是强类型语言，不能直接把 Float64 和 Int64 做运算：

Cangjie 
let x: Float64 = 10.0
let y: Int64 = 3
// let z = x / y    // ❌ 编译报错：'/' 不能用于 Float64 和 Int64
let z = x / Float64(y)   // ✅ 先转成相同类型

你需要先用 T(e) 语法把类型统一，再进行运算。详见类型转换。

ℹ️NOTE

整数除法会向 0 取整：7 / 3 结果是 2（而不是 2.333...）。

求幂运算 `**`

Cangjie 
let p1 = 2 ** 3          // p1 = 8（2 的 3 次方）
let p2 = 2.0 ** 3.0      // p2 = 8.0
let p3 = 2 ** UInt64(4)  // p3 = 16
左操作数类型 右操作数类型 结果类型
Int64 UInt64 Int64
Float64 Int64 或 Float64 Float64

字符串拼接

+ 也可以用于拼接字符串：

Cangjie 
let s1 = "abc"
let s2 = "ABC"
let r = s1 + s2     // r = "abcABC"

一元负号

Cangjie 
let num1: Int64 = 8
let num2 = -num1        // num2 = -8
let num3 = -(-num1)     // num3 = 8

2.11.3 赋值运算符

基本赋值 `=`

Cangjie 
var a = 1
var b = 1
a = 5           // a = 5
⚠️WARNING

赋值表达式 = 的类型是 Unit（值是 ()），不是 Bool。这有一个重要的好处：

Cangjie 
if (a == 5) { ... }   // ✅ 判等
if (a = 5) { ... }    // ❌ 编译报错：赋值表达式是 Unit，不能作为条件

很多语言（如 C）中把 == 误写成 = 不会报错，导致隐蔽的 bug。仓颉直接编译报错，帮你提前发现。

多赋值

可以一次性给多个变量赋值（右侧必须是元组）：

Cangjie 
var a: Int64
var b: Int64

(a, b) = (1, 2)     // a = 1, b = 2
(a, b) = (b, a)     // 交换：a = 2, b = 1
(a, _) = (3, 4)     // a = 3，用 _ 忽略第二个值

复合赋值

Cangjie 
//以下依次运行
var a: Int64 = 10
a += 2      // a = 12
a -= 2      // a = 10
a **= 2     // a = 100
a *= 2      // a = 200
a /= 10     // a = 20
a %= 6      // a = 2
💡TIP

a += 2 等价于 a = a + 2，其他以此类推。复合赋值让代码更简洁。

2.11.4 关系运算符

关系运算符用于比较两个值，结果都是 Bool 类型（true 或 false）：

运算符 含义 示例 结果
== 相等 3 == 3 true
!= 不等 3 != 4 true
< 小于 3 < 4 true
<= 小于等于 3 <= 3 true
> 大于 3 > 4 false
>= 大于等于 3 >= 3 true
Cangjie 
main(): Int64 {
    println(3 < 4)      // true
    println(3 <= 3)     // true
    println(3 > 4)      // false
    println(3.14 == 3.15)  // false
    println(3.14 != 3.15)  // true
    return 0
}
ℹ️NOTE

元组也可以判等：(1, 3) == (1, 3) 结果为 true，但要求两个元组的类型和长度一致。

2.11.5 逻辑运算符

逻辑运算符用于组合布尔条件：

运算符 含义 用法
! 逻辑非（取反） !true = false
&& 逻辑与（且） true && false = false
|| 逻辑或（或） true || false = true
Cangjie 
var a: Bool = true && true     // a = true
var b: Bool = true && false    // b = false
var c: Bool = false || false   // c = false
var d: Bool = false || true    // d = true
var e: Bool = !true            // e = false

短路求值

&& 和 || 采用短路求值策略：

  • expr1 && expr2：如果 expr1 为 false，直接返回 false，不再计算 expr2
  • expr1 || expr2：如果 expr1 为 true，直接返回 true，不再计算 expr2
Cangjie 
func isEven(a: Int64): Bool {
    println("判断 ${a}")
    a % 2 == 0
}

main() {
    var r = isEven(3) && isEven(30)   // isEven(3) 返回 false，isEven(30) 不会执行
}

运行结果：

Bash 
判断 3
💡TIP

短路求值很有用——你可以把"代价高"或"有副作用"的表达式放在后面，当前面的条件不满足时就不会执行。

2.11.6 位运算符

位运算符把整数当作二进制序列，对每一位进行操作：

运算符 含义 示例 结果
! 按位取反 !10 -11
<< 左移 10 << 1 20
>> 右移 10 >> 1 5
& 按位与 10 & 15 10
^ 按位异或 10 ^ 15 5
| 按位或 10 | 15 15
Cangjie 
var a = !10         // -11
a = 10 << 1         // 20
a = 10 >> 1         // 5
a = 10 & 15         // 10
a = 10 ^ 15         // 5
a = 10 | 15         // 15
ℹ️NOTE

位运算的操作数只能是整数类型。移位操作的右操作数不能为负数，也不能超过操作数的位宽。

2.11.7 区间运算符

运算符 含义 示例
.. 左闭右开区间 1..5 → 1, 2, 3, 4
..= 左闭右闭区间 1..=5 → 1, 2, 3, 4, 5

详细用法请参考区间类型。

2.11.8 自增自减

仓颉支持后缀 ++ 和 --，用于对整数加 1 或减 1：

Cangjie 
var i: Int32 = 5
i++     // i = 6
i--     // i = 5
⚠️WARNING

仓颉的 ++ / -- 只能作为后缀（i++），不能作为前缀（++i）。且它们的类型是 Unit（不能用在赋值中）。

Cangjie 
var j = 0
j = i--     // ❌ 语义错误：i-- 的类型是 Unit，不能赋值

2.11.9 Coalescing 运算符 `??`

?? 用于 Option 类型的解构。如果值存在（Some(v)），返回 v；如果不存在（None），返回默认值：

Cangjie 
main(): Int64 {
    let v1 = Option<Int64>.Some(100)
    let v2 = Option<Int64>.None
    let r1 = v1 ?? 0       // r1 = 100
    let r2 = v2 ?? 0       // r2 = 0
    println(r1)
    println(r2)
    return 0
}

2.11.10 小结

类别 运算符 说明
算术 + - * / % ** 数学运算，+ 也可拼接字符串
赋值 = += -= 等 赋值类型是 Unit，避免误写
关系 == != < > <= >= 结果都是 Bool
逻辑 ! && || 支持短路求值
位运算 ! << >> & ^ | 只能用于整数类型
区间 .. ..= 创建区间实例
自增自减 ++ -- 仅后缀，类型为 Unit
Coalescing ?? Option 类型解构

常见问题

Q1：`/` 和 `%` 对负数怎么运算？

整数除法向 0 取整。取余运算 a % b 的定义是 a - b * (a / b)：

Cangjie 
let a = 7 / 3       // a = 2
let b = 7 / -3      // b = -2
let c = -7 / 3      // c = -2
let d = 4 % -3      // d = 1
let e = -4 % 3      // e = -1

Q2：`&&` 和 `||` 与 `&` 和 `|` 有什么区别？

  • && 和 || 是逻辑运算符，操作数是 Bool，结果也是 Bool，支持短路求值
  • & 和 | 是位运算符，操作数是整数，对每一位进行运算，不短路
Cangjie 
let a = true && false   // 逻辑与，false
let b = 10 & 15         // 位与，10

Q3：为什么仓颉不推荐链式赋值？

链式赋值（如 a = b = 0）在很多语言中允许，但仓颉不支持：

Cangjie 
a = b = 0   // ❌ 语法错误

因为赋值表达式的类型是 Unit，链式赋值没有意义。推荐分开写：

Cangjie 
b = 0
a = b

Q4：运算符有优先级吗？

有。和数学中的"先乘除后加减"一样，仓颉的运算符也有优先级。简单记忆：

!（一元） > **（幂） > * / %（乘除取余） > + -（加减） > 关系运算符 > && > || > 赋值运算符

不确定优先级时，可以用括号 () 明确指定：

Cangjie 
let x = (2 + 3) * 4      // 更清晰
