函数类型

仓颉中函数也是一等公民

4.3.1 什么是函数类型？

回顾：函数是一等公民

在仓颉编程语言中，函数是一等公民（first-class citizens）。这个概念听起来有点抽象，简单来说就是：函数可以和整数、字符串一样被自由传递。

整数能做什么？整数可以：

  • 赋值给变量：let x = 42
  • 作为参数传递：println(42)
  • 作为返回值返回：func getNum(): Int64 { return 42 }

函数也能做同样的事：

  • 赋值给变量：let f = add
  • 作为参数传递：printAdd(add, 3, 5)
  • 作为返回值返回：return add

正因为函数可以像普通数据一样被传递，函数本身也必须有类型——这就是函数类型。

💡TIP

"一等公民"可以这样记：在古代，只有"一等公民"（贵族）拥有被投票选举的权利。在编程中，"一等公民"意味着这个"人"（函数）拥有和其他"人"（整数、字符串）一样的权利——能存到变量里，能在函数间传来传去。

函数类型的写法

函数类型由参数类型和返回类型组成，两者之间使用 -> 连接：

Cangjie 
(参数类型列表) -> 返回类型

参数类型使用圆括号 () 括起来，可以有 0 个或多个参数；如果参数超过一个，参数类型之间使用逗号 , 分隔。

比如 (Int64, Int64) -> Int64 表示"接收两个 Int64 参数，返回一个 Int64 值的函数"。

ℹ️NOTE

函数类型的语法只关心参数类型和返回类型，不关心参数名。(Int64, Int64) -> Int64 就准确地描述了一类函数——不管你这个函数叫 add 还是 sum，只要它接收两个 Int64 返回一个 Int64，它的类型就是 (Int64, Int64) -> Int64。

4.3.2 基本示例（多看几个就懂了）

无参函数

Cangjie 
func hello(): Unit {
    println("Hello!")
}

函数 hello 的类型是 () -> Unit。

  • () 表示没有参数
  • -> Unit 表示返回类型是 Unit（什么都不返回）

一个参数的函数

Cangjie 
func display(a: Int64): Unit {
    println(a)
}

函数 display 的类型是 (Int64) -> Unit。

  • (Int64) 表示有一个 Int64 类型的参数
  • -> Unit 表示不返回有意义的值

两个参数的函数

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    a + b
}

函数 add 的类型是 (Int64, Int64) -> Int64。

  • (Int64, Int64) 表示有两个 Int64 参数
  • -> Int64 表示返回一个 Int64

返回元组的函数

Cangjie 
func returnTuple(a: Int64, b: Int64): (Int64, Int64) {
    (a, b)
}

函数 returnTuple 的类型是 (Int64, Int64) -> (Int64, Int64)。

  • 两个参数都是 Int64
  • 返回类型是一个元组 (Int64, Int64)
💡TIP

初学者经常把函数类型和函数定义搞混。注意区别：

  • 函数定义：func add(a: Int64, b: Int64): Int64 { a + b } ← 有参数名、有函数体
  • 函数类型：(Int64, Int64) -> Int64 ← 只有类型，没有参数名、没有函数体

函数类型就像"职业分类"（医生、老师、程序员），函数定义就像具体的某个人（张医生、李老师）。

4.3.3 函数类型的类型参数名

可以为函数类型的参数标记显式的类型参数名，这能让代码的意图更清晰：

Cangjie 
func showFruitPrice(name: String, price: Int64) {
    println("fruit: ${name} price: ${price} yuan")
}

main() {
    let fruitPriceHandler: (name: String, price: Int64) -> Unit
    fruitPriceHandler = showFruitPrice
    fruitPriceHandler("banana", 10)
}

上例中，变量 fruitPriceHandler 的类型是 (name: String, price: Int64) -> Unit。参数名 name 和 price 清楚地说明了每个参数的用途——第一个是水果名字（String），第二个是水果价格（Int64）。

🚨CAUTION

在一个函数类型中，要么统一写类型参数名，要么统一不写，不能交替存在：

Cangjie 
let handler: (name: String, Int64) -> Int64   // Error：不能一个参数有名，一个没名

4.3.4 函数类型作为参数类型（"传函数"）

函数类型最常见的用途之一，就是作为另一个函数的参数——即传一个函数进去。

Cangjie 
func printAdd(add: (Int64, Int64) -> Int64, a: Int64, b: Int64): Unit {
    println(add(a, b))
}

来拆解一下 printAdd 的参数：这个函数的类型是 ((Int64, Int64) -> Int64, Int64, Int64) -> Unit

参数 类型 说明
add (Int64, Int64) -> Int64 这是一个"函数参数"——传入一个加法函数
a Int64 第一个加数
b Int64 第二个加数
Cangjie 
func add(a: Int64, b: Int64): Int64 {
    a + b
}

main() {
    printAdd(add, 3, 5)   // 把 add 函数传进去 → 输出：8
}
💡TIP

把函数作为参数传递，本质上是让调用者自定义行为。就像你去餐厅点餐：

  • 普通参数：告诉厨师"我要什么食材"（3 个鸡蛋、2 个番茄）
  • 函数参数：告诉厨师"我要怎么做"（炒着吃、煮着吃、蒸着吃）

比如排序函数可以接收一个"比较函数"作为参数来决定排序规则——按价格排序、按销量排序、按日期排序，全看调用者传什么比较函数进来。

4.3.5 函数类型作为返回类型

函数类型也可以作为另一个函数的返回类型——即返回一个函数。

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    a + b
}

func returnAdd(): (Int64, Int64) -> Int64 {
    add                         // 返回 add 这个函数本身（不是调用它！）
}

main() {
    var a = returnAdd()         // a 是一个函数，类型是 (Int64, Int64) -> Int64
    println(a(1, 2))            // 调用 a，相当于调用 add(1, 2)
}

上例中，returnAdd 的类型是 () -> (Int64, Int64) -> Int64：

  • () 表示它没有参数
  • (Int64, Int64) -> Int64 表示它返回一个"接收两个 Int64 返回 Int64 的函数"
ℹ️NOTE

returnAdd 返回的是 add 这个函数本身，而不是调用 add 的结果。关键区别：

  • return add → 返回函数本身
  • return add(1, 2) → 返回调用结果（3）
💡TIP

-> 是右结合的，所以 () -> (Int64, Int64) -> Int64 等价于 () -> ((Int64, Int64) -> Int64)。
你可以这样理解：从右往左看，先看到最右边的参数和返回值构成一个函数类型，再跟左边的组成更大的类型。

4.3.6 函数类型作为变量类型

函数名本身就是一个表达式，它的类型就是对应的函数类型。因此你可以把函数赋值给变量：

🚨CAUTION

这两个写法要分清：

  • 函数名 → 表达式，类型是"函数类型"（如 (Int64, Int64) -> Int64）
  • 函数名() → 调用函数，类型是"返回值的类型"（如 Int64）
Cangjie 
func add(p1: Int64, p2: Int64): Int64 {
    p1 + p2
}

let f: (Int64, Int64) -> Int64 = add   // 把函数 add 赋值给变量 f

上例中，add 的类型是 (Int64, Int64) -> Int64，变量 f 的类型与之相同。之后 f 就可以像 add 一样被调用：

Cangjie 
main() {
    println(f(3, 5))   // 输出：8（通过变量 f 调用）
    println(add(3, 5)) // 输出：8（直接调用）
}

两者效果完全一样！f 就像是 add 的一个"别名"。

函数重载时的歧义

若一个函数在当前作用域中被重载（有多个同名版本但参数不同），直接使用函数名作为表达式可能产生歧义：

Cangjie 
func add(i: Int64, j: Int64) {
    i + j
}

func add(i: Float64, j: Float64) {
    i + j
}

main() {
    var f = add                   // Error：有两个 add，该用哪个？
    var plus: (Int64, Int64) -> Int64 = add // ✅ OK，指定了类型，告诉编译器我要 Int64 版本
}
⚠️WARNING

当函数被重载时，直接赋值会因为"不知道该用哪个版本"而报错。解决方法是为变量显式标注类型，明确告诉编译器需要哪个版本。

就像你去图书馆借书，说"我要《百年孤独》"——如果图书馆只有一本，没问题。如果有两本（一本中文版、一本英文版），你得说清"我要中文版的《百年孤独》"。

4.3.7 总结

知识点 说明 一句话记忆
函数类型语法 (参数类型列表) -> 返回类型 像函数定义但是不写参数名
一等公民 函数可作为参数、返回值、赋值给变量 函数和整数、字符串地位一样
类型参数名 可为函数类型参数标注名称 要么全写要么全不写
作为参数类型 把函数传进去，自定义行为 就像告诉厨师"怎么做"
作为返回类型 从函数里返回另一个函数 就像工厂生产机器
作为变量类型 把函数存到变量里 给函数取个别名
重载歧义 重载的函数需显式标注类型 借书说清楚要哪个版本
