闭包

捕获变量的函数与lambda

4.6.1 什么是闭包？

从嵌套函数说起

上一节我们学会了嵌套函数——函数内部定义函数。并且我们还发现，嵌套函数可以访问外部函数的变量。

那如果我们把这个嵌套函数返回出去，在外部调用它，它还能访问到那些变量吗？

答案是：能！——这就是闭包。

正式定义

一个函数或 lambda 从定义它的静态作用域中捕获了变量，函数或 lambda 和捕获的变量一起被称为一个闭包。

💡TIP

可以把闭包想象成一个"背包"：

  • 函数就像一个人
  • 它定义时周围的变量就像家里的物品
  • 当这个函数要"出远门"（被返回到外部作用域）时，它会把需要用到的变量装进背包里背着走

这样即使离开了变量原本定义的地方，函数也能正常使用它们。

最简单的闭包例子

Cangjie 
func returnAddNum(): (Int64) -> Int64 {
    let num: Int64 = 10          // num 定义在 returnAddNum 内

    func add(a: Int64) {         // add 捕获了 num
        return a + num
    }
    add                          // 返回 add（add + num 一起构成闭包）
}

main() {
    let f = returnAddNum()
    println(f(10))               // 输出：20
}

程序输出的结果为：

Bash 
20

逐行解读：

  1. 调用 returnAddNum() 进入函数
  2. 创建变量 num = 10
  3. 定义嵌套函数 add，它用到了外部的 num
  4. 返回 add——此时 add 和它捕获的 num 一起打包成了闭包
  5. returnAddNum 执行完毕——按理说 num 应该被销毁了
  6. 但 f(10) 仍然能正确输出 20——因为 num 被"装进了闭包的背包里"，没有被销毁
ℹ️NOTE

上例中，add 函数捕获了外部变量 num。即使 returnAddNum 执行完毕，num 已经"离开了它的作用域"，但闭包 add 仍然可以正常访问 num。

4.6.2 什么是变量捕获？

函数或 lambda 的定义中对以下几种变量的访问，称为变量捕获：

情形 说明 示例
1. 参数默认值中的外部局部变量 函数的参数缺省值里用了外面的变量 func f(x: Int64 = count)
2. 函数/lambda 内访问外部的局部变量 在函数里用了外面定义的变量 func g() { println(a) }
3. 非成员函数中访问实例成员 class/struct 内非成员函数访问了 this 或成员变量 后续章节会讲

以下情形的变量访问不是变量捕获：

情形 为什么不是？ 类比
本函数或 lambda 内的局部变量 就是函数自己的东西，不需要"捕获" 自己的书包，不算借
本函数或 lambda 的形参 参数本身就是函数的一部分 买手机送的充电器
全局变量和静态成员变量 全局变量一直都在，不需要"捕获" 公共资源，人人可用
实例成员函数中的实例成员变量 通过 this 参数访问的 用钥匙开自家门，不算偷

4.6.3 变量捕获的规则

变量的捕获发生在闭包定义时（而不是调用时），因此有两条重要规则：

规则一：被捕获的变量必须在闭包定义时**可见**

Cangjie 
func f() {
    let x = 99
    func f1() {
        println(x)          // ✅ x 在 f1 定义之前已声明，可见
    }
    let f2 = {
        => println(y)       // ❌ Error: y 还没声明呢！
    }
    let y = 88
    f1()                    // 输出：99
    f2()
}
🚨CAUTION

闭包只能捕获定义时已经存在的变量。f2 定义的时候 y 还没声明，就像你想看一本书，但这本书还没出版——自然看不到。

规则二：被捕获的变量必须在闭包定义前**完成初始化**

Cangjie 
func f() {
    let x: Int64              // 声明了，但还没赋值
    func f1() {
        println(x)            // ❌ Error: x 还没初始化，不能捕获
    }
    x = 99                    // 这里才赋值
    f1()
}
🚨CAUTION

即使变量已经声明（名字存在了），但如果还没完成初始化（没有值），闭包也不能捕获它。
就像一个空盒子，标签贴好了（声明了），但盒子里还没放东西（没初始化），你没法把它装进背包里。

4.6.4 捕获引用类型变量（可以修改内部成员）

如果捕获的变量是引用类型（如 class），可以修改其可变实例成员变量的值：

Cangjie 
class C {
    public var num: Int64 = 0
}

func returnIncrementer(): () -> Unit {
    let c: C = C()              // c 是引用类型（class）

    func incrementer() {
        c.num++                 // ✅ 可以修改引用类型的可变成员
    }

    incrementer
}

main() {
    let f = returnIncrementer()
    f()                         // c.num 从 0 增加到 1
}
ℹ️NOTE

这里 c 本身用 let 声明（不可变引用），但 c.num 是 var（可变成员）。

因为 c 是引用类型（class），let 只保证"不能把 c 指向另一个对象"，但不保证对象内部的成员不可变。

类比：你借了一本书给朋友（let 引用），朋友不能把书换成另一本，但可以在书上做笔记（修改内部成员 num）。

4.6.5 捕获 var 变量的限制（"危险"的闭包）

什么是"逃逸"？

"逃逸"（escape）是指闭包在定义它的函数执行完毕后，仍然可以在函数外部被调用。上面的 returnAddNum 例子就是典型的逃逸。

为什么捕获 `var` 有限制？

let 变量不可变——闭包捕获了它，只能读，不会出乱子。

var 变量可变——如果闭包捕获了 var，还能逃逸出去，那多个地方同时修改就可能出问题。所以仓颉做了安全限制。

具体限制：捕获了 var 变量的闭包只能被调用，不能作为一等公民使用。

💡TIP

"不能作为一等公民"的意思是：不能把它赋值给变量、不能作为参数传递、不能作为返回值返回、不能把它当表达式用——你唯一能做的就是直接调用它。

Cangjie 
func f() {
    var x = 1

    func g() {
        println(x)          // ✅ g 捕获了 var 变量 x
    }
    let b = g               // ❌ Error：g 捕获了 var，不能赋值给变量

    g                       // ❌ Error：不能作为表达式使用
    g()                     // ✅ 唯一的允许操作：直接调用

    return g                // ❌ Error：不能作为返回值
}
⚠️WARNING

捕获 let 变量的闭包是"安全的"，可以作为一等公民自由使用；但捕获 var 变量的闭包因为涉及可变状态，为了安全限制了它的逃逸。

4.6.6 捕获的传递性（像"传染"一样）

如果一个函数 f 调用了捕获 var 变量的函数 g，且 g 捕获的 var 变量不在函数 f 内定义，那么函数 f 也相当于捕获了 var 变量。

这就像"传染"：g 接触了 var，f 调用了 g，f 也被"传染"了。

示例 1：被传染（不能逃逸）

Cangjie 
func h() {
    var x = 1                // x 定义在 h 内

    func g() {
        x                    // g 捕获了 var 变量 x（x 不在 g 内定义）
    }

    func f() {
        g()                  // f 调用了 g，而 g 捕获了外部的 var
    }
    return f                 // ❌ Error：f 被"传染"了，不能逃逸
}

示例 2：没被传染（可以逃逸）

Cangjie 
func h() {
    func f() {
        var x = 1            // x 定义在 f 内
        func g() {
            x                // g 捕获了 var 变量 x，但 x 在 f 内定义
        }

        g()
    }
    return f                 // ✅ OK：x 是 f 自己的变量，不构成传染
}
💡TIP

简单记忆：一个函数是否"被传染"了 var 捕获，取决于它调用的函数所捕获的 var 变量是否定义在自身之外。

  • 是 → 被传染，不能逃逸
  • 否 → 没被传染，可以逃逸

4.6.7 全局变量和静态成员变量（不受捕获规则限制）

对全局变量和静态成员变量的访问，不属于变量捕获。

Cangjie 
class C {
    public static var a: Int32 = 0
    public static func foo() {
        a++                 // ✅ 静态成员变量，不是捕获
        return a
    }
}

var globalV1 = 0

func countGlobalV1() {
    globalV1++              // ✅ 全局变量，不是捕获
    C.a = 99
    let g = C.foo           // ✅ OK，可作为一等公民
}

func g() {
    let f = countGlobalV1   // ✅ OK
    f()
}
ℹ️NOTE

为什么全局变量和静态成员变量没有限制？

因为全局变量和静态成员变量存在于程序的整个生命周期，不依赖某个特定的函数作用域。它们一直都在，所以访问它们不构成"捕获"，也不受逃逸限制。

类比：你在房间（函数）里用客厅的公共电视（全局变量），不算"把客厅电视搬进房间"——电视本来就在那里，谁都能看。

4.6.8 总结

知识点 说明 关键记忆
闭包定义 函数/lambda + 捕获的变量 像背着背包的人
变量捕获 访问本函数外定义的局部变量 "借"外面的东西用
捕获条件 变量在闭包定义时必须可见且已初始化 东西要存在且能用
捕获 let 变量 ✅ 可作为一等公民自由使用 安全，随便用
捕获 var 变量 ❌ 只能调用，不能赋值/传递/返回 危险，怕乱改
捕获传递性 调用捕获了 var 的函数，自身也可能受限 像"传染"一样
全局/静态变量 不属于捕获，不受限制 公共资源，随便用
