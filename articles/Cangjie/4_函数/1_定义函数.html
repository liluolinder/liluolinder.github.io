定义函数

学会编写可复用的代码块

4.1.1 什么是函数？

你有没有遇到过这种情况——写了一段代码做加法，过一会又需要做同样的加法，只能把代码再写一遍？如果这样的操作重复很多次，代码就会变得又长又乱，而且如果某天你想修改加法的逻辑，你还得找到每一处重复的地方去改。

函数就是解决这个问题的好帮手。

把函数想象成"加工机器"

code 
原材料（参数） → [ 函数这台机器 ] → 成品（返回值）
  • 你给机器输入一些材料（参数，比如两个数字 3 和 5）
  • 机器按照预先设定好的步骤加工（函数体，比如把两个数相加）
  • 机器给你输出一个结果（返回值，比如 8）

函数最大的三个好处

好处 解释 类比
复用 写一次，用无数次 就像烤箱买了回来，可以烤无数次面包
可读 给代码块起个名字，一看就懂 add(3, 5) 比直接写 3 + 5 更清楚地表达了"我在做加法"
易维护 要改逻辑只改一处 如果想把加法改成减法，改函数内部就行，调用处不用动

生活中的类比

假设你每天中午都做同一道菜——番茄炒蛋：

  • 没有函数：每天都要写一遍"打蛋→切番茄→热油→炒→装盘"，很啰嗦
  • 有函数：先定义好一个叫 番茄炒蛋 的"函数"，以后每天只需要说一句"今天做番茄炒蛋"就行了

函数就是给一段操作取个名字，以后通过这个名字来调用它。

4.1.2 函数定义的语法

在仓颉中，函数使用关键字 func 定义，基本格式如下：

Cangjie 
func 函数名(参数列表): 返回值类型 {
    // 函数体——要执行的代码
}

逐项拆解

组成部分 说明 类比
func 告诉编译器"我要定义一个函数" 就像说"我来设计一台机器"
函数名 合法的标识符，用来调用这个函数 机器的名字，比如"加法器"
(参数列表) 函数需要的输入数据，多个参数用逗号隔开 机器的进料口
: 返回值类型 函数加工完后返回什么类型的数据（可选） 机器的出料口规格
{ } 函数体，大括号里写具体要执行的代码 机器内部的运作流程
💡TIP

新手最容易犯的错：注意参数列表和返回值类型之间用的是 :（冒号），不是 ,（逗号）。

✅ func add(a: Int64, b: Int64): Int64 { ← 参数列表后面是冒号
❌ func add(a: Int64, b: Int64), Int64 { ← 逗号是错的

第一个函数示例：先看完整代码

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    return a + b
}

我们一句一句来看：

部分 含义
func 关键字，表示"我要定义一个函数"
add 函数名，我给它起的名字，叫"add（加法）"
(a: Int64, b: Int64) 参数列表：两个参数 a 和 b，都是 Int64（整数）类型
: Int64 返回值类型：这个函数会返回一个整数
{ return a + b } 函数体：把 a 和 b 相加，然后返回结果

所以这个函数的意思是：你给我两个整数，我把它们加起来还给你。

💡TIP

定义好函数后，你可以在代码中随时调用它：

Cangjie 
let result = add(3, 5)     // result = 8
println(add(10, 20))       // 输出 30

定义一个没有参数的函数

不是所有函数都需要参数。比如一个"打招呼"的函数：

Cangjie 
func sayHello() {
    println("你好，欢迎来到仓颉的世界！")
}

这个函数不需要任何输入，它只管"做事"——在屏幕上打印一句话。调用它也很简单：

Cangjie 
main() {
    sayHello()    // 输出：你好，欢迎来到仓颉的世界！
}
ℹ️NOTE

即使没有参数，() 也不能省略。sayHello 表示"这个函数本身"（就像问"机器在哪？"），而 sayHello() 表示"调用这个函数"（就像按下了机器的启动按钮）。

4.1.3 参数列表

参数是函数接收的"输入数据"。一个函数可以有 0 个或多个参数。

Cangjie 
func sayHello() {                      // 无参数
    println("你好！")
}

func greet(name: String) {              // 1 个参数
    println("你好，${name}！")
}

func add(a: Int64, b: Int64): Int64 {   // 2 个参数
    return a + b
}

仓颉的参数分为两类：非命名参数和命名参数。下面我们详细看看它们的区别。

非命名参数

非命名参数的定义方式：参数名: 类型

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    return a + b
}

调用时按位置传参——第一个值给第一个参数，第二个值给第二个参数：

Cangjie 
add(3, 5)    // 3 给 a, 5 给 b → 结果 8
add(5, 3)    // 5 给 a, 3 给 b → 结果 8（虽然结果一样，但含义变了）
💡TIP

非命名参数就像"排队买票"——第一个人买第一张票，第二个人买第二张票，顺序很重要。

命名参数（带了"标签"）

命名参数的定义方式：参数名!: 类型 —— 注意参数名后面多了个 !

Cangjie 
func add(a!: Int64, b!: Int64): Int64 {
    return a + b
}

调用时需要指定参数名：

Cangjie 
add(a: 3, b: 5)       // ✅ 指定了参数名
add(3, 5)             // ❌ 报错：命名参数必须指定参数名

命名参数最大的好处：顺序可以随意调换

Cangjie 
add(b: 5, a: 3)       // ✅ 顺序无所谓
💡TIP

命名参数就像"快递包裹上的标签"——不管包裹怎么摆放，快递员看标签就知道它该送到哪。同理，不管参数顺序怎么变，编译器看参数名就知道值该给谁。

什么时候用命名参数？

当函数的参数很多（3 个以上）或者有些参数含义不太明显时，用命名参数能让代码更清晰：

Cangjie 
// 没有命名参数，你可能会搞混 order 和 count 谁先谁后
createOrder("可乐", 2, 5.5, false)

// 用了命名参数，一目了然
createOrder("可乐", quantity: 2, price: 5.5, isVIP: false)

命名参数还能设置默认值

通过 参数名!: 类型 = 默认值 的语法，可以给命名参数设置默认值：

Cangjie 
func add(a!: Int64 = 1, b!: Int64 = 1): Int64 {
    return a + b
}

main() {
    add()            // 两个参数都没传 → 都用默认值：a=1, b=1 → 结果 2
    add(a: 10)       // 只传了 a，b 用默认值 1 → 结果 11
    add(b: 20)       // 只传了 b，a 用默认值 1 → 结果 21
    add(a: 3, b: 7)  // 两个都传了 → 结果 10
}
❗IMPORTANT

只有命名参数可以设置默认值，非命名参数不行。

💡TIP

默认值在你"大多数情况下都用同一个值，偶尔需要改"的时候非常有用。

比如一个发通知邮件的函数，大多数情况下收件人是管理员，但偶尔也需要发给其他人：

Cangjie 
func sendEmail(title: String, content: String, recipient!: String = "admin@example.com")

非命名参数和命名参数混用

两者可以在同一个函数中同时使用，但注意顺序：非命名参数必须在命名参数之前。

Cangjie 
func example(a: Int64, b!: Int64, c!: Int64) {  // ✅ 先非命名，后命名
    // ...
}

func example(a!: Int64, b: Int64) {              // ❌ 报错：命名参数后面不能再跟非命名参数
    // ...
}
ℹ️NOTE

为什么非命名参数必须在命名参数之前？

因为非命名参数是按位置匹配的，如果命名参数在前，编译器就搞不清楚"这个位置的值该给谁"了。

参数的特性

1. 参数是不可变的

在函数内部不能给参数重新赋值：

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    a = a + b     // ❌ 报错：不能给不可变变量赋值
    return a
}
💡TIP

为什么参数不可变？这是为了防止你在函数内部不小心改掉了传入的值。

如果确实需要修改，可以先把参数赋值给一个 var 变量：

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    var _a = a     // ✅ 拷贝一份到可变变量
    _a = _a + b
    return _a
}

2. 参数的作用域

参数从函数开始到函数结束都有效：

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    var _a = a      // ✅ 可以用参数 a
    var b = b       // ❌ 报错：参数 b 已经被定义了，不能重复声明
    return a
}

4.1.4 函数返回值类型

返回值类型就是函数"加工完"返回的数据类型。在仓颉中，返回值类型是可选的。

方式一：显式指定返回值类型（推荐）

Cangjie 
func add(a: Int64, b: Int64): Int64 {   // : Int64 表示返回整数
    return a + b
}
🚨CAUTION

如果指定了返回值类型，函数体内所有 return 后面的表达式类型都必须匹配：

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    return (a, b)     // ❌ 报错：返回的是元组 (Int64, Int64)，不是 Int64
}

方式二：省略返回值类型（编译器自动推导）

Cangjie 
func add(a: Int64, b: Int64) {   // 没写返回值类型
    return a + b                 // 编译器自动推导出返回类型是 Int64
}

编译器会根据 return 后面的表达式类型，自动推导出函数的返回值类型。

⚠️WARNING

并不是所有情况都能成功推导。如果推导失败，编译器会报错，此时需要显式指定返回值类型。

特殊：返回类型为 Unit

Unit 表示"什么都不返回"（类似其他语言的 void）。

Cangjie 
func foo(): Unit {
    println("你好")
    // 不需要写 return，编译器自动处理
}
ℹ️NOTE

当指定返回类型为 Unit 时，编译器会自动在所有可能返回的地方插入 return ()。所以你不写 return 也没关系。

在理解上，如果你看到一个函数没有写 return xxx，那它的返回类型大概率是 Unit。比如 println 函数的返回类型就是 Unit——它把内容打印到屏幕上，但不会给你返回一个有用的值。

4.1.5 函数体

函数体就是大括号 {} 里的内容，定义了函数被调用时具体执行的代码。

函数体里可以写什么？

  • 变量定义：比如 var r = 0
  • 表达式：比如 a + b
  • 控制流：if、for、while 等
  • 甚至可以定义新的函数（这叫"嵌套函数"，后面的文章会讲）
Cangjie 
func add(a: Int64, b: Int64) {
    var r = 0          // 变量定义
    r = a + b          // 表达式
    return r           // 返回
}

return 语句

函数体中用 return 来终止函数的执行并返回结果。有两种形式：

形式 说明 适用场景
return expr 返回表达式 expr 的值 函数有返回值
return 相当于 return () 函数返回类型为 Unit
Cangjie 
func add(a: Int64, b: Int64): Int64 {
    return a + b        // ✅ 返回表达式结果
}

func foo(): Unit {
    println("干活")
    return              // ✅ 什么都不返回，仅结束函数
}

函数体也有类型？

是的，这是仓颉的一个重要特性。函数体本身也有"类型"——类型是函数体内最后一项的类型：

最后一项 函数体类型 示例
表达式 该表达式的类型 a + b → Int64
变量定义 Unit let x = 1 → Unit
函数声明 Unit func inner() {} → Unit
空函数体 Unit {} → Unit
Cangjie 
func add(a: Int64, b: Int64): Int64 {
    a + b                     // 最后一项是表达式，类型 Int64
}                             // 函数体类型 = Int64，匹配返回值类型 ✅

func foo(): Unit {
    let s = "Hello"          // 最后一项是变量定义，类型 Unit
    print(s)                 // ✅ 但这行才是真正的最后一项
}                            // print(s) 返回 Unit，所以匹配 ✅
💡TIP

正因为函数体最后一项的类型决定函数返回类型，我们可以写出不用 return 的函数：

Cangjie 
func add(a: Int64, b: Int64): Int64 {
    a + b      // 没有 return，没有分号，最后一项就是 a + b
}

这个特性在 if 分支结构中也能看到：

Cangjie 
let bit = if (voltage < 2.5) {
    0
} else {
    1
}

这和函数体的类型规则是同一个道理——if 表达式的最后一项就是它的"值"。借助这个特性，我们可以写出更简洁的 main 函数：

Cangjie 
main(): Int64 {
    println("Hello World")
    0               // 最后一项是 0，返回 Int64 类型的 0
}
Cangjie 
main() {
    println("Hello World")
    ()              // 最后一项是 ()，返回 Unit 类型
}

局部变量和作用域

在函数体内定义的变量，作用域从定义处开始，到函数体结束：

Cangjie 
let r = 0                     // 全局变量 r

func add(a: Int64, b: Int64) {
    var r = 0                 // 局部变量 r（遮盖了全局的 r）
    r = a + b
    return r                  // 这里用的是局部变量 r
}
💡TIP

作用域规则：局部变量可以"遮盖"外层的同名变量。在函数体内，同名的局部变量优先级更高。这就像"你的书桌上放了一本书，书包里也有一本同名的书——你看桌上的那本，不看书包里的"。

简单记：变量名查找时，先找最近的大括号内，找不到再往外找。

4.1.6 动手试一试 ✏️

新建一个仓颉项目，试着编写一个函数实现"长方形面积计算"：

Cangjie 
func area(width: Float64, height: Float64): Float64 {
    return width * height
}

main() {
    let w = 5.0
    let h = 3.0
    println("面积 = ${area(w, h)}")   // 输出：面积 = 15
}

挑战：试着修改函数，让它支持默认值（宽和高都默认为 1.0），然后分别测试以下三种调用方式：

  1. area() —— 两个参数都用默认值
  2. area(w: 5.0) —— 只传宽，高用默认值
  3. area(w: 5.0, h: 3.0) —— 两个都传
提示：需要使用命名参数来设置默认值哦！

4.1.7 总结

知识点 说明
func 关键字 用于定义函数
函数名 合法的标识符，用来调用这个函数
非命名参数 按位置传参，格式 p: T
命名参数 按名称传参，格式 p!: T，可设置默认值，顺序可调换
返回值类型 可选，编译器可自动推导；推荐在公开函数中显式写出
函数体 用 {} 包裹，最后一项决定函数体的类型
return 终止函数执行并返回值；也可以省略（利用最后一表达式）
Unit 类型 相当于"没有返回值"，return 可以省略
局部变量 函数内定义，作用域从定义处到函数结束，可遮盖外层同名变量
参数不可变 参数是 let 的，不能修改；可以赋值给 var 变量后再修改
