函数重载

同名函数的不同形态

4.7.1 什么是函数重载？

你有没有遇到过这种情况：你想写一个"加法"函数，但它既要能加整数、又要能加小数，甚至还要能加三个数？如果每种情况都要起不同的名字（比如 addInt、addFloat、addThree），用起来会很麻烦。

函数重载就是解决这个问题的——同一个函数名，可以对应多个不同的函数定义，只要它们的参数不同就行。

💡TIP

生活中的类比："打开"这个动作，可以有不同的打开方式：

  • 打开门 → 握住门把手转动
  • 打开窗户 → 向上推
  • 打开电脑 → 按电源键

动作都叫"打开"，但具体怎么做取决于你面对的是什么东西。

函数重载也是一样：函数名相同，但参数不同，它们就是不同的函数。编译器会根据你调用时传入的参数，自动选择正确的那个。

对比：没有重载 vs 有重载

Cangjie 
// 没有重载：每种情况都要取不同的名字
func addInt(a: Int64, b: Int64): Int64 { return a + b }
func addFloat(a: Float64, b: Float64): Float64 { return a + b }

// 有重载：同一个名字，参数不同就行
func add(a: Int64, b: Int64): Int64 { return a + b }
func add(a: Float64, b: Float64): Float64 { return a + b }

有重载的好处：调用时只用记一个名字 add，编译器自动匹配合适的版本。

4.7.2 重载的条件：同名不同参

两个函数构成重载的条件是：函数名相同，参数不同。

"参数不同"有两种情况：

情况 说明 例子
参数个数不同 一个 1 个参数，另一个 2 个参数 f(a) 和 f(a, b)
参数类型不同 参数个数相同，但类型不同 f(Int64) 和 f(Float64)
Cangjie 
func f(a: Int64): Unit {}          // 一个 Int64 参数

func f(a: Float64): Unit {}         // 一个 Float64 参数（类型不同）

func f(a: Int64, b: Float64): Unit {} // 两个参数（个数不同）
ℹ️NOTE

上例定义了三个同名函数 f：

  • f(Int64) — 接收一个整数
  • f(Float64) — 接收一个小数
  • f(Int64, Float64) — 接收一个整数加一个小数

当你调用 f(42) 时，编译器自动选第一个；调用 f(3.14) 时，选第二个；调用 f(42, 3.14) 时，选第三个。

你不必告诉编译器用哪个版本——它会自动帮你选。

4.7.3 不同作用域的重载

函数定义在不同的作用域（一个在外面，一个在里面），也可以构成重载——在两个函数都可见的地方，它们同时存在。

Cangjie 
func f(a: Int64): Unit {}          // 全局作用域：f(Int64)

func g() {
    func f(a: Float64): Unit {}    // g 函数内部：f(Float64)
    // 在 g 内部，两个 f 都可见
    f(42)     // 调用全局的 f(Int64)
    f(3.14)   // 调用局部的 f(Float64)
}
💡TIP

这就像"小区的快递收发室"和"你家门口"都有笔——你站在家门口时，优先用门口的笔（内层作用域），但两支笔都能用。

4.7.4 什么情况**不**构成重载？

不是所有同名的情况都叫重载。以下几种情况不能重载，如果写在同一个作用域会报错：

1. 变量不能重载

变量即使类型不同，也不能用同一个名字定义两次：

Cangjie 
main() {
    var f: (Int64) -> Unit    // 第一次声明变量 f
    var f: (Float64) -> Unit  // ❌ 报错：变量 f 已经定义了
}
ℹ️NOTE

重载是函数声明的"特权"，变量不适用。

2. 函数和变量不能同名

即使一个是变量、一个是函数，也不能同名：

Cangjie 
main() {
    var f: (Int64) -> Unit           // 变量 f
    func f(a: Float64): Unit {}      // ❌ 报错：函数和变量不能同名
}

3. 静态函数和实例函数不能重载

在 class 中，静态成员函数和实例成员函数之间不能重载：

Cangjie 
class C {
    public static func f(a: Int64): Unit {}   // 静态函数
    public func f(a: Float64): Unit {}         // ❌ 报错：实例函数，不能和静态的重载
}
🚨CAUTION

简单记忆：只有"函数-函数"可以重载，而且必须在同一类角色下（不能一个静态、一个实例）。

4.7.5 函数重载决议：编译器怎么选？

当你调用一个重载函数时，编译器会从候选函数中挑出最合适的一个。这个过程叫重载决议，主要有两条规则：

规则一：内层的先选

如果同一个函数名在外层和内层都有定义，优先选内层的。

Cangjie 
open class Base {}

class Sub <: Base {}

func outer() {
    func g(a: Sub) {
        print("1")
    }

    func inner() {
        func g(a: Base) {
            print("2")
        }

        g(Sub()) // 输出：2（选内层的 g）
    }
}
💡TIP

就像你口袋里有一支笔（内层），包里也有一支笔（外层）——你站在那儿的时候，优先摸口袋里的。

规则二：参数更匹配的先选

如果几个函数在同一层，编译器选参数最匹配的那个。

什么叫"最匹配"？简单说：你的实参类型和哪个形参类型最接近，就选哪个。

Cangjie 
open class Base {}

class Sub <: Base {}

func outer() {
    func g(a: Sub) {      // g 接收 Sub 类型
        print("1")
    }
    func g(a: Base) {     // g 接收 Base 类型
        print("2")
    }

    g(Sub()) // 输出：1（Sub 比 Base 更匹配 Sub 类型的实参）
}
💡TIP

规则二很好理解：你递了一瓶可乐，服务员会给你一个"可乐杯"（最匹配的），而不是泛泛的"杯子"。

特殊情况：父子类视为同一层

父类和子类中定义的函数，在重载决议时当成同一层处理，按"参数最匹配"来选：

Cangjie 
open class Base {
    public func g(a: Sub) {
        print("1")
    }
}

class Sub <: Base {
    public func g(a: Base) {
        print("2")
    }
}

func outer() {
    let s: Sub = Sub()
    s.g(Sub()) // 输出：1（父类的 g 参数是 Sub，更匹配）
}
ℹ️NOTE

这里的 s.g(Sub()) 调用时，父类的 g(Sub) 和子类的 g(Base) 都能匹配。但 Sub 类型的实参显然更匹配 g(Sub)，所以选父类的版本。

4.7.6 小结

知识点 要点 一句话记住
重载条件 同名函数，参数个数或类型不同 同名不同参
不同作用域 内层外层的同名函数在同一作用域中构成重载 远近都有
不构成重载 静态与实例成员函数、变量之间、变量与函数 有些不能同名
重载决议规则一 优先选内层作用域的 近的先选
重载决议规则二 同一层内选参数最匹配的 准的先选
父子类 视为同一作用域，按规则二选 不分你我，只看谁合适
