## 4.1.1 什么是函数？

你有没有遇到过这种情况——写了一段代码做加法，过一会又需要做同样的加法，只能把代码再写一遍？如果这样的操作重复很多次，代码就会变得又长又乱。

**函数**就是解决这个问题的好帮手。你可以把函数想象成一个"加工机器"：

```
原材料（参数） → [ 函数这台机器 ] → 成品（返回值）
```

- 你给机器**输入**一些材料（参数）
- 机器按照预先设定好的步骤**加工**（函数体）
- 机器给你**输出**一个结果（返回值）

函数最大的好处是：**写一次，用无数次**。而且函数还能让代码更容易阅读——给一段操作起一个有意义的名字，别人看代码时就能立马明白它的作用。

---

## 4.1.2 函数定义的语法

在仓颉中，函数使用关键字 `func` 定义，基本格式如下：

```Cangjie
func 函数名(参数列表): 返回值类型 {
    // 函数体——要执行的代码
}
```

逐项拆解：

| 组成部分 | 说明 | 类比 |
|---------|------|------|
| `func` | 告诉编译器"我要定义一个函数" | 就像说"我来设计一台机器" |
| `函数名` | 合法的标识符，用来调用这个函数 | 机器的名字，比如"加法器" |
| `(参数列表)` | 函数需要的输入数据，多个参数用逗号隔开 | 机器的进料口 |
| `: 返回值类型` | 函数加工完后返回什么类型的数据（可选） | 机器的出料口规格 |
| `{ }` | 函数体，大括号里写具体要执行的代码 | 机器内部的运作流程 |

### 第一个函数示例

来看一个最简单的函数：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b
}
```

这个函数叫 `add`（加法），它：
- 接收两个整数 `a` 和 `b`（参数）
- 返回一个整数（`: Int64`）
- 把 `a` 和 `b` 相加并返回结果（函数体）

> [!TIP]
> 定义好函数后，你可以在代码中随时调用它：
> ```Cangjie
> let result = add(3, 5)     // result = 8
> println(add(10, 20))       // 输出 30
> ```

---

## 4.1.3 参数列表

参数是函数接收的"输入数据"。一个函数可以有 **0 个**或多个参数。

```Cangjie
func sayHello() {                      // 无参数
    println("你好！")
}

func greet(name: String) {              // 1 个参数
    println("你好，${name}！")
}

func add(a: Int64, b: Int64): Int64 {   // 2 个参数
    return a + b
}
```

仓颉的参数分为两类：**非命名参数**和**命名参数**。

### 非命名参数

非命名参数的定义方式：`参数名: 类型`

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b
}
```

调用时按**位置**传参——第一个值给第一个参数，第二个值给第二个参数：

```Cangjie
add(3, 5)    // a = 3, b = 5
```

### 命名参数

命名参数的定义方式：`参数名!: 类型` —— 在参数名后面多了个 `!`

```Cangjie
func add(a!: Int64, b!: Int64): Int64 {
    return a + b
}
```

调用时需要指定参数名：

```Cangjie
add(a: 3, b: 5)       // ✅ 命名参数调用
add(3, 5)             // ❌ 报错：命名参数必须指定参数名
```

> [!NOTE]
> 命名参数的最大好处是在调用时**参数顺序可以随意调换**：
> ```Cangjie
> add(b: 5, a: 3)    // ✅ 顺序无所谓
> ```

### 命名参数还能设置默认值

通过 `参数名!: 类型 = 默认值` 的语法，可以给命名参数设置默认值：

```Cangjie
func add(a!: Int64 = 1, b!: Int64 = 1): Int64 {
    return a + b
}

add()            // 使用默认值：a=1, b=1 → 结果 2
add(a: 10)       // a=10, b=1(默认) → 结果 11
add(b: 20)       // a=1(默认), b=20 → 结果 21
add(a: 3, b: 7)  // a=3, b=7 → 结果 10
```

> [!IMPORTANT]
> 只有**命名参数**可以设置默认值，非命名参数不行。

### 非命名参数和命名参数混用

两者可以在同一个函数中同时使用，但注意顺序：**非命名参数必须在命名参数之前**。

```Cangjie
func example(a: Int64, b!: Int64, c!: Int64) {  // ✅ 先非命名，后命名
    // ...
}

func example(a!: Int64, b: Int64) {              // ❌ 报错：命名参数后面不能再跟非命名参数
    // ...
}
```

### 参数的特性

**1. 参数是不可变的**

在函数内部不能给参数重新赋值：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a = a + b     // ❌ 报错：不能给不可变变量赋值
    return a
}
```

> [!TIP]
> 参数默认就是不可变的（相当于 `let`），如果需要修改，可以先把参数赋值给一个 `var` 变量：
> ```Cangjie
> func add(a: Int64, b: Int64): Int64 {
>     var _a = a     // ✅ 拷贝一份到可变变量
>     _a = _a + b
>     return _a
> }
> ```

**2. 参数的作用域**

参数从函数开始到函数结束都有效：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    var _a = a      // ✅ 可以用参数 a
    var b = b       // ❌ 报错：参数 b 已经被定义了，不能重复声明
    return a
}
```

## 4.1.4 函数返回值类型

返回值类型就是函数"加工完"返回的数据类型。在仓颉中，返回值类型是**可选**的。

### 方式一：显式指定返回值类型

```Cangjie
func add(a: Int64, b: Int64): Int64 {   // : Int64 表示返回整数
    return a + b
}
```

> [!CAUTION]
> 如果指定了返回值类型，函数体内所有 `return` 后面的表达式类型都必须匹配：
> ```Cangjie
> func add(a: Int64, b: Int64): Int64 {
>     return (a, b)     // ❌ 报错：返回的是元组，不是 Int64
> }
> ```

### 方式二：省略返回值类型（编译器自动推导）

```Cangjie
func add(a: Int64, b: Int64) {   // 没写返回值类型
    return a + b                 // 编译器自动推导出返回类型是 Int64
}
```

编译器会根据 `return` 后面的表达式类型，自动推导出函数的返回值类型。

> [!WARNING]
> 并不是所有情况都能成功推导。如果推导失败，编译器会报错，此时需要显式指定返回值类型。

### 特殊：返回类型为 Unit

`Unit` 表示"什么都不返回"（类似其他语言的 `void`）。当指定返回类型为 `Unit` 时，编译器会在所有可能返回的地方自动插入 `return ()`：

```Cangjie
func foo(): Unit {
    println("你好")
    // 不需要写 return，编译器自动处理
}
```

## 4.1.5 函数体

函数体就是大括号 `{}` 里的内容，定义了函数被调用时具体执行的代码。

### 函数体里可以写什么？

- 变量定义
- 表达式
- 控制流（if、for、while 等）
- 甚至可以**定义新的函数**（嵌套函数）

```Cangjie
func add(a: Int64, b: Int64) {
    var r = 0          // 变量定义
    r = a + b          // 表达式
    return r           // 返回
}
```

### return 语句

函数体中用 `return` 来终止函数的执行并返回结果。有两种形式：

| 形式 | 说明 | 适用场景 |
|------|------|----------|
| `return expr` | 返回表达式 expr 的值 | 函数有返回值 |
| `return` | 相当于 `return ()` | 函数返回类型为 Unit |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b        // ✅ 返回表达式结果
}

func foo(): Unit {
    println("干活")
    return              // ✅ 什么都不返回，仅结束函数
}
```

> [!NOTE]
> `return` 表达式本身是一个特殊类型——`Nothing` 类型，并不是它后面跟着的表达式类型。
> `Nothing` 是一个特殊的类型，表示"永远不会有值"

### 函数体的类型

函数体本身也有"类型"——类型是函数体内**最后一项**的类型：

| 最后一项 | 函数体类型 | 示例 |
|---------|-----------|------|
| 表达式 | 该表达式的类型 | `a + b` → Int64 |
| 变量定义 | Unit | `let x = 1` → Unit |
| 函数声明 | Unit | `func inner() {}` → Unit |
| 空函数体 | Unit | `{}` → Unit |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b                     // 最后一项是表达式，类型 Int64
}                             // 函数体类型 = Int64，匹配返回值类型

func foo(): Unit {
    let s = "Hello"          // 最后一项是变量定义，类型 Unit
    print(s)                 // ✅ 最后一项是 println()，这是表达式
}                            // 但 println() 返回 Unit，所以匹配
```

> [!TIP]
> 这是Cangjie的一大特性

>在[if分支结构](../3_逻辑控制/1_if分支结构)中，我们介绍了
```Cangjie 
let bit = if (voltage < 2.5) {
    0
} else {
    1
}
```
>其本质也是此特性，借助此特性，我们在主函数返回Int64类型的时候也可以省略return 直接写0，返回Unit类型除了不写，也可以直接写`()`例如

```Cangjie
main():Int64{
    println('Hello World')
    0
}
```

```Cangjie
main(){
    println('Hello World')
    ()
}
```

### 局部变量和作用域

在函数体内定义的变量，作用域从定义处开始，到函数体结束：

```Cangjie
let r = 0                     // 全局变量 r

func add(a: Int64, b: Int64) {
    var r = 0                 // 局部变量 r（遮盖了全局的 r）
    r = a + b
    return r                  // 这里用的是局部变量 r
}
```

> [!TIP]
> 局部变量可以"遮盖"外层的同名变量。在函数体内，同名的局部变量优先级更高。

## 4.1.1 什么是函数？

你有没有遇到过这种情况——写了一段代码做加法，过一会又需要做同样的加法，只能把代码再写一遍？如果这样的操作重复很多次，代码就会变得又长又乱。

**函数**就是解决这个问题的好帮手。你可以把函数想象成一个"加工机器"：

```
原材料（参数） → [ 函数这台机器 ] → 成品（返回值）
```

- 你给机器**输入**一些材料（参数）
- 机器按照预先设定好的步骤**加工**（函数体）
- 机器给你**输出**一个结果（返回值）

函数最大的好处是：**写一次，用无数次**。而且函数还能让代码更容易阅读——给一段操作起一个有意义的名字，别人看代码时就能立马明白它的作用。

---

## 4.1.2 函数定义的语法

在仓颉中，函数使用关键字 `func` 定义，基本格式如下：

```Cangjie
func 函数名(参数列表): 返回值类型 {
    // 函数体——要执行的代码
}
```

逐项拆解：

| 组成部分 | 说明 | 类比 |
|---------|------|------|
| `func` | 告诉编译器"我要定义一个函数" | 就像说"我来设计一台机器" |
| `函数名` | 合法的标识符，用来调用这个函数 | 机器的名字，比如"加法器" |
| `(参数列表)` | 函数需要的输入数据，多个参数用逗号隔开 | 机器的进料口 |
| `: 返回值类型` | 函数加工完后返回什么类型的数据（可选） | 机器的出料口规格 |
| `{ }` | 函数体，大括号里写具体要执行的代码 | 机器内部的运作流程 |

### 第一个函数示例

来看一个最简单的函数：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b
}
```

这个函数叫 `add`（加法），它：
- 接收两个整数 `a` 和 `b`（参数）
- 返回一个整数（`: Int64`）
- 把 `a` 和 `b` 相加并返回结果（函数体）

> [!TIP]
> 定义好函数后，你可以在代码中随时调用它：
> ```Cangjie
> let result = add(3, 5)     // result = 8
> println(add(10, 20))       // 输出 30
> ```

## 4.1.3 参数列表

参数是函数接收的"输入数据"。一个函数可以有 **0 个**或多个参数。

```Cangjie
func sayHello() {                      // 无参数
    println("你好！")
}

func greet(name: String) {              // 1 个参数
    println("你好，${name}！")
}

func add(a: Int64, b: Int64): Int64 {   // 2 个参数
    return a + b
}
```

仓颉的参数分为两类：**非命名参数**和**命名参数**。

### 非命名参数

非命名参数的定义方式：`参数名: 类型`

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b
}
```

调用时按**位置**传参——第一个值给第一个参数，第二个值给第二个参数：

```Cangjie
add(3, 5)    // a = 3, b = 5
```

### 命名参数

命名参数的定义方式：`参数名!: 类型` —— 在参数名后面多了个 `!`

```Cangjie
func add(a!: Int64, b!: Int64): Int64 {
    return a + b
}
```

调用时需要指定参数名：

```Cangjie
add(a: 3, b: 5)       // ✅ 命名参数调用
add(3, 5)             // ❌ 报错：命名参数必须指定参数名
```

> [!NOTE]
> 命名参数的最大好处是在调用时**参数顺序可以随意调换**：
> ```Cangjie
> add(b: 5, a: 3)    // ✅ 顺序无所谓
> ```

### 命名参数还能设置默认值

通过 `参数名!: 类型 = 默认值` 的语法，可以给命名参数设置默认值：

```Cangjie
func add(a!: Int64 = 1, b!: Int64 = 1): Int64 {
    return a + b
}

add()            // 使用默认值：a=1, b=1 → 结果 2
add(a: 10)       // a=10, b=1(默认) → 结果 11
add(b: 20)       // a=1(默认), b=20 → 结果 21
add(a: 3, b: 7)  // a=3, b=7 → 结果 10
```

> [!IMPORTANT]
> 只有**命名参数**可以设置默认值，非命名参数不行。

### 非命名参数和命名参数混用

两者可以在同一个函数中同时使用，但注意顺序：**非命名参数必须在命名参数之前**。

```Cangjie
func example(a: Int64, b!: Int64, c!: Int64) {  // ✅ 先非命名，后命名
    // ...
}

func example(a!: Int64, b: Int64) {              // ❌ 报错：命名参数后面不能再跟非命名参数
    // ...
}
```

### 参数的特性

**1. 参数是不可变的**

在函数内部不能给参数重新赋值：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a = a + b     // ❌ 报错：不能给不可变变量赋值
    return a
}
```

> [!TIP]
> 参数默认就是不可变的（相当于 `let`），如果需要修改，可以先把参数赋值给一个 `var` 变量：
> ```Cangjie
> func add(a: Int64, b: Int64): Int64 {
>     var _a = a     // ✅ 拷贝一份到可变变量
>     _a = _a + b
>     return _a
> }
> ```

**2. 参数的作用域**

参数从函数开始到函数结束都有效：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    var _a = a      // ✅ 可以用参数 a
    var b = b       // ❌ 报错：参数 b 已经被定义了，不能重复声明
    return a
}
```

## 4.1.4 函数返回值类型

返回值类型就是函数"加工完"返回的数据类型。在仓颉中，返回值类型是**可选**的。

### 方式一：显式指定返回值类型

```Cangjie
func add(a: Int64, b: Int64): Int64 {   // : Int64 表示返回整数
    return a + b
}
```

> [!CAUTION]
> 如果指定了返回值类型，函数体内所有 `return` 后面的表达式类型都必须匹配：
> ```Cangjie
> func add(a: Int64, b: Int64): Int64 {
>     return (a, b)     // ❌ 报错：返回的是元组，不是 Int64
> }
> ```

### 方式二：省略返回值类型（编译器自动推导）

```Cangjie
func add(a: Int64, b: Int64) {   // 没写返回值类型
    return a + b                 // 编译器自动推导出返回类型是 Int64
}
```

编译器会根据 `return` 后面的表达式类型，自动推导出函数的返回值类型。

> [!WARNING]
> 并不是所有情况都能成功推导。如果推导失败，编译器会报错，此时需要显式指定返回值类型。

### 特殊：返回类型为 Unit

`Unit` 表示"什么都不返回"（类似其他语言的 `void`）。当指定返回类型为 `Unit` 时，编译器会在所有可能返回的地方自动插入 `return ()`：

```Cangjie
func foo(): Unit {
    println("你好")
    // 不需要写 return，编译器自动处理
}
```

## 4.1.5 函数体

函数体就是大括号 `{}` 里的内容，定义了函数被调用时具体执行的代码。

### 函数体里可以写什么？

- 变量定义
- 表达式
- 控制流（if、for、while 等）
- 甚至可以**定义新的函数**（嵌套函数）

```Cangjie
func add(a: Int64, b: Int64) {
    var r = 0          // 变量定义
    r = a + b          // 表达式
    return r           // 返回
}
```

### return 语句

函数体中用 `return` 来终止函数的执行并返回结果。有两种形式：

| 形式 | 说明 | 适用场景 |
|------|------|----------|
| `return expr` | 返回表达式 expr 的值 | 函数有返回值 |
| `return` | 相当于 `return ()` | 函数返回类型为 Unit |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b        // ✅ 返回表达式结果
}

func foo(): Unit {
    println("干活")
    return              // ✅ 什么都不返回，仅结束函数
}
```

> [!NOTE]
> `return` 表达式本身是一个特殊类型——`Nothing` 类型，并不是它后面跟着的表达式类型。
> `Nothing` 是一个特殊的类型，表示"永远不会有值"

### 函数体的类型

函数体本身也有"类型"——类型是函数体内**最后一项**的类型：

| 最后一项 | 函数体类型 | 示例 |
|---------|-----------|------|
| 表达式 | 该表达式的类型 | `a + b` → Int64 |
| 变量定义 | Unit | `let x = 1` → Unit |
| 函数声明 | Unit | `func inner() {}` → Unit |
| 空函数体 | Unit | `{}` → Unit |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b                     // 最后一项是表达式，类型 Int64
}                             // 函数体类型 = Int64，匹配返回值类型

func foo(): Unit {
    let s = "Hello"          // 最后一项是变量定义，类型 Unit
    print(s)                 // ✅ 最后一项是 println()，这是表达式
}                            // 但 println() 返回 Unit，所以匹配
```

> [!TIP]
> 这是Cangjie的一大特性

>在[if分支结构](../3_逻辑控制/1_if分支结构)中，我们介绍了
```Cangjie 
let bit = if (voltage < 2.5) {
    0
} else {
    1
}
```
>其本质也是此特性，借助此特性，我们在主函数返回Int64类型的时候也可以省略return 直接写0，返回Unit类型除了不写，也可以直接写`()`例如

```Cangjie
main():Int64{
    println('Hello World')
    0
}
```

```Cangjie
main(){
    println('Hello World')
    ()
}
```

### 局部变量和作用域

在函数体内定义的变量，作用域从定义处开始，到函数体结束：

```Cangjie
let r = 0                     // 全局变量 r

func add(a: Int64, b: Int64) {
    var r = 0                 // 局部变量 r（遮盖了全局的 r）
    r = a + b
    return r                  // 这里用的是局部变量 r
}
```

> [!TIP]
> 局部变量可以"遮盖"外层的同名变量。在函数体内，同名的局部变量优先级更高。

## 4.1.1 什么是函数？

你有没有遇到过这种情况——写了一段代码做加法，过一会又需要做同样的加法，只能把代码再写一遍？如果这样的操作重复很多次，代码就会变得又长又乱。

**函数**就是解决这个问题的好帮手。你可以把函数想象成一个"加工机器"：

```
原材料（参数） → [ 函数这台机器 ] → 成品（返回值）
```

- 你给机器**输入**一些材料（参数）
- 机器按照预先设定好的步骤**加工**（函数体）
- 机器给你**输出**一个结果（返回值）

函数最大的好处是：**写一次，用无数次**。而且函数还能让代码更容易阅读——给一段操作起一个有意义的名字，别人看代码时就能立马明白它的作用。

---

## 4.1.2 函数定义的语法

在仓颉中，函数使用关键字 `func` 定义，基本格式如下：

```Cangjie
func 函数名(参数列表): 返回值类型 {
    // 函数体——要执行的代码
}
```

逐项拆解：

| 组成部分 | 说明 | 类比 |
|---------|------|------|
| `func` | 告诉编译器"我要定义一个函数" | 就像说"我来设计一台机器" |
| `函数名` | 合法的标识符，用来调用这个函数 | 机器的名字，比如"加法器" |
| `(参数列表)` | 函数需要的输入数据，多个参数用逗号隔开 | 机器的进料口 |
| `: 返回值类型` | 函数加工完后返回什么类型的数据（可选） | 机器的出料口规格 |
| `{ }` | 函数体，大括号里写具体要执行的代码 | 机器内部的运作流程 |

### 第一个函数示例

来看一个最简单的函数：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b
}
```

这个函数叫 `add`（加法），它：
- 接收两个整数 `a` 和 `b`（参数）
- 返回一个整数（`: Int64`）
- 把 `a` 和 `b` 相加并返回结果（函数体）

> [!TIP]
> 定义好函数后，你可以在代码中随时调用它：
> ```Cangjie
> let result = add(3, 5)     // result = 8
> println(add(10, 20))       // 输出 30
> ```

---

## 4.1.3 参数列表

参数是函数接收的"输入数据"。一个函数可以有 **0 个**或多个参数。

```Cangjie
func sayHello() {                      // 无参数
    println("你好！")
}

func greet(name: String) {              // 1 个参数
    println("你好，${name}！")
}

func add(a: Int64, b: Int64): Int64 {   // 2 个参数
    return a + b
}
```

仓颉的参数分为两类：**非命名参数**和**命名参数**。

### 非命名参数

非命名参数的定义方式：`参数名: 类型`

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b
}
```

调用时按**位置**传参——第一个值给第一个参数，第二个值给第二个参数：

```Cangjie
add(3, 5)    // a = 3, b = 5
```

### 命名参数

命名参数的定义方式：`参数名!: 类型` —— 在参数名后面多了个 `!`

```Cangjie
func add(a!: Int64, b!: Int64): Int64 {
    return a + b
}
```

调用时需要指定参数名：

```Cangjie
add(a: 3, b: 5)       // ✅ 命名参数调用
add(3, 5)             // ❌ 报错：命名参数必须指定参数名
```

> [!NOTE]
> 命名参数的最大好处是在调用时**参数顺序可以随意调换**：
> ```Cangjie
> add(b: 5, a: 3)    // ✅ 顺序无所谓
> ```

### 命名参数还能设置默认值

通过 `参数名!: 类型 = 默认值` 的语法，可以给命名参数设置默认值：

```Cangjie
func add(a!: Int64 = 1, b!: Int64 = 1): Int64 {
    return a + b
}

add()            // 使用默认值：a=1, b=1 → 结果 2
add(a: 10)       // a=10, b=1(默认) → 结果 11
add(b: 20)       // a=1(默认), b=20 → 结果 21
add(a: 3, b: 7)  // a=3, b=7 → 结果 10
```

> [!IMPORTANT]
> 只有**命名参数**可以设置默认值，非命名参数不行。

### 非命名参数和命名参数混用

两者可以在同一个函数中同时使用，但注意顺序：**非命名参数必须在命名参数之前**。

```Cangjie
func example(a: Int64, b!: Int64, c!: Int64) {  // ✅ 先非命名，后命名
    // ...
}

func example(a!: Int64, b: Int64) {              // ❌ 报错：命名参数后面不能再跟非命名参数
    // ...
}
```

### 参数的特性

**1. 参数是不可变的**

在函数内部不能给参数重新赋值：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a = a + b     // ❌ 报错：不能给不可变变量赋值
    return a
}
```

> [!TIP]
> 参数默认就是不可变的（相当于 `let`），如果需要修改，可以先把参数赋值给一个 `var` 变量：
> ```Cangjie
> func add(a: Int64, b: Int64): Int64 {
>     var _a = a     // ✅ 拷贝一份到可变变量
>     _a = _a + b
>     return _a
> }
> ```

**2. 参数的作用域**

参数从函数开始到函数结束都有效：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    var _a = a      // ✅ 可以用参数 a
    var b = b       // ❌ 报错：参数 b 已经被定义了，不能重复声明
    return a
}
```

---

## 4.1.4 函数返回值类型

返回值类型就是函数"加工完"返回的数据类型。在仓颉中，返回值类型是**可选**的。

### 方式一：显式指定返回值类型

```Cangjie
func add(a: Int64, b: Int64): Int64 {   // : Int64 表示返回整数
    return a + b
}
```

> [!CAUTION]
> 如果指定了返回值类型，函数体内所有 `return` 后面的表达式类型都必须匹配：
> ```Cangjie
> func add(a: Int64, b: Int64): Int64 {
>     return (a, b)     // ❌ 报错：返回的是元组，不是 Int64
> }
> ```

### 方式二：省略返回值类型（编译器自动推导）

```Cangjie
func add(a: Int64, b: Int64) {   // 没写返回值类型
    return a + b                 // 编译器自动推导出返回类型是 Int64
}
```

编译器会根据 `return` 后面的表达式类型，自动推导出函数的返回值类型。

> [!WARNING]
> 并不是所有情况都能成功推导。如果推导失败，编译器会报错，此时需要显式指定返回值类型。

### 特殊：返回类型为 Unit

`Unit` 表示"什么都不返回"（类似其他语言的 `void`）。当指定返回类型为 `Unit` 时，编译器会在所有可能返回的地方自动插入 `return ()`：

```Cangjie
func foo(): Unit {
    println("你好")
    // 不需要写 return，编译器自动处理
}
```

---

## 4.1.5 函数体

函数体就是大括号 `{}` 里的内容，定义了函数被调用时具体执行的代码。

### 函数体里可以写什么？

- 变量定义
- 表达式
- 控制流（if、for、while 等）
- 甚至可以**定义新的函数**（嵌套函数）

```Cangjie
func add(a: Int64, b: Int64) {
    var r = 0          // 变量定义
    r = a + b          // 表达式
    return r           // 返回
}
```

### return 语句

函数体中用 `return` 来终止函数的执行并返回结果。有两种形式：

| 形式 | 说明 | 适用场景 |
|------|------|----------|
| `return expr` | 返回表达式 expr 的值 | 函数有返回值 |
| `return` | 相当于 `return ()` | 函数返回类型为 Unit |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    return a + b        // ✅ 返回表达式结果
}

func foo(): Unit {
    println("干活")
    return              // ✅ 什么都不返回，仅结束函数
}
```

> [!NOTE]
> `return` 表达式本身是一个特殊类型——`Nothing` 类型，并不是它后面跟着的表达式类型。
> `Nothing` 是一个特殊的类型，表示"永远不会有值"

### 函数体的类型

函数体本身也有"类型"——类型是函数体内**最后一项**的类型：

| 最后一项 | 函数体类型 | 示例 |
|---------|-----------|------|
| 表达式 | 该表达式的类型 | `a + b` → Int64 |
| 变量定义 | Unit | `let x = 1` → Unit |
| 函数声明 | Unit | `func inner() {}` → Unit |
| 空函数体 | Unit | `{}` → Unit |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b                     // 最后一项是表达式，类型 Int64
}                             // 函数体类型 = Int64，匹配返回值类型

func foo(): Unit {
    let s = "Hello"          // 最后一项是变量定义，类型 Unit
    print(s)                 // ✅ 最后一项是 println()，这是表达式
}                            // 但 println() 返回 Unit，所以匹配
```

> [!TIP]
> 这是Cangjie的一大特性

>在[if分支结构](../3_逻辑控制/1_if分支结构)中，我们介绍了
```Cangjie 
let bit = if (voltage < 2.5) {
    0
} else {
    1
}
```
>其本质也是此特性，借助此特性，我们在主函数返回Int64类型的时候也可以省略return 直接写0，返回Unit类型除了不写，也可以直接写`()`例如

```Cangjie
main():Int64{
    println('Hello World')
    0
}
```

```Cangjie
main(){
    println('Hello World')
    ()
}
```

### 局部变量和作用域

在函数体内定义的变量，作用域从定义处开始，到函数体结束：

```Cangjie
let r = 0                     // 全局变量 r

func add(a: Int64, b: Int64) {
    var r = 0                 // 局部变量 r（遮盖了全局的 r）
    r = a + b
    return r                  // 这里用的是局部变量 r
}
```

> [!TIP]
> 局部变量可以"遮盖"外层的同名变量。在函数体内，同名的局部变量优先级更高。
