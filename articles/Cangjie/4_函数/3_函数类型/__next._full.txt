## 4.3.1 什么是函数类型？

在仓颉编程语言中，**函数是一等公民（first-class citizens）**，这意味着函数可以：

- 作为函数的**参数**传递
- 作为函数的**返回值**返回
- **赋值给变量**

正因为函数可以像普通数据一样被传递，函数本身也必须有类型——这就是**函数类型**。

> [!TIP]
> "一等公民"的意思是：函数拥有和整数、字符串等普通数据一样的"待遇"——能存到变量里，也能在函数间传来传去。

函数类型由**参数类型**和**返回类型**组成，两者之间使用 `->` 连接：

```Cangjie
(参数类型列表) -> 返回类型
```

参数类型使用圆括号 `()` 括起来，可以有 0 个或多个参数；如果参数超过一个，参数类型之间使用逗号 `,` 分隔。

---

## 4.3.2 基本示例

看几个例子，直观感受函数类型的写法：

### 无参函数

```Cangjie
func hello(): Unit {
    println("Hello!")
}
```

函数 `hello` 的类型是 **`() -> Unit`**，表示该函数没有参数，返回类型为 `Unit`。

### 一个参数的函数

```Cangjie
func display(a: Int64): Unit {
    println(a)
}
```

函数 `display` 的类型是 **`(Int64) -> Unit`**，表示该函数有一个 `Int64` 参数，返回类型为 `Unit`。

### 两个参数的函数

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b
}
```

函数 `add` 的类型是 **`(Int64, Int64) -> Int64`**，表示该函数有两个 `Int64` 参数，返回类型为 `Int64`。

### 返回元组的函数

```Cangjie
func returnTuple(a: Int64, b: Int64): (Int64, Int64) {
    (a, b)
}
```

函数 `returnTuple` 的类型是 **`(Int64, Int64) -> (Int64, Int64)`**，两个参数类型均为 `Int64`，返回类型为元组 `(Int64, Int64)`。

> [!NOTE]
> 函数类型的语法**只关心参数类型和返回类型**，不关心参数名。`(Int64, Int64) -> Int64` 就准确地描述了一类函数。

---

## 4.3.3 函数类型的类型参数名

可以为函数类型的参数标记显式的**类型参数名**，这能让代码的意图更清晰：

```Cangjie
func showFruitPrice(name: String, price: Int64) {
    println("fruit: ${name} price: ${price} yuan")
}

main() {
    let fruitPriceHandler: (name: String, price: Int64) -> Unit
    fruitPriceHandler = showFruitPrice
    fruitPriceHandler("banana", 10)
}
```

上例中，变量 `fruitPriceHandler` 的类型是 `(name: String, price: Int64) -> Unit`，参数名 `name` 和 `price` 清楚地说明了每个参数的用途。

> [!CAUTION]
> 在一个函数类型中，要么**统一写**类型参数名，要么**统一不写**，不能交替存在：
>
> ```Cangjie
> let handler: (name: String, Int64) -> Int64   // Error
> ```

---

## 4.3.4 函数类型作为参数类型

函数类型最常见的用途之一，就是作为另一个函数的参数：

```Cangjie
func printAdd(add: (Int64, Int64) -> Int64, a: Int64, b: Int64): Unit {
    println(add(a, b))
}
```

函数 `printAdd` 的类型是 **`((Int64, Int64) -> Int64, Int64, Int64) -> Unit`**，它有三个参数：

| 参数 | 类型 | 说明 |
|------|------|------|
| `add` | `(Int64, Int64) -> Int64` | 一个加法函数 |
| `a` | `Int64` | 第一个加数 |
| `b` | `Int64` | 第二个加数 |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b
}

main() {
    printAdd(add, 3, 5)   // 输出：8
}
```

> [!TIP]
> 把函数作为参数传递，让调用者可以**自定义行为**。比如排序函数可以接收一个"比较函数"作为参数来决定排序规则。

---

## 4.3.5 函数类型作为返回类型

函数类型也可以作为另一个函数的**返回类型**：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b
}

func returnAdd(): (Int64, Int64) -> Int64 {
    add
}

main() {
    var a = returnAdd()
    println(a(1, 2))      // 输出：3
}
```

上例中，`returnAdd` 的类型是 **`() -> (Int64, Int64) -> Int64`**，表示它无参数，返回一个 `(Int64, Int64) -> Int64` 类型的函数。

> [!NOTE]
> `->` 是**右结合**的，所以 `() -> (Int64, Int64) -> Int64` 等价于 `() -> ((Int64, Int64) -> Int64)`。

---

## 4.3.6 函数类型作为变量类型

函数名本身也是表达式，它的类型就是对应的函数类型。因此你可以把函数赋值给变量：

> [!CAUTION]
> 直接使用`函数名`是表达式，类型为函数类型
> 使用函数名`()`则是调用函数，类型为返回值的类型

```Cangjie
func add(p1: Int64, p2: Int64): Int64 {
    p1 + p2
}

let f: (Int64, Int64) -> Int64 = add
```

上例中，函数名 `add` 的类型是 `(Int64, Int64) -> Int64`，变量 `f` 的类型与之相同，`add` 被用来初始化 `f`。之后 `f` 就可以像 `add` 一样被调用：

```Cangjie
main() {
    println(f(3, 5))   // 输出：8
}
```

### 函数重载时的歧义

若一个函数在当前作用域中被**重载**（有多个同名版本），直接使用函数名作为表达式可能产生歧义：

```Cangjie
func add(i: Int64, j: Int64) {
    i + j
}

func add(i: Float64, j: Float64) {
    i + j
}

main() {
    var f = add                   // Error, ambiguous function 'add'
    var plus: (Int64, Int64) -> Int64 = add // OK，指定了类型，消除歧义
}
```

> [!WARNING]
> 当函数被重载时，直接赋值会因为"不知道该用哪个版本"而报错。解决方法是为变量**显式标注类型**，明确告诉编译器需要哪个版本。

---

| 知识点 | 说明 |
|--------|------|
| 函数类型语法 | `(参数类型列表) -> 返回类型` |
| 一等公民 | 函数可作为参数、返回值、赋值给变量 |
| 类型参数名 | 可为函数类型参数标注名称（需统一标注或不标注） |
| 作为参数类型 | 传递函数以自定义行为 |
| 作为返回类型 | 返回函数实现工厂模式等 |
| 作为变量类型 | 将函数赋值给变量后通过变量调用 |
| 重载歧义 | 重载函数需显式标注类型以消除歧义 |

## 4.3.1 什么是函数类型？

在仓颉编程语言中，**函数是一等公民（first-class citizens）**，这意味着函数可以：

- 作为函数的**参数**传递
- 作为函数的**返回值**返回
- **赋值给变量**

正因为函数可以像普通数据一样被传递，函数本身也必须有类型——这就是**函数类型**。

> [!TIP]
> "一等公民"的意思是：函数拥有和整数、字符串等普通数据一样的"待遇"——能存到变量里，也能在函数间传来传去。

函数类型由**参数类型**和**返回类型**组成，两者之间使用 `->` 连接：

```Cangjie
(参数类型列表) -> 返回类型
```

参数类型使用圆括号 `()` 括起来，可以有 0 个或多个参数；如果参数超过一个，参数类型之间使用逗号 `,` 分隔。

---

## 4.3.2 基本示例

看几个例子，直观感受函数类型的写法：

### 无参函数

```Cangjie
func hello(): Unit {
    println("Hello!")
}
```

函数 `hello` 的类型是 **`() -> Unit`**，表示该函数没有参数，返回类型为 `Unit`。

### 一个参数的函数

```Cangjie
func display(a: Int64): Unit {
    println(a)
}
```

函数 `display` 的类型是 **`(Int64) -> Unit`**，表示该函数有一个 `Int64` 参数，返回类型为 `Unit`。

### 两个参数的函数

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b
}
```

函数 `add` 的类型是 **`(Int64, Int64) -> Int64`**，表示该函数有两个 `Int64` 参数，返回类型为 `Int64`。

### 返回元组的函数

```Cangjie
func returnTuple(a: Int64, b: Int64): (Int64, Int64) {
    (a, b)
}
```

函数 `returnTuple` 的类型是 **`(Int64, Int64) -> (Int64, Int64)`**，两个参数类型均为 `Int64`，返回类型为元组 `(Int64, Int64)`。

> [!NOTE]
> 函数类型的语法**只关心参数类型和返回类型**，不关心参数名。`(Int64, Int64) -> Int64` 就准确地描述了一类函数。

---

## 4.3.3 函数类型的类型参数名

可以为函数类型的参数标记显式的**类型参数名**，这能让代码的意图更清晰：

```Cangjie
func showFruitPrice(name: String, price: Int64) {
    println("fruit: ${name} price: ${price} yuan")
}

main() {
    let fruitPriceHandler: (name: String, price: Int64) -> Unit
    fruitPriceHandler = showFruitPrice
    fruitPriceHandler("banana", 10)
}
```

上例中，变量 `fruitPriceHandler` 的类型是 `(name: String, price: Int64) -> Unit`，参数名 `name` 和 `price` 清楚地说明了每个参数的用途。

> [!CAUTION]
> 在一个函数类型中，要么**统一写**类型参数名，要么**统一不写**，不能交替存在：
>
> ```Cangjie
> let handler: (name: String, Int64) -> Int64   // Error
> ```

---

## 4.3.4 函数类型作为参数类型

函数类型最常见的用途之一，就是作为另一个函数的参数：

```Cangjie
func printAdd(add: (Int64, Int64) -> Int64, a: Int64, b: Int64): Unit {
    println(add(a, b))
}
```

函数 `printAdd` 的类型是 **`((Int64, Int64) -> Int64, Int64, Int64) -> Unit`**，它有三个参数：

| 参数 | 类型 | 说明 |
|------|------|------|
| `add` | `(Int64, Int64) -> Int64` | 一个加法函数 |
| `a` | `Int64` | 第一个加数 |
| `b` | `Int64` | 第二个加数 |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b
}

main() {
    printAdd(add, 3, 5)   // 输出：8
}
```

> [!TIP]
> 把函数作为参数传递，让调用者可以**自定义行为**。比如排序函数可以接收一个"比较函数"作为参数来决定排序规则。

---

## 4.3.5 函数类型作为返回类型

函数类型也可以作为另一个函数的**返回类型**：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b
}

func returnAdd(): (Int64, Int64) -> Int64 {
    add
}

main() {
    var a = returnAdd()
    println(a(1, 2))      // 输出：3
}
```

上例中，`returnAdd` 的类型是 **`() -> (Int64, Int64) -> Int64`**，表示它无参数，返回一个 `(Int64, Int64) -> Int64` 类型的函数。

> [!NOTE]
> `->` 是**右结合**的，所以 `() -> (Int64, Int64) -> Int64` 等价于 `() -> ((Int64, Int64) -> Int64)`。

---

## 4.3.6 函数类型作为变量类型

函数名本身也是表达式，它的类型就是对应的函数类型。因此你可以把函数赋值给变量：

> [!CAUTION]
> 直接使用`函数名`是表达式，类型为函数类型
> 使用函数名`()`则是调用函数，类型为返回值的类型

```Cangjie
func add(p1: Int64, p2: Int64): Int64 {
    p1 + p2
}

let f: (Int64, Int64) -> Int64 = add
```

上例中，函数名 `add` 的类型是 `(Int64, Int64) -> Int64`，变量 `f` 的类型与之相同，`add` 被用来初始化 `f`。之后 `f` 就可以像 `add` 一样被调用：

```Cangjie
main() {
    println(f(3, 5))   // 输出：8
}
```

### 函数重载时的歧义

若一个函数在当前作用域中被**重载**（有多个同名版本），直接使用函数名作为表达式可能产生歧义：

```Cangjie
func add(i: Int64, j: Int64) {
    i + j
}

func add(i: Float64, j: Float64) {
    i + j
}

main() {
    var f = add                   // Error, ambiguous function 'add'
    var plus: (Int64, Int64) -> Int64 = add // OK，指定了类型，消除歧义
}
```

> [!WARNING]
> 当函数被重载时，直接赋值会因为"不知道该用哪个版本"而报错。解决方法是为变量**显式标注类型**，明确告诉编译器需要哪个版本。

---

## 4.3.4 函数类型作为参数类型

函数类型最常见的用途之一，就是作为另一个函数的参数：

```Cangjie
func printAdd(add: (Int64, Int64) -> Int64, a: Int64, b: Int64): Unit {
    println(add(a, b))
}
```

函数 `printAdd` 的类型是 **`((Int64, Int64) -> Int64, Int64, Int64) -> Unit`**，它有三个参数：

| 参数 | 类型 | 说明 |
|------|------|------|
| `add` | `(Int64, Int64) -> Int64` | 一个加法函数 |
| `a` | `Int64` | 第一个加数 |
| `b` | `Int64` | 第二个加数 |

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b
}

main() {
    printAdd(add, 3, 5)   // 输出：8
}
```

> [!TIP]
> 把函数作为参数传递，让调用者可以**自定义行为**。比如排序函数可以接收一个"比较函数"作为参数来决定排序规则。

---

## 4.3.5 函数类型作为返回类型

函数类型也可以作为另一个函数的**返回类型**：

```Cangjie
func add(a: Int64, b: Int64): Int64 {
    a + b
}

func returnAdd(): (Int64, Int64) -> Int64 {
    add
}

main() {
    var a = returnAdd()
    println(a(1, 2))      // 输出：3
}
```

上例中，`returnAdd` 的类型是 **`() -> (Int64, Int64) -> Int64`**，表示它无参数，返回一个 `(Int64, Int64) -> Int64` 类型的函数。

> [!NOTE]
> `->` 是**右结合**的，所以 `() -> (Int64, Int64) -> Int64` 等价于 `() -> ((Int64, Int64) -> Int64)`。

---

## 4.3.6 函数类型作为变量类型

函数名本身也是表达式，它的类型就是对应的函数类型。因此你可以把函数赋值给变量：

> [!CAUTION]
> 直接使用`函数名`是表达式，类型为函数类型
> 使用函数名`()`则是调用函数，类型为返回值的类型

```Cangjie
func add(p1: Int64, p2: Int64): Int64 {
    p1 + p2
}

let f: (Int64, Int64) -> Int64 = add
```

上例中，函数名 `add` 的类型是 `(Int64, Int64) -> Int64`，变量 `f` 的类型与之相同，`add` 被用来初始化 `f`。之后 `f` 就可以像 `add` 一样被调用：

```Cangjie
main() {
    println(f(3, 5))   // 输出：8
}
```

### 函数重载时的歧义

若一个函数在当前作用域中被**重载**（有多个同名版本），直接使用函数名作为表达式可能产生歧义：

```Cangjie
func add(i: Int64, j: Int64) {
    i + j
}

func add(i: Float64, j: Float64) {
    i + j
}

main() {
    var f = add                   // Error, ambiguous function 'add'
    var plus: (Int64, Int64) -> Int64 = add // OK，指定了类型，消除歧义
}
```

> [!WARNING]
> 当函数被重载时，直接赋值会因为"不知道该用哪个版本"而报错。解决方法是为变量**显式标注类型**，明确告诉编译器需要哪个版本。
