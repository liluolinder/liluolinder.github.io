## 4.4.1 什么是 Lambda 表达式？

**Lambda 表达式**是一种**匿名函数**——即没有函数名的函数。它的核心设计目的是在程序中快速定义简短的函数逻辑，无需显式声明函数名称。

这一概念起源于数学中的 λ 演算（lambda calculus），后被引入多种编程语言（如 C++、Python、C# 等），用于简化代码并提升灵活性。仓颉编程语言中也引入了 Lambda 表达式。


---

## 4.4.2 Lambda 表达式的基本语法

Lambda 表达式的语法如下：

```Cangjie
{ p1: T1, ..., pn: Tn => expressions | declarations }
```

- `=>` **之前**为参数列表，多个参数之间使用 `,` 分隔，每个参数名和参数类型之间使用 `:` 分隔
- `=>` **之后**为 Lambda 表达式体，是一组表达式或声明序列

```Cangjie
let f1 = { a: Int64, b: Int64 => a + b }
```

这段代码定义了一个 Lambda 表达式：接收两个 `Int64` 参数 `a` 和 `b`，返回它们的和，然后赋值给变量 `f1`。

Lambda 表达式也可以没有参数：

```Cangjie
var display = {
    =>                  // 无参 lambda
        println("Hello")
        println("World")
}
```

> [!IMPORTANT]
> Lambda 表达式不管有没有参数，**都不可以省略 `=>`**，除非它作为尾随 lambda（[见下文](#446-尾随-lambda)）。

```Cangjie
var display = { => println("Hello") }     // ✅ 有 =>
```

---

## 4.4.3 参数类型省略与推断

Lambda 表达式中参数的类型标注可以**缺省**。编译器会尝试从上下文中推断参数类型，以下情形中推断可以成功：

### 1. 赋值时根据变量类型推断

```Cangjie
var sum1: (Int64, Int64) -> Int64 = { a, b => a + b }    // a 和 b 的类型被推断为 Int64  
var sum2: (Int64, Int64) -> Int64 = { a: Int64, b => a + b }  // 可以部分标注
```

### 2. 作为实参时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    // 参数 a2 的类型由函数 f 的形参推断为 Int64
    f({ a2 => a2 + 10 })
}
```

> [!WARNING]
> 当编译器无法推断出参数类型时会编译报错。此时需要显式标注参数类型。

---

## 4.4.4 返回类型推断

Lambda 表达式中**不支持显式声明返回类型**，其返回类型总是从上下文中推断出来。

### 1. 赋值时根据变量类型推断

```Cangjie
let f: () -> Unit = { => println(10) }
```

### 2. 作为参数时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    f({ a2: Int64 => a2 + 10 })
}
```

### 3. 作为返回值时根据函数返回类型推断

```Cangjie
func f(): (Int64) -> Int64 {
    { a: Int64 => a }
}
```

### 4. 从函数体自身推断

若上下文中类型未明确，编译器会根据 Lambda 表达式体中所有 `return xxx` 中 `xxx` 的类型，以及 Lambda 表达式体的类型，来共同推导出返回类型。

```Cangjie
let sum1 = { a: Int64, b: Int64 => a + b }    // 返回类型为 Int64  仓颉的特性 不写return
let f = { => }                                 // 右侧为空，返回类型为 Unit
```

> [!NOTE]
> `=>` 右侧的内容与普通函数体的规则一致。`=>` 右侧为空时，返回类型为 `Unit`。

---

## 4.4.5 调用 Lambda 表达式

### 立即调用

Lambda 表达式支持定义后**立即调用**：

```Cangjie
let r1 = { a: Int64, b: Int64 => a + b }(1, 2)   // r1 = 3
let r2 = { => 123 }()                              // r2 = 123
```

### 赋值后调用

Lambda 表达式也可以赋值给变量，通过变量名调用：

```Cangjie
func f() {
    var g = { x: Int64 => println("x = ${x}") }
    g(2)      // 输出：x = 2
}
```

---

## 4.4.6 尾随 Lambda

**尾随 lambda** 是一种特殊的语法，可以使函数的调用看起来像是语言内置的语法一样，增加代码的可读性。

当函数**最后一个形参是函数类型**，并且函数调用对应的实参是 lambda 时，可以将 lambda **放在函数调用的尾部，圆括号外面**：

```Cangjie
func myIf(a: Bool, fn: () -> Int64) {
    if (a) {
        fn()
    } else {
        0
    }
}

func test() {
    myIf(true, { => 100 })   // ✅ 普通函数调用

    myIf(true) {             // ✅ 尾随 lambda 调用
        100
    }
}
```

如果函数调用**有且只有一个 lambda 实参**，还可以省略 `()`，只写 lambda：

```Cangjie
func f(fn: (Int64) -> Int64) {
    fn(1)
}

func test() {
    f { i => i * i }   // ✅ 省略 ()，只写 lambda
}
```

> [!TIP]
> 尾随 lambda 让代码更"自然"——`myIf(true) { 100 }` 读起来就像 `myIf` 是一个内置的关键字结构一样。这种语法在 Kotlin、Swift 等语言中也非常常见。
> 在仓颉开发鸿蒙APP时，会大量用到

## 4.4.1 什么是 Lambda 表达式？

**Lambda 表达式**是一种**匿名函数**——即没有函数名的函数。它的核心设计目的是在程序中快速定义简短的函数逻辑，无需显式声明函数名称。

这一概念起源于数学中的 λ 演算（lambda calculus），后被引入多种编程语言（如 C++、Python、C# 等），用于简化代码并提升灵活性。仓颉编程语言中也引入了 Lambda 表达式。


---

## 4.4.2 Lambda 表达式的基本语法

Lambda 表达式的语法如下：

```Cangjie
{ p1: T1, ..., pn: Tn => expressions | declarations }
```

- `=>` **之前**为参数列表，多个参数之间使用 `,` 分隔，每个参数名和参数类型之间使用 `:` 分隔
- `=>` **之后**为 Lambda 表达式体，是一组表达式或声明序列

```Cangjie
let f1 = { a: Int64, b: Int64 => a + b }
```

这段代码定义了一个 Lambda 表达式：接收两个 `Int64` 参数 `a` 和 `b`，返回它们的和，然后赋值给变量 `f1`。

Lambda 表达式也可以没有参数：

```Cangjie
var display = {
    =>                  // 无参 lambda
        println("Hello")
        println("World")
}
```

> [!IMPORTANT]
> Lambda 表达式不管有没有参数，**都不可以省略 `=>`**，除非它作为尾随 lambda（[见下文](#446-尾随-lambda)）。

```Cangjie
var display = { => println("Hello") }     // ✅ 有 =>
```

---

## 4.4.3 参数类型省略与推断

Lambda 表达式中参数的类型标注可以**缺省**。编译器会尝试从上下文中推断参数类型，以下情形中推断可以成功：

### 1. 赋值时根据变量类型推断

```Cangjie
var sum1: (Int64, Int64) -> Int64 = { a, b => a + b }    // a 和 b 的类型被推断为 Int64  
var sum2: (Int64, Int64) -> Int64 = { a: Int64, b => a + b }  // 可以部分标注
```

### 2. 作为实参时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    // 参数 a2 的类型由函数 f 的形参推断为 Int64
    f({ a2 => a2 + 10 })
}
```

> [!WARNING]
> 当编译器无法推断出参数类型时会编译报错。此时需要显式标注参数类型。

---

## 4.4.4 返回类型推断

Lambda 表达式中**不支持显式声明返回类型**，其返回类型总是从上下文中推断出来。

### 1. 赋值时根据变量类型推断

```Cangjie
let f: () -> Unit = { => println(10) }
```

### 2. 作为参数时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    f({ a2: Int64 => a2 + 10 })
}
```

### 3. 作为返回值时根据函数返回类型推断

```Cangjie
func f(): (Int64) -> Int64 {
    { a: Int64 => a }
}
```

### 4. 从函数体自身推断

## 4.4.1 什么是 Lambda 表达式？

**Lambda 表达式**是一种**匿名函数**——即没有函数名的函数。它的核心设计目的是在程序中快速定义简短的函数逻辑，无需显式声明函数名称。

这一概念起源于数学中的 λ 演算（lambda calculus），后被引入多种编程语言（如 C++、Python、C# 等），用于简化代码并提升灵活性。仓颉编程语言中也引入了 Lambda 表达式。


---

## 4.4.2 Lambda 表达式的基本语法

Lambda 表达式的语法如下：

```Cangjie
{ p1: T1, ..., pn: Tn => expressions | declarations }
```

- `=>` **之前**为参数列表，多个参数之间使用 `,` 分隔，每个参数名和参数类型之间使用 `:` 分隔
- `=>` **之后**为 Lambda 表达式体，是一组表达式或声明序列

```Cangjie
let f1 = { a: Int64, b: Int64 => a + b }
```

这段代码定义了一个 Lambda 表达式：接收两个 `Int64` 参数 `a` 和 `b`，返回它们的和，然后赋值给变量 `f1`。

Lambda 表达式也可以没有参数：

```Cangjie
var display = {
    =>                  // 无参 lambda
        println("Hello")
        println("World")
}
```

> [!IMPORTANT]
> Lambda 表达式不管有没有参数，**都不可以省略 `=>`**，除非它作为尾随 lambda（[见下文](#446-尾随-lambda)）。

```Cangjie
var display = { => println("Hello") }     // ✅ 有 =>
```

---

## 4.4.3 参数类型省略与推断

Lambda 表达式中参数的类型标注可以**缺省**。编译器会尝试从上下文中推断参数类型，以下情形中推断可以成功：

### 1. 赋值时根据变量类型推断

```Cangjie
var sum1: (Int64, Int64) -> Int64 = { a, b => a + b }    // a 和 b 的类型被推断为 Int64  
var sum2: (Int64, Int64) -> Int64 = { a: Int64, b => a + b }  // 可以部分标注
```

### 2. 作为实参时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    // 参数 a2 的类型由函数 f 的形参推断为 Int64
    f({ a2 => a2 + 10 })
}
```

> [!WARNING]
> 当编译器无法推断出参数类型时会编译报错。此时需要显式标注参数类型。

### 1. 赋值时根据变量类型推断
