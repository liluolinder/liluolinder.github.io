## 4.4.1 什么是 Lambda 表达式？

**Lambda 表达式**是一种**匿名函数**——即没有函数名的函数。

### 生活中的类比

想象你去一个餐厅面试做厨师：
- **普通函数**：餐厅正式录用你，给你一个工牌（函数名），你每天上班（调用函数）
- **Lambda 表达式**：餐厅忙不过来，临时叫了一个路人来帮忙炒一道菜。炒完菜，路人就走了，连名字都不需要记

Lambda 就是那个"用完即走、不需要名字的临时厨师"。

它的核心设计目的是在程序中**快速定义简短的函数逻辑**，省去"先起名字、再写定义、再使用"的繁琐步骤。

> [!TIP]
> Lambda 表达式的概念起源于数学中的 λ 演算（lambda calculus），后被引入多种编程语言（如 C++、Python、C# 等）。仓颉编程语言中也引入了 Lambda 表达式，在开发鸿蒙应用时会大量用到。

### 普通函数 vs Lambda

```Cangjie
// 普通函数：需要起个名字
func add(a: Int64, b: Int64): Int64 {
    return a + b
}

// Lambda 表达式：不需要名字，直接写逻辑
let addLambda = { a: Int64, b: Int64 => a + b }
```

两者做的事情完全一样。区别是：
- 普通函数有名字（`add`），可以多次复用
- Lambda 没有名字，更**简洁**，适合"临时用一下"的场景

---

## 4.4.2 Lambda 表达式的基本语法

Lambda 表达式的语法如下：

```Cangjie
{ p1: T1, ..., pn: Tn => expressions | declarations }
```

逐段拆解：

| 部分 | 说明 |
|------|------|
| `{ }` | 一对大括号，把 Lambda 包起来 |
| `p1: T1, ..., pn: Tn` | `=>` 之前是参数列表，参数名和类型之间用 `:` 分隔 |
| `=>` | 分隔符（Lambda 的"标志"），**不可省略** |
| `expressions` | `=>` 之后是函数体，可以写表达式或声明 |

```Cangjie
let f1 = { a: Int64, b: Int64 => a + b }
```

这段代码定义了一个 Lambda：接收两个 `Int64` 参数 `a` 和 `b`，返回它们的和，然后赋值给变量 `f1`。

### 无参数的 Lambda

```Cangjie
var display = {
    =>                  // 无参 lambda，=> 前面没有参数
        println("Hello")
        println("World")
}
```

> [!IMPORTANT]
> Lambda 表达式不管有没有参数，**都不可以省略 `=>`**，除非它作为尾随 lambda。

```Cangjie
var display = { => println("Hello") }     // ✅ 有 =>
var display = { println("Hello") }        // ❌ 错误：缺少 =>
```

## 4.4.3 参数类型省略与推断

Lambda 表达式中参数的类型标注可以**缺省**——编译器会自动推断参数类型。

> [!TIP]
> 这就像"你去食堂打饭，不告诉阿姨你要几两饭。阿姨看你一眼，就知道你大概吃多少"——编译器通过上下文猜出参数的类型。

### 1. 赋值时根据变量类型推断

```Cangjie
// sum1 的类型是 (Int64, Int64) -> Int64
// 编译器推断 a 和 b 都是 Int64
var sum1: (Int64, Int64) -> Int64 = { a, b => a + b }

// 也可以只标注部分参数
var sum2: (Int64, Int64) -> Int64 = { a: Int64, b => a + b }
```

### 2. 作为实参时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    // 参数 a2 的类型由函数 f 的形参推断为 Int64
    f({ a2 => a2 + 10 })
}
```

> [!WARNING]
> 如果编译器无法推断出参数类型，就会报错。这时候你需要显式标注参数类型，给编译器"一点提示"。

---

## 4.4.4 返回类型推断

Lambda 表达式中**不支持显式声明返回类型**，返回类型总是从上下文中推断出来。

### 1. 赋值时根据变量类型推断

```Cangjie
let f: () -> Unit = { => println(10) }
```

### 2. 作为参数时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    f({ a2: Int64 => a2 + 10 })
}
```

### 3. 作为返回值时根据函数返回类型推断

```Cangjie
func f(): (Int64) -> Int64 {
    { a: Int64 => a }
}
```

### 4. 从函数体自身推断

若上下文中类型未明确，编译器会根据 Lambda 表达式体中所有 `return xxx` 的类型，以及 Lambda 表达式体的类型来推导：

```Cangjie
// 最后一项 a + b 是 Int64 → 返回类型 Int64
let sum1 = { a: Int64, b: Int64 => a + b }

// 右侧为空 → 返回类型 Unit
let f = { => }
```

> [!NOTE]
> `=>` 右侧的内容与普通函数体的规则一致：最后一项决定类型。`=>` 右侧为空时，返回类型为 `Unit`。

---

## 4.4.5 调用 Lambda 表达式

### 方式一：立即调用（定义完就执行）

Lambda 表达式支持定义后**立即调用**——不需要赋值给变量，直接在代码中写 Lambda，后面跟上参数：

```Cangjie
let r1 = { a: Int64, b: Int64 => a + b }(1, 2)   // r1 = 3
let r2 = { => 123 }()                              // r2 = 123
```

> [!TIP]
> 这种"定义完就调用"的方式在 JavaScript 中被称为 **IIFE**（立即执行函数表达式）。就像"你买了一包零食，拆开包装（定义），马上吃掉（调用）"。

### 方式二：赋值后调用（最常用）

Lambda 表达式赋值给变量后，通过变量名调用：

```Cangjie
func f() {
    var g = { x: Int64 => println("x = ${x}") }
    g(2)      // 输出：x = 2
}
```

## 4.4.6 尾随 Lambda

**尾随 lambda** 是一种特殊的语法，让函数调用写起来更像是**语言内置的关键字结构**，增加代码的可读性。

### 规则

当函数**最后一个形参是函数类型**，并且函数调用对应的实参是 lambda 时，可以将 lambda **放在函数调用的尾部，圆括号外面**。

### 例子

```Cangjie
func myIf(a: Bool, fn: () -> Int64) {
    if (a) {
        fn()
    } else {
        0
    }
}

func test() {
    // ✅ 普通函数调用
    myIf(true, { => 100 })

    // ✅ 尾随 lambda 调用（lambda 放到了括号外面）
    myIf(true) {
        100
    }
}
```

如果函数调用**有且只有一个 lambda 实参**，还可以省略 `()`，只写 lambda：

```Cangjie
func f(fn: (Int64) -> Int64) {
    fn(1)
}

func test() {
    f { i => i * i }   // ✅ 省略 ()，只写 lambda
}
```

> [!TIP]
> 尾随 lambda 让代码读起来更自然。`myIf(true) { 100 }` 读起来就像 `myIf` 是仓颉语言内置的关键字（类似 `if`、`while`），而不是一个普通的函数调用。
>
> 这种语法在 Kotlin、Swift 等语言中非常常见。在仓颉中开发鸿蒙应用时，你会大量用到尾随 lambda。

### 什么时候用尾随 lambda？

- 当 lambda 比较长时，写在括号外面更清晰
- 当 lambda 是函数的最后一个参数时（大部分情况下）
- 想把多个函数调用写成"嵌套结构"时

## 4.4.1 什么是 Lambda 表达式？

**Lambda 表达式**是一种**匿名函数**——即没有函数名的函数。

### 生活中的类比

想象你去一个餐厅面试做厨师：
- **普通函数**：餐厅正式录用你，给你一个工牌（函数名），你每天上班（调用函数）
- **Lambda 表达式**：餐厅忙不过来，临时叫了一个路人来帮忙炒一道菜。炒完菜，路人就走了，连名字都不需要记

Lambda 就是那个"用完即走、不需要名字的临时厨师"。

它的核心设计目的是在程序中**快速定义简短的函数逻辑**，省去"先起名字、再写定义、再使用"的繁琐步骤。

> [!TIP]
> Lambda 表达式的概念起源于数学中的 λ 演算（lambda calculus），后被引入多种编程语言（如 C++、Python、C# 等）。仓颉编程语言中也引入了 Lambda 表达式，在开发鸿蒙应用时会大量用到。

### 普通函数 vs Lambda

```Cangjie
// 普通函数：需要起个名字
func add(a: Int64, b: Int64): Int64 {
    return a + b
}

// Lambda 表达式：不需要名字，直接写逻辑
let addLambda = { a: Int64, b: Int64 => a + b }
```

两者做的事情完全一样。区别是：
- 普通函数有名字（`add`），可以多次复用
- Lambda 没有名字，更**简洁**，适合"临时用一下"的场景

---

## 4.4.2 Lambda 表达式的基本语法

Lambda 表达式的语法如下：

```Cangjie
{ p1: T1, ..., pn: Tn => expressions | declarations }
```

逐段拆解：

| 部分 | 说明 |
|------|------|
| `{ }` | 一对大括号，把 Lambda 包起来 |
| `p1: T1, ..., pn: Tn` | `=>` 之前是参数列表，参数名和类型之间用 `:` 分隔 |
| `=>` | 分隔符（Lambda 的"标志"），**不可省略** |
| `expressions` | `=>` 之后是函数体，可以写表达式或声明 |

```Cangjie
let f1 = { a: Int64, b: Int64 => a + b }
```

这段代码定义了一个 Lambda：接收两个 `Int64` 参数 `a` 和 `b`，返回它们的和，然后赋值给变量 `f1`。

### 无参数的 Lambda

```Cangjie
var display = {
    =>                  // 无参 lambda，=> 前面没有参数
        println("Hello")
        println("World")
}
```

> [!IMPORTANT]
> Lambda 表达式不管有没有参数，**都不可以省略 `=>`**，除非它作为尾随 lambda。

```Cangjie
var display = { => println("Hello") }     // ✅ 有 =>
var display = { println("Hello") }        // ❌ 错误：缺少 =>
```

## 4.4.3 参数类型省略与推断

Lambda 表达式中参数的类型标注可以**缺省**——编译器会自动推断参数类型。

> [!TIP]
> 这就像"你去食堂打饭，不告诉阿姨你要几两饭。阿姨看你一眼，就知道你大概吃多少"——编译器通过上下文猜出参数的类型。

### 1. 赋值时根据变量类型推断

```Cangjie
// sum1 的类型是 (Int64, Int64) -> Int64
// 编译器推断 a 和 b 都是 Int64
var sum1: (Int64, Int64) -> Int64 = { a, b => a + b }

// 也可以只标注部分参数
var sum2: (Int64, Int64) -> Int64 = { a: Int64, b => a + b }
```

### 2. 作为实参时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    // 参数 a2 的类型由函数 f 的形参推断为 Int64
    f({ a2 => a2 + 10 })
}
```

> [!WARNING]
> 如果编译器无法推断出参数类型，就会报错。这时候你需要显式标注参数类型，给编译器"一点提示"。

---

## 4.4.4 返回类型推断

Lambda 表达式中**不支持显式声明返回类型**，返回类型总是从上下文中推断出来。

### 1. 赋值时根据变量类型推断

```Cangjie
let f: () -> Unit = { => println(10) }
```

### 2. 作为参数时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    f({ a2: Int64 => a2 + 10 })
}
```

### 3. 作为返回值时根据函数返回类型推断

```Cangjie
func f(): (Int64) -> Int64 {
    { a: Int64 => a }
}
```

### 4. 从函数体自身推断

若上下文中类型未明确，编译器会根据 Lambda 表达式体中所有 `return xxx` 的类型，以及 Lambda 表达式体的类型来推导：

```Cangjie
// 最后一项 a + b 是 Int64 → 返回类型 Int64
let sum1 = { a: Int64, b: Int64 => a + b }

// 右侧为空 → 返回类型 Unit
let f = { => }
```

> [!NOTE]
> `=>` 右侧的内容与普通函数体的规则一致：最后一项决定类型。`=>` 右侧为空时，返回类型为 `Unit`。

## 4.4.5 调用 Lambda 表达式

### 方式一：立即调用（定义完就执行）

Lambda 表达式支持定义后**立即调用**——不需要赋值给变量，直接在代码中写 Lambda，后面跟上参数：

```Cangjie
let r1 = { a: Int64, b: Int64 => a + b }(1, 2)   // r1 = 3
let r2 = { => 123 }()                              // r2 = 123
```

> [!TIP]
> 这种"定义完就调用"的方式在 JavaScript 中被称为 **IIFE**（立即执行函数表达式）。就像"你买了一包零食，拆开包装（定义），马上吃掉（调用）"。

### 方式二：赋值后调用（最常用）

Lambda 表达式赋值给变量后，通过变量名调用：

```Cangjie
func f() {
    var g = { x: Int64 => println("x = ${x}") }
    g(2)      // 输出：x = 2
}
```

## 4.4.6 尾随 Lambda

**尾随 lambda** 是一种特殊的语法，让函数调用写起来更像是**语言内置的关键字结构**，增加代码的可读性。

### 规则

当函数**最后一个形参是函数类型**，并且函数调用对应的实参是 lambda 时，可以将 lambda **放在函数调用的尾部，圆括号外面**。

### 例子

```Cangjie
func myIf(a: Bool, fn: () -> Int64) {
    if (a) {
        fn()
    } else {
        0
    }
}

func test() {
    // ✅ 普通函数调用
    myIf(true, { => 100 })

    // ✅ 尾随 lambda 调用（lambda 放到了括号外面）
    myIf(true) {
        100
    }
}
```

如果函数调用**有且只有一个 lambda 实参**，还可以省略 `()`，只写 lambda：

```Cangjie
func f(fn: (Int64) -> Int64) {
    fn(1)
}

func test() {
    f { i => i * i }   // ✅ 省略 ()，只写 lambda
}
```

> [!TIP]
> 尾随 lambda 让代码读起来更自然。`myIf(true) { 100 }` 读起来就像 `myIf` 是仓颉语言内置的关键字（类似 `if`、`while`），而不是一个普通的函数调用。
>
> 这种语法在 Kotlin、Swift 等语言中非常常见。在仓颉中开发鸿蒙应用时，你会大量用到尾随 lambda。

### 什么时候用尾随 lambda？

- 当 lambda 比较长时，写在括号外面更清晰
- 当 lambda 是函数的最后一个参数时（大部分情况下）
- 想把多个函数调用写成"嵌套结构"时

- **Lambda 表达式**：餐厅忙不过来，临时叫了一个路人来帮忙炒一道菜。炒完菜，路人就走了，连名字都不需要记

Lambda 就是那个"用完即走、不需要名字的临时厨师"。

它的核心设计目的是在程序中**快速定义简短的函数逻辑**，省去"先起名字、再写定义、再使用"的繁琐步骤。

> [!TIP]
> Lambda 表达式的概念起源于数学中的 λ 演算（lambda calculus），后被引入多种编程语言（如 C++、Python、C# 等）。仓颉编程语言中也引入了 Lambda 表达式，在开发鸿蒙应用时会大量用到。

### 普通函数 vs Lambda

```Cangjie
// 普通函数：需要起个名字
func add(a: Int64, b: Int64): Int64 {
    return a + b
}

// Lambda 表达式：不需要名字，直接写逻辑
let addLambda = { a: Int64, b: Int64 => a + b }
```

两者做的事情完全一样。区别是：
- 普通函数有名字（`add`），可以多次复用
- Lambda 没有名字，更**简洁**，适合"临时用一下"的场景

---

## 4.4.2 Lambda 表达式的基本语法

Lambda 表达式的语法如下：

```Cangjie
{ p1: T1, ..., pn: Tn => expressions | declarations }
```

逐段拆解：

| 部分 | 说明 |
|------|------|
| `{ }` | 一对大括号，把 Lambda 包起来 |
| `p1: T1, ..., pn: Tn` | `=>` 之前是参数列表，参数名和类型之间用 `:` 分隔 |
| `=>` | 分隔符（Lambda 的"标志"），**不可省略** |
| `expressions` | `=>` 之后是函数体，可以写表达式或声明 |

```Cangjie
let f1 = { a: Int64, b: Int64 => a + b }
```

这段代码定义了一个 Lambda：接收两个 `Int64` 参数 `a` 和 `b`，返回它们的和，然后赋值给变量 `f1`。

### 无参数的 Lambda

```Cangjie
var display = {
    =>                  // 无参 lambda，=> 前面没有参数
        println("Hello")
        println("World")
}
```

> [!IMPORTANT]
> Lambda 表达式不管有没有参数，**都不可以省略 `=>`**，除非它作为尾随 lambda。

```Cangjie
var display = { => println("Hello") }     // ✅ 有 =>
var display = { println("Hello") }        // ❌ 错误：缺少 =>
```

---

## 4.4.3 参数类型省略与推断

Lambda 表达式中参数的类型标注可以**缺省**——编译器会自动推断参数类型。

> [!TIP]
> 这就像"你去食堂打饭，不告诉阿姨你要几两饭。阿姨看你一眼，就知道你大概吃多少"——编译器通过上下文猜出参数的类型。

### 1. 赋值时根据变量类型推断

```Cangjie
// sum1 的类型是 (Int64, Int64) -> Int64
// 编译器推断 a 和 b 都是 Int64
var sum1: (Int64, Int64) -> Int64 = { a, b => a + b }

// 也可以只标注部分参数
var sum2: (Int64, Int64) -> Int64 = { a: Int64, b => a + b }
```

### 2. 作为实参时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    // 参数 a2 的类型由函数 f 的形参推断为 Int64
    f({ a2 => a2 + 10 })
}
```

> [!WARNING]
> 如果编译器无法推断出参数类型，就会报错。这时候你需要显式标注参数类型，给编译器"一点提示"。

---

## 4.4.4 返回类型推断

Lambda 表达式中**不支持显式声明返回类型**，返回类型总是从上下文中推断出来。

### 1. 赋值时根据变量类型推断

```Cangjie
let f: () -> Unit = { => println(10) }
```

### 2. 作为参数时根据形参类型推断

```Cangjie
func f(a1: (Int64) -> Int64): Int64 {
    a1(1)
}

main(): Int64 {
    f({ a2: Int64 => a2 + 10 })
}
```

### 3. 作为返回值时根据函数返回类型推断

```Cangjie
func f(): (Int64) -> Int64 {
    { a: Int64 => a }
}
```

### 4. 从函数体自身推断

若上下文中类型未明确，编译器会根据 Lambda 表达式体中所有 `return xxx` 的类型，以及 Lambda 表达式体的类型来推导：

```Cangjie
// 最后一项 a + b 是 Int64 → 返回类型 Int64
let sum1 = { a: Int64, b: Int64 => a + b }

// 右侧为空 → 返回类型 Unit
let f = { => }
```

> [!NOTE]
> `=>` 右侧的内容与普通函数体的规则一致：最后一项决定类型。`=>` 右侧为空时，返回类型为 `Unit`。

---

## 4.4.5 调用 Lambda 表达式

### 方式一：立即调用（定义完就执行）

Lambda 表达式支持定义后**立即调用**——不需要赋值给变量，直接在代码中写 Lambda，后面跟上参数：

```Cangjie
let r1 = { a: Int64, b: Int64 => a + b }(1, 2)   // r1 = 3
let r2 = { => 123 }()                              // r2 = 123
```

> [!TIP]
> 这种"定义完就调用"的方式在 JavaScript 中被称为 **IIFE**（立即执行函数表达式）。就像"你买了一包零食，拆开包装（定义），马上吃掉（调用）"。

### 方式二：赋值后调用（最常用）

Lambda 表达式赋值给变量后，通过变量名调用：

```Cangjie
func f() {
    var g = { x: Int64 => println("x = ${x}") }
    g(2)      // 输出：x = 2
}
```

## 4.4.6 尾随 Lambda

**尾随 lambda** 是一种特殊的语法，让函数调用写起来更像是**语言内置的关键字结构**，增加代码的可读性。

### 规则

当函数**最后一个形参是函数类型**，并且函数调用对应的实参是 lambda 时，可以将 lambda **放在函数调用的尾部，圆括号外面**。

### 例子

```Cangjie
func myIf(a: Bool, fn: () -> Int64) {
    if (a) {
        fn()
    } else {
        0
    }
}

func test() {
    // ✅ 普通函数调用
    myIf(true, { => 100 })

    // ✅ 尾随 lambda 调用（lambda 放到了括号外面）
    myIf(true) {
        100
    }
}
```

如果函数调用**有且只有一个 lambda 实参**，还可以省略 `()`，只写 lambda：

```Cangjie
func f(fn: (Int64) -> Int64) {
    fn(1)
}

func test() {
    f { i => i * i }   // ✅ 省略 ()，只写 lambda
}
```

> [!TIP]
> 尾随 lambda 让代码读起来更自然。`myIf(true) { 100 }` 读起来就像 `myIf` 是仓颉语言内置的关键字（类似 `if`、`while`），而不是一个普通的函数调用。
>
> 这种语法在 Kotlin、Swift 等语言中非常常见。在仓颉中开发鸿蒙应用时，你会大量用到尾随 lambda。

### 什么时候用尾随 lambda？

- 当 lambda 比较长时，写在括号外面更清晰
- 当 lambda 是函数的最后一个参数时（大部分情况下）
- 想把多个函数调用写成"嵌套结构"时
