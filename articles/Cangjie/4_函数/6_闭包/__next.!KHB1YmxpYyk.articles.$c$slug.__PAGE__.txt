## 4.6.1 什么是闭包？

一个函数或 lambda **从定义它的静态作用域中捕获了变量**，函数或 lambda 和捕获的变量一起被称为一个**闭包**。

> [!TIP]
> 可以把闭包想象成一个"背包"——函数出门时，把需要用到的外部变量装进背包里背着走。这样即使离开了变量定义的地方，函数也能正常使用它们。

```Cangjie
func returnAddNum(): (Int64) -> Int64 {
    let num: Int64 = 10          // num 定义在 returnAddNum 内

    func add(a: Int64) {         // add 捕获了 num
        return a + num
    }
    add                          // add 和 num 一起构成闭包
}

main() {
    let f = returnAddNum()
    println(f(10))               // 输出：20
}
```

程序输出的结果为：

```Bash
20
```

> [!NOTE]
> 上例中，`add` 函数捕获了外部变量 `num`。即使 `returnAddNum` 执行完毕，`num` 已经"离开了它的作用域"，但闭包 `add` 仍然可以正常访问 `num`——因为 `num` 被装进了闭包的"背包"里。

---

## 4.6.2 什么是变量捕获？

函数或 lambda 的定义中对以下几种变量的访问，称为**变量捕获**：

1. 函数的参数缺省值中访问了本函数之外定义的**局部变量**
2. 函数或 lambda 内访问了本函数或本 lambda 之外定义的**局部变量**
3. `class` / `struct` 内定义的不是成员函数的函数或 lambda 访问了**实例成员变量**或 `this`

以下情形的变量访问**不是**变量捕获：

| 情形 | 说明 |
|------|------|
| 本函数或本 lambda 内的局部变量 | 就是函数自己的变量 |
| 本函数或本 lambda 的形参 | 参数本身就是函数的一部分 |
| 全局变量和静态成员变量 | 全局/静态变量不属于"作用域捕获" |
| 实例成员函数或属性中的实例成员变量 | 通过 `this` 参数访问，不是捕获 |

---

## 4.6.3 变量捕获的规则

变量的捕获发生在闭包**定义时**，因此有以下规则：

### 规则一：被捕获的变量必须在闭包定义时可见

```Cangjie
func f() {
    let x = 99
    func f1() {
        println(x)          // ✅ x 在 f1 定义之前已声明
    }
    let f2 = {
        => println(y)       // undeclared identifier 'y'
    }
    let y = 88
    f1()                    // 输出：99
    f2()
}
```

> [!CAUTION]
> 闭包只能捕获**定义时已经存在**的变量。上例中 `f2` 定义时 `y` 还没声明，因此编译报错。

### 规则二：被捕获的变量必须在闭包定义前完成初始化

```Cangjie
func f() {
    let x: Int64
    func f1() {
        println(x)          // cannot capture variable 'x' before initialization
    }
    x = 99
    f1()
}
```

> [!CAUTION]
> 即使变量已经声明（存在），但如果**尚未完成初始化**，闭包也不能捕获它。

---

## 4.6.4 捕获引用类型变量

如果捕获的变量是**引用类型**（如 `class`），可以修改其**可变实例成员变量**的值：

```Cangjie
class C {
    public var num: Int64 = 0
}

func returnIncrementer(): () -> Unit {
    let c: C = C()              // c 是引用类型

    func incrementer() {
        c.num++                 // ✅ 可以修改引用类型的可变成员
    }

    incrementer
}

main() {
    let f = returnIncrementer()
    f()                         // c.num 增加 1
}
```

> [!NOTE]
> 这里 `c` 本身是 `let`（不可变），但 `c.num` 是 `var`（可变成员）。因为 `c` 是引用类型，`let` 只保证引用关系不变，不保证引用对象内部不可变。

---

## 4.6.5 捕获 var 变量的限制

为了防止捕获了 `var` 声明变量的闭包**逃逸**（即在函数执行完毕后仍然可以在函数外部被调用），这类闭包的使用有严格限制：

> 捕获了 `var` 变量的闭包**只能被调用**，不能作为一等公民使用。

具体来说，不能：

- ❌ 赋值给变量
- ❌ 作为实参传递
- ❌ 作为返回值返回
- ❌ 直接将闭包的名字作为表达式使用

```Cangjie
func f() {
    var x = 1
    let y = 2

    func g() {
        println(x)          // ✅ 捕获了 var 变量 x
    }
    let b = g               // Error, g 不能赋值给变量

    g                       // Error, g 不能作为表达式使用
    g()                     // ✅ g 可以被调用

    g                       // Error, g 不能作为返回值
}
```

> [!WARNING]
> 捕获 `let` 变量的闭包是"安全的"，可以作为一等公民自由使用；但捕获 `var` 变量的闭包因为涉及可变状态，为了安全限制其逃逸。

---

## 4.6.6 捕获的传递性

需要注意的是，捕获具有**传递性**。如果一个函数 `f` 调用了捕获 `var` 变量的函数 `g`，且 `g` 捕获的 `var` 变量**不在**函数 `f` 内定义，那么函数 `f` 同样捕获了 `var` 变量。

**示例 6.1：传递捕获**

`g` 捕获了 `var` 声明的变量 `x`，`f` 调用了 `g`，且 `g` 捕获的 `x` 不在 `f` 内定义，`f` 同样不能作为一等公民使用：

```Cangjie
func h() {
    var x = 1

    func g() {
        x
    }                       // g 捕获了 var 变量 x

    func f() {
        g()                 // f 调用了 g
    }
    return f                // Error：传递捕获了 var 变量 x
}
```

**示例 6.2：非传递捕获**

`g` 捕获了 `var` 声明的变量 `x`，`f` 调用了 `g`。但 `g` 捕获的 `x` **在 `f` 内定义**，`f` 没有捕获其他 `var` 声明的变量，因此 `f` 仍可作为一等公民使用：

```Cangjie
func h() {
    func f() {
        var x = 1
        func g() {
            x
        }                   // g 捕获了 var 变量 x（在 f 内定义）

        g()
    }
    return f                // ✅ OK：x 在 f 内定义，不构成传递捕获
}
```

> [!TIP]
> 简单记忆：一个函数是否"被传染"了 var 捕获，取决于它调用的函数所捕获的 var 变量是否**定义在自身之外**。

---

## 4.6.7 全局变量和静态成员变量

对**全局变量**和**静态成员变量**的访问，**不属于变量捕获**。因此访问了 `var` 修饰的全局变量、静态成员变量的函数或 lambda 仍可作为一等公民使用：
> [!NOTE]
> 此处留作印象，后续学完class再回来学习

```Cangjie
class C {
    public static var a: Int32 = 0
    public static func foo() {
        a++                 // ✅ 静态成员变量，不是捕获
        return a
    }
}

var globalV1 = 0

func countGlobalV1() {
    globalV1++              // ✅ 全局变量，不是捕获
    C.a = 99
    let g = C.foo           // ✅ OK，可作为一等公民
}

func g() {
    let f = countGlobalV1   // ✅ OK
    f()
}
```

> [!NOTE]
> 全局变量和静态成员变量存在于程序的整个生命周期，不依赖某个特定的作用域，因此访问它们不构成"捕获"，也不受逃逸限制。

---

## 4.6.1 什么是闭包？

一个函数或 lambda **从定义它的静态作用域中捕获了变量**，函数或 lambda 和捕获的变量一起被称为一个**闭包**。

> [!TIP]
> 可以把闭包想象成一个"背包"——函数出门时，把需要用到的外部变量装进背包里背着走。这样即使离开了变量定义的地方，函数也能正常使用它们。

```Cangjie
func returnAddNum(): (Int64) -> Int64 {
    let num: Int64 = 10          // num 定义在 returnAddNum 内

    func add(a: Int64) {         // add 捕获了 num
        return a + num
    }
    add                          // add 和 num 一起构成闭包
}

main() {
    let f = returnAddNum()
    println(f(10))               // 输出：20
}
```

程序输出的结果为：

```Bash
20
```

> [!NOTE]
> 上例中，`add` 函数捕获了外部变量 `num`。即使 `returnAddNum` 执行完毕，`num` 已经"离开了它的作用域"，但闭包 `add` 仍然可以正常访问 `num`——因为 `num` 被装进了闭包的"背包"里。

---

## 4.6.2 什么是变量捕获？

函数或 lambda 的定义中对以下几种变量的访问，称为**变量捕获**：

1. 函数的参数缺省值中访问了本函数之外定义的**局部变量**
2. 函数或 lambda 内访问了本函数或本 lambda 之外定义的**局部变量**
3. `class` / `struct` 内定义的不是成员函数的函数或 lambda 访问了**实例成员变量**或 `this`

以下情形的变量访问**不是**变量捕获：

| 情形 | 说明 |
|------|------|
| 本函数或本 lambda 内的局部变量 | 就是函数自己的变量 |
| 本函数或本 lambda 的形参 | 参数本身就是函数的一部分 |
| 全局变量和静态成员变量 | 全局/静态变量不属于"作用域捕获" |
| 实例成员函数或属性中的实例成员变量 | 通过 `this` 参数访问，不是捕获 |

---

## 4.6.3 变量捕获的规则

变量的捕获发生在闭包**定义时**，因此有以下规则：

### 规则一：被捕获的变量必须在闭包定义时可见

```Cangjie
func f() {
    let x = 99
    func f1() {
        println(x)          // ✅ x 在 f1 定义之前已声明
    }
    let f2 = {
        => println(y)       // undeclared identifier 'y'
    }
    let y = 88
    f1()                    // 输出：99
    f2()
}
```

> [!CAUTION]
> 闭包只能捕获**定义时已经存在**的变量。上例中 `f2` 定义时 `y` 还没声明，因此编译报错。

### 规则二：被捕获的变量必须在闭包定义前完成初始化

```Cangjie
func f() {
    let x: Int64
    func f1() {
        println(x)          // cannot capture variable 'x' before initialization
    }
    x = 99
    f1()
}
```

> [!CAUTION]
> 即使变量已经声明（存在），但如果**尚未完成初始化**，闭包也不能捕获它。

---

## 4.6.4 捕获引用类型变量

如果捕获的变量是**引用类型**（如 `class`），可以修改其**可变实例成员变量**的值：

```Cangjie
class C {
    public var num: Int64 = 0
}

func returnIncrementer(): () -> Unit {
    let c: C = C()              // c 是引用类型

    func incrementer() {
        c.num++                 // ✅ 可以修改引用类型的可变成员
    }

    incrementer
}

main() {
    let f = returnIncrementer()
    f()                         // c.num 增加 1
}
```

> [!NOTE]
> 这里 `c` 本身是 `let`（不可变），但 `c.num` 是 `var`（可变成员）。因为 `c` 是引用类型，`let` 只保证引用关系不变，不保证引用对象内部不可变。

---

## 4.6.5 捕获 var 变量的限制

为了防止捕获了 `var` 声明变量的闭包**逃逸**（即在函数执行完毕后仍然可以在函数外部被调用），这类闭包的使用有严格限制：

> 捕获了 `var` 变量的闭包**只能被调用**，不能作为一等公民使用。

具体来说，不能：

- ❌ 赋值给变量
- ❌ 作为实参传递
- ❌ 作为返回值返回
- ❌ 直接将闭包的名字作为表达式使用

```Cangjie
func f() {
    var x = 1
    let y = 2

    func g() {
        println(x)          // ✅ 捕获了 var 变量 x
    }
    let b = g               // Error, g 不能赋值给变量

    g                       // Error, g 不能作为表达式使用
    g()                     // ✅ g 可以被调用

    g                       // Error, g 不能作为返回值
}
```

> [!WARNING]
> 捕获 `let` 变量的闭包是"安全的"，可以作为一等公民自由使用；但捕获 `var` 变量的闭包因为涉及可变状态，为了安全限制其逃逸。

---

## 4.6.6 捕获的传递性

需要注意的是，捕获具有**传递性**。如果一个函数 `f` 调用了捕获 `var` 变量的函数 `g`，且 `g` 捕获的 `var` 变量**不在**函数 `f` 内定义，那么函数 `f` 同样捕获了 `var` 变量。

**示例 6.1：传递捕获**

`g` 捕获了 `var` 声明的变量 `x`，`f` 调用了 `g`，且 `g` 捕获的 `x` 不在 `f` 内定义，`f` 同样不能作为一等公民使用：

```Cangjie
func h() {
    var x = 1

    func g() {
        x
    }                       // g 捕获了 var 变量 x

    func f() {
        g()                 // f 调用了 g
    }
    return f                // Error：传递捕获了 var 变量 x
}
```

**示例 6.2：非传递捕获**

`g` 捕获了 `var` 声明的变量 `x`，`f` 调用了 `g`。但 `g` 捕获的 `x` **在 `f` 内定义**，`f` 没有捕获其他 `var` 声明的变量，因此 `f` 仍可作为一等公民使用：

```Cangjie
func h() {
    func f() {
        var x = 1
        func g() {
            x
        }                   // g 捕获了 var 变量 x（在 f 内定义）

        g()
    }
    return f                // ✅ OK：x 在 f 内定义，不构成传递捕获
}
```

> [!TIP]
> 简单记忆：一个函数是否"被传染"了 var 捕获，取决于它调用的函数所捕获的 var 变量是否**定义在自身之外**。

---

## 4.6.7 全局变量和静态成员变量

对**全局变量**和**静态成员变量**的访问，**不属于变量捕获**。因此访问了 `var` 修饰的全局变量、静态成员变量的函数或 lambda 仍可作为一等公民使用：
> [!NOTE]
> 此处留作印象，后续学完class再回来学习

```Cangjie
class C {
    public static var a: Int32 = 0
    public static func foo() {
        a++                 // ✅ 静态成员变量，不是捕获
        return a
    }
}

var globalV1 = 0

func countGlobalV1() {
    globalV1++              // ✅ 全局变量，不是捕获
    C.a = 99
    let g = C.foo           // ✅ OK，可作为一等公民
}

func g() {
    let f = countGlobalV1   // ✅ OK
    f()
}
```

> [!NOTE]
> 全局变量和静态成员变量存在于程序的整个生命周期，不依赖某个特定的作用域，因此访问它们不构成"捕获"，也不受逃逸限制。

## 4.6.1 什么是闭包？

一个函数或 lambda **从定义它的静态作用域中捕获了变量**，函数或 lambda 和捕获的变量一起被称为一个**闭包**。

    }
    add                          // add 和 num 一起构成闭包
}

main() {
    let f = returnAddNum()
    println(f(10))               // 输出：20
}
```
