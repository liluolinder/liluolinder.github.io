## 4.5.1 什么是嵌套函数？

定义在源文件**顶层**的函数被称为**全局函数**。定义在**函数体内**的函数被称为**嵌套函数**。

> [!TIP]
> 嵌套函数就像"函数中的函数"——你可以把它想象成在一个大车间里再搭一个小工位，这个小工位只在这个大车间里有效，外面的人看不到它。

```Cangjie
func outer() {           // 外层函数
    func inner() {       // 嵌套函数
        println("我是嵌套函数")
    }
    inner()              // 在外层函数中调用嵌套函数
}
```

---

## 4.5.2 嵌套函数的使用范围

嵌套函数有几个重要的特性：

### 作用域

嵌套函数的作用域**仅限于其所在的外部函数**。外部函数之外无法访问嵌套函数。

```Cangjie
func foo() {
    func nest() {
        println("我是嵌套函数")
    }
    nest()    // ✅ 可以在 foo 内调用
}

main() {
    nest()  // ❌ 错误：在 foo 外部无法访问 nest
}
```

### 访问外部变量

嵌套函数可以**访问外部函数的变量和参数**：

```Cangjie
func foo(x: Int64) {
    let y = 10
    func nestAdd(a: Int64, b: Int64) {
        a + b + x + y    // ✅ 可以访问外部函数的参数 x 和变量 y
    }
    println(nestAdd(1, 2))
}
```

> [!NOTE]
> 反过来是不行的——**外部函数不能直接访问嵌套函数的内部变量**。

### 生命周期

嵌套函数的生命周期与外部函数紧密相关：

- 每次外部函数调用时，嵌套函数被"创建"
- 外部函数执行完毕后，嵌套函数通常被销毁
- **除非**嵌套函数通过返回值或闭包被外部引用，则可以继续存在

---

## 4.5.3 返回嵌套函数

嵌套函数可以作为**返回值**返回，在外部函数之外被调用：

```Cangjie
func foo() {
    func nestAdd(a: Int64, b: Int64) {
        a + b + 3
    }

    println(nestAdd(1, 2))   // 输出：6

    return nestAdd
}

main() {
    let f = foo()            // 拿到返回的嵌套函数
    let x = f(1, 2)          // 在外部调用嵌套函数
    println("result: ${x}")
}
```

程序输出：

```Bash
6
result: 6
```

> [!IMPORTANT]
> 当嵌套函数被返回时，它就变成了一个**闭包**——它"记住"了创建时的环境（包括外部函数的变量）。即使外部函数已经执行完毕，嵌套函数仍然可以访问那些变量。
> 详细的闭包内容将在下节讲解

---

## 4.5.4 使用规则与注意事项

### ✅ 仅在外部函数中使用

嵌套函数的设计初衷是**只在外部函数中使用**。如果某个函数逻辑只在另一个函数内部有用，就可以考虑写成嵌套函数，避免"污染"全局命名空间。

### ⚠️ 避免过度嵌套

过度嵌套会使代码结构变得复杂，难以理解和维护：

```Cangjie
func a() {
    func b() {
        func c() {
            func d() {
                // 这样嵌套太多层，阅读起来很困难
            }
        }
    }
}
```

> [!TIP]
> 通常嵌套**1~2 层**就够了。如果需要更多层嵌套，说明代码可能需要重构。

### ⚠️ 注意闭包的使用

如果嵌套函数被返回并作为闭包使用，需要注意闭包可能会**捕获外部函数的变量**，导致外部函数的变量在外部函数结束后仍然被占用，从而影响内存管理。

## 4.5.1 什么是嵌套函数？

```Cangjie

    func inner() {       // 嵌套函数
        println("我是嵌套函数")
    }
    inner()              // 在外层函数中调用嵌套函数

        println("我是嵌套函数")

    nest()  // ❌ 错误：在 foo 外部无法访问 nest

```Cangjie

```

> 反过来是不行的——**外部函数不能直接访问嵌套函数的内部变量**。

## 4.5.1 什么是嵌套函数？

定义在源文件**顶层**的函数被称为**全局函数**。定义在**函数体内**的函数被称为**嵌套函数**。

> [!TIP]
> 嵌套函数就像"函数中的函数"——你可以把它想象成在一个大车间里再搭一个小工位，这个小工位只在这个大车间里有效，外面的人看不到它。

```Cangjie
func outer() {           // 外层函数
    func inner() {       // 嵌套函数
        println("我是嵌套函数")
    }
    inner()              // 在外层函数中调用嵌套函数
}
```

---

## 4.5.2 嵌套函数的使用范围

嵌套函数有几个重要的特性：

### 作用域

嵌套函数的作用域**仅限于其所在的外部函数**。外部函数之外无法访问嵌套函数。

```Cangjie
func foo() {
    func nest() {
        println("我是嵌套函数")
    }
    nest()    // ✅ 可以在 foo 内调用
}

main() {
    nest()  // ❌ 错误：在 foo 外部无法访问 nest
}
```

### 访问外部变量

嵌套函数可以**访问外部函数的变量和参数**：

```Cangjie
func foo(x: Int64) {
    let y = 10
    func nestAdd(a: Int64, b: Int64) {
        a + b + x + y    // ✅ 可以访问外部函数的参数 x 和变量 y
    }
    println(nestAdd(1, 2))
}
```

> [!NOTE]
> 反过来是不行的——**外部函数不能直接访问嵌套函数的内部变量**。

### 生命周期

嵌套函数的生命周期与外部函数紧密相关：

- 每次外部函数调用时，嵌套函数被"创建"
- 外部函数执行完毕后，嵌套函数通常被销毁
- **除非**嵌套函数通过返回值或闭包被外部引用，则可以继续存在

---

## 4.5.3 返回嵌套函数

嵌套函数可以作为**返回值**返回，在外部函数之外被调用：

```Cangjie
func foo() {
    func nestAdd(a: Int64, b: Int64) {
        a + b + 3
    }

    println(nestAdd(1, 2))   // 输出：6

    return nestAdd
}

main() {
    let f = foo()            // 拿到返回的嵌套函数
    let x = f(1, 2)          // 在外部调用嵌套函数
    println("result: ${x}")
}
```

程序输出：

```Bash
6
result: 6
```

> [!IMPORTANT]
> 当嵌套函数被返回时，它就变成了一个**闭包**——它"记住"了创建时的环境（包括外部函数的变量）。即使外部函数已经执行完毕，嵌套函数仍然可以访问那些变量。
> 详细的闭包内容将在下节讲解

---

## 4.5.4 使用规则与注意事项

### ✅ 仅在外部函数中使用

嵌套函数的设计初衷是**只在外部函数中使用**。如果某个函数逻辑只在另一个函数内部有用，就可以考虑写成嵌套函数，避免"污染"全局命名空间。

### ⚠️ 避免过度嵌套

过度嵌套会使代码结构变得复杂，难以理解和维护：

```Cangjie
func a() {
    func b() {
        func c() {
            func d() {
                // 这样嵌套太多层，阅读起来很困难
            }
        }
    }
}
```

> [!TIP]
> 通常嵌套**1~2 层**就够了。如果需要更多层嵌套，说明代码可能需要重构。

### ⚠️ 注意闭包的使用

如果嵌套函数被返回并作为闭包使用，需要注意闭包可能会**捕获外部函数的变量**，导致外部函数的变量在外部函数结束后仍然被占用，从而影响内存管理。
