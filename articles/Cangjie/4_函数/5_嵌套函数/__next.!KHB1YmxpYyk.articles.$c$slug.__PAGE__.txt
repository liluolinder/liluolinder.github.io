## 4.5.1 什么是嵌套函数？

定义在源文件**顶层**的函数被称为**全局函数**。定义在**函数体内**的函数被称为**嵌套函数**。

> [!TIP]
> 可以这样想象：
> - **全局函数**：是建在"城市中心广场"的建筑，所有人都能进去
> - **嵌套函数**：是建在"某栋大楼内部"的小房间，只有这栋楼里的人能进去

```Cangjie
func outer() {           // 外层函数
    func inner() {       // 嵌套函数（定义在 outer 里面）
        println("我是嵌套函数")
    }
    inner()              // 在外层函数中调用嵌套函数
}
```

### 为什么要用嵌套函数？

嵌套函数的主要用途是**封装只在某个函数内部用到的辅助逻辑**。比如：

- 一个函数比较复杂，你想把内部步骤拆成几个小函数
- 但这些小函数只在当前函数内有用，不想暴露给外面

就像你在家里做饭：切菜、洗菜、炒菜，这些步骤你只需要在厨房里完成，不需要让客厅的客人看到。

```Cangjie
func makeDinner() {
    // 这些辅助步骤只在这个函数内有用
    func washVegetables() {
        println("洗菜中...")
    }
    func cutVegetables() {
        println("切菜中...")
    }
    func cook() {
        println("炒菜中...")
    }

    // 组合调用
    washVegetables()
    cutVegetables()
    cook()
    println("开饭！")
}
```

## 4.5.2 嵌套函数的使用范围

### 作用域（嵌套函数对谁可见？）

嵌套函数的作用域**仅限于其所在的外部函数**。外部函数之外无法访问嵌套函数。

```Cangjie
func foo() {
    func nest() {
        println("我是嵌套函数")
    }
    nest()    // ✅ 可以在 foo 内调用
}

main() {
    nest()  // ❌ 错误：在 foo 外部无法访问 nest
}
```

> [!NOTE]
> 就像"你房间里的东西只有你能拿"——嵌套函数是外部函数的"私人物品"。

### 访问外部变量

嵌套函数可以**访问外部函数的变量和参数**：

```Cangjie
func foo(x: Int64) {
    let y = 10
    func nestAdd(a: Int64, b: Int64) {
        a + b + x + y    // ✅ 可以访问外部函数的参数 x 和变量 y
    }
    println(nestAdd(1, 2))   // 1 + 2 + x + y
}
```

> [!CAUTION]
> 反过来是不行的——**外部函数不能直接访问嵌套函数的内部变量**。
>
> ```Cangjie
> func foo() {
>     func nest() {
>         let secret = 42   // 嵌套函数的内部变量
>     }
>     println(secret)       // ❌ 错误：外部无法访问嵌套函数的变量
> }
> ```
>
> 就像"外面的人看不到你的日记本，但你在房间里可以看到外面的景色"。

### 生命周期

嵌套函数的生命周期与外部函数紧密相关：

| 阶段 | 嵌套函数的状态 |
|------|---------------|
| 外部函数被调用时 | 嵌套函数被"创建" |
| 外部函数执行过程中 | 嵌套函数可以被调用 |
| 外部函数执行完毕 | 嵌套函数通常被**销毁** |
| 特殊情况 | 如果嵌套函数被返回（作为返回值），它可以**继续存活**（见下一节） |

## 4.5.3 返回嵌套函数（让嵌套函数"活"出去）

嵌套函数可以作为**返回值**返回，这样在外部函数结束之后，嵌套函数仍然可以被调用：

```Cangjie
func foo() {
    func nestAdd(a: Int64, b: Int64) {
        a + b + 3
    }

    println(nestAdd(1, 2))   // 在 foo 内部调用：1 + 2 + 3 = 6

    return nestAdd           // 返回嵌套函数本身
}

main() {
    let f = foo()            // 调用 foo，拿到返回的嵌套函数
    let x = f(1, 2)          // 在外部调用嵌套函数：1 + 2 + 3 = 6
    println("result: ${x}")
}
```

程序输出：

```Bash
6
result: 6
```

> [!IMPORTANT]
> 当嵌套函数被返回时，它就变成了一个**闭包**——它"记住"了创建时的环境（包括外部函数的变量）。即使外部函数已经执行完毕，嵌套函数仍然可以访问那些变量。
>
> 详细的闭包内容将在下一节讲解。

## 4.5.4 使用规则与注意事项

### ✅ 仅在外部函数中使用

嵌套函数的设计初衷是**只在外部函数中使用**。如果某个函数逻辑只在另一个函数内部有用，就可以考虑写成嵌套函数，避免"污染"全局空间。

```Cangjie
// ✅ 好：辅助逻辑只在 validateUser 内有用
func validateUser(name: String, age: Int64): Bool {
    func isNotEmpty(s: String): Bool {
        s.size > 0
    }
    func isAdult(a: Int64): Bool {
        a >= 18
    }
    return isNotEmpty(name) && isAdult(age)
}

// ❌ 不好：如果 isNotEmpty 和 isAdult 在其他地方也要用，就应该提到外面去
```

### ⚠️ 避免过度嵌套

过度嵌套会使代码结构变得复杂，难以理解和维护：

```Cangjie
func a() {
    func b() {
        func c() {
            func d() {
                // 这样嵌套太多层，阅读起来很困难
                // 像俄罗斯套娃，一层套一层
            }
        }
    }
}
```

> [!TIP]
> 通常嵌套**1~2 层**就够了。如果发现嵌套了 3 层以上，说明代码可能需要重构。一个实用的判断标准：**你需要在脑子里不停地回想当前在第几层花括号里，就说明嵌套太深了**。

### ⚠️ 注意闭包的内存管理

如果嵌套函数被返回并作为闭包使用，需要注意闭包可能会**捕获外部函数的变量**，导致这些变量在外部函数结束后仍然被占用，从而影响内存管理。

```Cangjie
func createGreeter(greeting: String) {
    func greet(name: String) {
        println("${greeting}, ${name}！")  // 捕获了 outer 的 greeting
    }
    return greet
}

main() {
    let sayHi = createGreeter("你好")
    let sayHello = createGreeter("Hello")
    
    sayHi("小明")     // 输出：你好, 小明！
    sayHello("Xiao")  // 输出：Hello, Xiao！
    
    // 即使 createGreeter 已经执行完毕，greeting 的值还被保留着
}
```

每个闭包都"记住"了自己创建时的 `greeting`，即使 `createGreeter` 执行完毕也不会消失。这很强大，但如果滥用可能导致内存占用过多。

## 4.5.1 什么是嵌套函数？

定义在源文件**顶层**的函数被称为**全局函数**。定义在**函数体内**的函数被称为**嵌套函数**。

> [!TIP]
> 可以这样想象：
> - **全局函数**：是建在"城市中心广场"的建筑，所有人都能进去
> - **嵌套函数**：是建在"某栋大楼内部"的小房间，只有这栋楼里的人能进去

```Cangjie
func outer() {           // 外层函数
    func inner() {       // 嵌套函数（定义在 outer 里面）
        println("我是嵌套函数")
    }
    inner()              // 在外层函数中调用嵌套函数
}
```

嵌套函数的主要用途是**封装只在某个函数内部用到的辅助逻辑**。比如：

- 一个函数比较复杂，你想把内部步骤拆成几个小函数

func makeDinner() {

## 4.5.2 嵌套函数的使用范围

### 作用域（嵌套函数对谁可见？）

嵌套函数的作用域**仅限于其所在的外部函数**。外部函数之外无法访问嵌套函数。

```Cangjie
func foo() {
    func nest() {
        println("我是嵌套函数")
    }
    nest()    // ✅ 可以在 foo 内调用
}

main() {
    nest()  // ❌ 错误：在 foo 外部无法访问 nest
}
```

> [!NOTE]
> 就像"你房间里的东西只有你能拿"——嵌套函数是外部函数的"私人物品"。

### 访问外部变量

嵌套函数可以**访问外部函数的变量和参数**：

```Cangjie
func foo(x: Int64) {
    let y = 10
    func nestAdd(a: Int64, b: Int64) {
        a + b + x + y    // ✅ 可以访问外部函数的参数 x 和变量 y
    }
    println(nestAdd(1, 2))   // 1 + 2 + x + y
}
```

> [!CAUTION]
> 反过来是不行的——**外部函数不能直接访问嵌套函数的内部变量**。
>
> ```Cangjie
> func foo() {
>     func nest() {
>         let secret = 42   // 嵌套函数的内部变量
>     }
>     println(secret)       // ❌ 错误：外部无法访问嵌套函数的变量
> }
> ```
>
> 就像"外面的人看不到你的日记本，但你在房间里可以看到外面的景色"。

### 生命周期

嵌套函数的生命周期与外部函数紧密相关：

| 阶段 | 嵌套函数的状态 |
|------|---------------|
| 外部函数被调用时 | 嵌套函数被"创建" |
| 外部函数执行过程中 | 嵌套函数可以被调用 |
| 外部函数执行完毕 | 嵌套函数通常被**销毁** |
| 特殊情况 | 如果嵌套函数被返回（作为返回值），它可以**继续存活**（见下一节） |

## 4.5.3 返回嵌套函数（让嵌套函数"活"出去）

嵌套函数可以作为**返回值**返回，这样在外部函数结束之后，嵌套函数仍然可以被调用：

```Cangjie
func foo() {
    func nestAdd(a: Int64, b: Int64) {
        a + b + 3
    }

    println(nestAdd(1, 2))   // 在 foo 内部调用：1 + 2 + 3 = 6

    return nestAdd           // 返回嵌套函数本身
}

main() {
    let f = foo()            // 调用 foo，拿到返回的嵌套函数
    let x = f(1, 2)          // 在外部调用嵌套函数：1 + 2 + 3 = 6
    println("result: ${x}")
}
```

程序输出：

```Bash
6
result: 6
```

> [!IMPORTANT]
> 当嵌套函数被返回时，它就变成了一个**闭包**——它"记住"了创建时的环境（包括外部函数的变量）。即使外部函数已经执行完毕，嵌套函数仍然可以访问那些变量。
>
> 详细的闭包内容将在下一节讲解。

## 4.5.4 使用规则与注意事项

### ✅ 仅在外部函数中使用

嵌套函数的设计初衷是**只在外部函数中使用**。如果某个函数逻辑只在另一个函数内部有用，就可以考虑写成嵌套函数，避免"污染"全局空间。

```Cangjie
// ✅ 好：辅助逻辑只在 validateUser 内有用
func validateUser(name: String, age: Int64): Bool {
    func isNotEmpty(s: String): Bool {
        s.size > 0
    }
    func isAdult(a: Int64): Bool {
        a >= 18
    }
    return isNotEmpty(name) && isAdult(age)
}

// ❌ 不好：如果 isNotEmpty 和 isAdult 在其他地方也要用，就应该提到外面去
```

### ⚠️ 避免过度嵌套

过度嵌套会使代码结构变得复杂，难以理解和维护：

```Cangjie
func a() {
    func b() {
        func c() {
            func d() {
                // 这样嵌套太多层，阅读起来很困难
                // 像俄罗斯套娃，一层套一层
            }
        }
    }
}
```

> [!TIP]
> 通常嵌套**1~2 层**就够了。如果发现嵌套了 3 层以上，说明代码可能需要重构。一个实用的判断标准：**你需要在脑子里不停地回想当前在第几层花括号里，就说明嵌套太深了**。

### ⚠️ 注意闭包的内存管理

如果嵌套函数被返回并作为闭包使用，需要注意闭包可能会**捕获外部函数的变量**，导致这些变量在外部函数结束后仍然被占用，从而影响内存管理。

```Cangjie
func createGreeter(greeting: String) {
    func greet(name: String) {
        println("${greeting}, ${name}！")  // 捕获了 outer 的 greeting
    }
    return greet
}

main() {
    let sayHi = createGreeter("你好")
    let sayHello = createGreeter("Hello")
    
    sayHi("小明")     // 输出：你好, 小明！
    sayHello("Xiao")  // 输出：Hello, Xiao！
    
    // 即使 createGreeter 已经执行完毕，greeting 的值还被保留着
}
```

每个闭包都"记住"了自己创建时的 `greeting`，即使 `createGreeter` 执行完毕也不会消失。这很强大，但如果滥用可能导致内存占用过多。
