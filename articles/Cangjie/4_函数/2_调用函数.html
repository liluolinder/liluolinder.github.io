调用函数

学会灵活调用仓颉函数

4.2.1 如何调用函数

上节课我们学会了如何定义函数，现在我们来学习怎么"用"它。

函数定义好之后，通过函数调用来使用它。基本形式为：

Cangjie 
f(arg1, arg2, ..., argn)

其中：

  • f 是要调用的函数的名字
  • arg1 到 argn 是传递给函数的实参（实际参数）
ℹ️NOTE

每个实参的类型必须是对应参数类型的子类型。新手可以简单理解为：传入的值类型要和参数定义的类型一致。

如果函数定义时参数是 Int64 类型，你就不能传一个 String 进去。

最简单的例子

Cangjie 
func sayHello() {
    println("你好！")
}

main() {
    sayHello()       // ✅ 调用无参函数
}
💡TIP

调用函数时，记得一定要加上 ()。

  • sayHello —— 指的是"这个函数本身"（就像指着机器说"这个机器"）
  • sayHello() —— 指的是"调用这个函数"（就像按下了机器的启动按钮）

根据函数定义时参数是非命名参数还是命名参数，调用时传参的方式也不同：

参数类型 调用方式 示例
非命名参数 直接传表达式 add(x, y)
命名参数 使用 参数名: 表达式 add(x, b: y)

4.2.2 非命名参数调用（按位置传参）

如果函数的参数都是非命名参数，调用时按位置依次传参即可：

Cangjie 
func add(a: Int64, b: Int64) {
    return a + b
}

main() {
    let x = 1
    let y = 2
    let r = add(x, y)        // 按位置：第一个值 1 给 a，第二个值 2 给 b
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 3
💡TIP

可以理解为"排队打饭"：第一个来的人（第一个参数值）去第一个窗口（第一个形参），第二个人去第二个窗口。

所以 add(x, y) 中：

  • 第一个值 x=1 → 第一个参数 a
  • 第二个值 y=2 → 第二个参数 b

顺序很重要！ 如果定义时是 add(a, b)，但你传了 add(y, x)，那就变成 a=y、b=x 了。

4.2.3 命名参数调用（按名称传参）

如果函数的参数是命名参数（定义时参数名后带 !），调用时需要指定参数名：

Cangjie 
func add(a: Int64, b!: Int64) {
    return a + b
}

main() {
    let x = 1
    let y = 2
    let r = add(x, b: y)     // a 是非命名参数（按位置），b 是命名参数（按名称）
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 3

命名参数的顺序可以调换

对于多个命名参数，调用时的传参顺序可以和定义时的参数顺序不同：

Cangjie 
func add(a!: Int64, b!: Int64) {
    return a + b
}

main() {
    let x = 1
    let y = 2
    let r = add(b: y, a: x)   // ✅ 命名参数可以调换顺序
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 3
❗IMPORTANT

非命名参数和命名参数混用时：

  1. 非命名参数必须在前，且按位置传参
  2. 命名参数必须在后，按名称传参

就像排队买东西：非命名参数是"按顺序排队"，命名参数是"叫号入场"——号在前的人（非命名）先处理，叫号的人（命名）后处理。

4.2.4 默认参数调用

对于拥有默认值的命名参数，调用时如果没有传实参，那么该参数将使用定义时的默认值：

Cangjie 
func add(a: Int64, b!: Int64 = 2) {  // b 的默认值是 2
    return a + b
}

main() {
    let x = 1
    let r = add(x)             // b 没传，使用默认值 2 → 1 + 2 = 3
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 3

调用时也可以为有默认值的参数传入新值，此时默认值被覆盖：

Cangjie 
func add(a: Int64, b!: Int64 = 2) {
    return a + b
}

main() {
    let x = 1
    let r = add(x, b: 20)      // b 传了新值 20，覆盖默认值 2 → 1 + 20 = 21
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 21
💡TIP

默认值就像"自动售货机的默认选项"——你什么都不按，它就给你默认的商品；你按了别的按钮，就给其他的。

默认值特别适合：

  • 大部分情况下参数值都一样，但偶尔需要改（如：timeout: Int64 = 30）
  • 函数有很多参数，但大多数参数有合理的默认值

4.2.5 流操作符（让代码像流水线一样清晰）

仓颉提供了两个流操作符，用于简化数据的链式处理和函数组合。这是仓颉的一个特色功能。

Pipeline 表达式 `|>`（数据流水线）

💡TIP

|> 的英文名叫 pipeline，中文叫"流水线"——就像工厂里的传送带，原材料从一头进去，经过一道道工序，从另一头出来成品。

当需要对输入数据做一系列处理时，可以用 pipeline 表达式来让代码更直观。

语法形式：

Cangjie 
e1 |> e2

这等价于：

Cangjie 
let v = e1
e2(v)

也就是：把 e1 的结果作为参数，传给 e2 函数。

举个实际例子——先给数组每个元素加 1，再求和：

Cangjie 
// 数组每个元素加 1
func inc(x: Array<Int64>): Array<Int64> {
    let s = x.size
    var i = 0
    for (e in x where i < s) {
        x[i] = e + 1
        i++
    }
    x
}

// 数组元素求和
func sum(y: Array<Int64>): Int64 {
    var s = 0
    for (j in y) {
        s += j
    }
    s
}

main() {
    let arr: Array<Int64> = [1, 3, 5]
    
    // 用 pipeline 的方式（推荐）：
    let res = arr |> inc |> sum      // 先 inc 再 sum —— 从左到右读，像流水线
    println(res)                     // 输出：12
    
    // 不用 pipeline 的方式（嵌套调用，从里往外读，容易晕）：
    // let res = sum(inc(arr))       // 先执行 inc(arr)，再把结果传给 sum
}

执行结果为：

Bash 
12
ℹ️NOTE

arr |> inc |> sum 看起来就像"数据在流动"：数组先流进 inc 处理，结果再流进 sum 求和。而嵌套写法 sum(inc(arr)) 需要从最内层开始往外面读，不够直观。

Composition 表达式 `~>`（函数组合）

composition 表达式表示把两个单参函数组合成一个新函数。

语法形式：

Cangjie 
f ~> g

这等价于：

Cangjie 
{ x => g(f(x)) }

也就是：先对 x 执行 f，再把结果传给 g。

💡TIP

可以把 ~> 想象成"乐高积木"——把两个小积木（函数）拼在一起，变成一块大的积木（新函数）。

示例：

Cangjie 
func f(x: Int64): Float64 {
    Float64(x)          // Int64 → Float64
}

func g(x: Float64): Float64 {
    x + 1.0             // 加 1
}

var fg = f ~> g        // 组合：先把 Int64 转 Float64，再加 1
// fg 等价于 { x: Int64 => g(f(x)) }
ℹ️NOTE

表达式 f ~> g 中，会先对 f 求值，再对 g 求值，最后才组合。组合后的新函数类型是 (Int64) -> Float64。

流操作符与命名参数的配合

流操作符不能与无默认值的命名形参函数直接一起使用——因为无默认值的命名形参函数必须给出命名实参才能调用：

Cangjie 
func f(a!: Int64): Unit {}

var a = 1 |> f // Error

通过 lambda 表达式可以解决（lambda 会在后续文章介绍）：

Cangjie 
func f(a!: Int64): Unit {}

var x = 1 |> { x: Int64 => f(a: x) } // ✅ 用 lambda 传命名参数

同样，当命名参数有默认值时，直接与流操作符一起使用也是错误的：

Cangjie 
func f(a!: Int64 = 2): Unit {}

var a = 1 |> f // Error

但是如果命名形参都有默认值，且函数只需要传入非命名形参，则可以与流操作符一起使用：

Cangjie 
func f(a: Int64, b!: Int64 = 2): Unit {}

var a = 1 |> f // ✅ OK，a 是非命名参数
💡TIP

简单记忆：|> 只能传递一个值。如果函数除了这个值还需要其他命名参数，就得用 lambda 包装一下。

4.2.6 变长参数（一次传任意多个值）

变长参数是一种特殊的函数调用语法糖。当形参的最后一个非命名参数是 Array 类型时，实参中对应位置可以直接传入参数序列来代替手动写 []：

Cangjie 
func sum(arr: Array<Int64>) {
    var total = 0
    for (x in arr) {
        total += x
    }
    return total
}

main() {
    println(sum())          // 0 个参数 → arr = []
    println(sum(1, 2, 3))   // 3 个参数 → arr = [1, 2, 3]
}

程序输出：

Bash 
0
6
💡TIP

变长参数就像"超市的自助称重"——你可以放一个苹果（1 个参数），也可以放一堆苹果（多个参数），称重台都能处理。

如果没有变长参数，你就得自己写 sum([]) 和 sum([1, 2, 3])——多写一对方括号。变长参数帮你省掉了这步。

❗IMPORTANT
  • 只有最后一个非命名参数可以作为变长参数
  • 命名参数不能使用变长参数语法
Cangjie 
func length(arr!: Array<Int64>) {
    return arr.size
}

main() {
    println(length())        // Error: 命名参数不能变长
    println(length(1, 2, 3)) // Error: 命名参数不能变长
}

变长参数的使用范围

变长参数可以用在很多地方：

  • ✅ 全局函数
  • ✅ 静态成员函数
  • ✅ 实例成员函数
  • ✅ 局部函数
  • ✅ 构造函数
  • ✅ 函数变量和 lambda
  • ❌ composition（~>）和 pipeline（|>）不支持
ℹ️NOTE

class 相关内容将在后续讲解，此处先留个印象。

Cangjie 
class Counter {
    var total = 0
    init(data: Array<Int64>) {
        total = data.size
    }
    operator func ()(data: Array<Int64>) {
        total += data.size
    }
}

main() {
    let counter = Counter(1, 2)    // 两个参数 → data = [1, 2]，total = 2
    println(counter.total)         // 输出：2
    counter(3, 4, 5)               // 三个参数 → data = [3, 4, 5]，total = 2 + 3 = 5
    println(counter.total)         // 输出：5
}

变长参数与函数重载

当同时有"普通版本"和"变长版本"的函数时，编译器会优先匹配普通版本：

Cangjie 
func f<T>(x: T) where T <: ToString {
    println("item: ${x}")
}

func f(arr: Array<Int64>) {
    println("array: ${arr}")
}

main() {
    f()       // 无参 → 只能匹配变长版本 → array: []
    f(1)      // 1 个参数 → 优先匹配普通版本 f<T>(x: T) → item: 1
    f(1, 2)   // 2 个参数 → 普通版本不匹配 → 变长版本 → array: [1, 2]
}
Bash 
array: []
item: 1
array: [1, 2]
⚠️WARNING

如果两个变长版本都能匹配，编译器无法决定用哪个，会报错。解决办法是给函数改名或者调整参数设计：

Cangjie 
func f(arr: Array<Int64>) { arr.size }
func f(first: Int64, arr: Array<Int64>) { first + arr.size }
main() {
    println(f(1, 2, 3)) // Error：两个变长函数都能匹配
}

4.2.7 总结

知识点 说明
基本调用 f(arg1, arg2, ...)，按位置传参
非命名参数 按位置传参，顺序必须与定义一致——像"排队打饭"
命名参数 按名称传参，使用 p: e 语法，顺序可调换——像"快递标签"
默认参数 调用时可不传，使用定义时的默认值——像"自动售货机默认选项"
Pipeline |> e1 |> e2 等价于 e2(e1)，数据像流水线一样从左流到右
Composition ~> f ~> g 等价于 { x => g(f(x)) }，像拼乐高一样组合函数
变长参数 最后一个非命名 Array 参数可接收 0 或多个实参——像"自助称重"
重载决议 优先匹配非变长版本，不行才尝试变长版本
