调用函数

学会灵活调用仓颉函数

4.2.1 如何调用函数

函数定义好之后，就可以通过函数调用来使用它了。函数调用的基本形式为：

Cangjie 
f(arg1, arg2, ..., argn)

其中：

  • f 是要调用的函数的名字
  • arg1 到 argn 是传递给函数的实参（实际参数）
ℹ️NOTE

每个实参的类型必须是对应参数类型的子类型（简单理解：类型要匹配或兼容）。

实参可以有 0 个或多个。当实参个数为 0 时，调用方式为 f()：

Cangjie 
func sayHello() {
    println("你好！")
}

main() {
    sayHello()       // ✅ 无参调用
}

根据函数定义时参数是非命名参数还是命名参数，调用时传参的方式也有所不同：

参数类型 调用方式 示例
非命名参数 直接传表达式 add(x, y)
命名参数 使用 参数名: 表达式 add(x, b: y)

4.2.2 非命名参数调用

如果函数的参数都是非命名参数，调用时按位置依次传参即可：

Cangjie 
func add(a: Int64, b: Int64) {
    return a + b
}

main() {
    let x = 1
    let y = 2
    let r = add(x, y)        // 按位置：a=1, b=2
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 3
💡TIP

非命名参数的调用顺序必须与定义时一致。add(x, y) 中第一个值给第一个参数 a，第二个值给第二个参数 b。

4.2.3 命名参数调用

如果函数的参数是命名参数（定义时参数名后带 !），调用时需要指定参数名：

Cangjie 
func add(a: Int64, b!: Int64) {
    return a + b
}

main() {
    let x = 1
    let y = 2
    let r = add(x, b: y)     // a 是非命名参数，b 是命名参数
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 3

命名参数的顺序可以调换

对于多个命名参数，调用时的传参顺序可以和定义时的参数顺序不同：

Cangjie 
func add(a!: Int64, b!: Int64) {
    return a + b
}

main() {
    let x = 1
    let y = 2
    let r = add(b: y, a: x)   // ✅ 命名参数可以调换顺序
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 3
❗IMPORTANT

非命名参数和命名参数混用时，非命名参数必须在前，且按位置传参；命名参数必须在后，按名称传参。

4.2.4 默认参数调用

对于拥有默认值的命名参数，调用时如果没有传实参，那么该参数将使用定义时的默认值：

Cangjie 
func add(a: Int64, b!: Int64 = 2) {
    return a + b
}

main() {
    let x = 1
    let r = add(x)             // b 没传，使用默认值 2
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 3

调用时也可以为有默认值的参数传入新的实参，此时默认值被覆盖：

Cangjie 
func add(a: Int64, b!: Int64 = 2) {
    return a + b
}

main() {
    let x = 1
    let r = add(x, b: 20)      // b 传了新值 20，覆盖默认值
    println("The sum of x and y is ${r}")
}

执行结果为：

Bash 
The sum of x and y is 21
💡TIP

默认值只在调用时没有传参的情况下生效。一旦传了实参，就以实参为准。

4.2.5 流操作符

仓颉提供了两个流操作符，用于简化数据的链式处理和函数组合。

Pipeline 表达式 `|>`

当需要对输入数据做一系列处理时，可以使用 pipeline 表达式来简化代码。语法形式如下：

Cangjie 
e1 |> e2

等价于：

Cangjie 
let v = e1
e2(v)

其中 e2 是函数类型的表达式，e1 的类型必须是 e2 的参数类型的子类型。

看一个实际例子——先给数组每个元素加 1，再求和：

Cangjie 
// 数组每个元素加 1
func inc(x: Array<Int64>): Array<Int64> {
    let s = x.size
    var i = 0
    for (e in x where i < s) {
        x[i] = e + 1
        i++
    }
    x
}

// 数组元素求和
func sum(y: Array<Int64>): Int64 {
    var s = 0
    for (j in y) {
        s += j
    }
    s
}

main() {
    let arr: Array<Int64> = [1, 3, 5]
    let res = arr |> inc |> sum      // 先 inc 再 sum
    println(res)                     // 输出：12
}

执行结果为：

Bash 
12
💡TIP

arr |> inc |> sum 就像流水线——数据从 arr 流入 inc 函数处理，结果再流入 sum 函数，最终得到结果。比嵌套写法 sum(inc(arr)) 更直观。

Composition 表达式 `~>`

composition 表达式表示两个单参函数的组合。语法为：

Cangjie 
f ~> g

等价于：

Cangjie 
{ x => g(f(x)) }

其中 f 和 g 均为只有一个参数的函数。f(x) 的返回类型必须是 g(...) 的参数类型的子类型。

示例 ：基本组合

Cangjie 
func f(x: Int64): Float64 {
    Float64(x)
}

func g(x: Float64): Float64 {
    x
}

var fg = f ~> g // 等价于 g(f(x))
ℹ️NOTE

表达式 f ~> g 中，会先对 f 求值，再对 g 求值，最后才会进行函数的组合。

流操作符与命名参数

流操作符不能与无默认值的命名形参函数直接一起使用，因为无默认值的命名形参函数必须给出命名实参才能调用：

Cangjie 
func f(a!: Int64): Unit {}

var a = 1 |> f // Error

如需使用，可以通过 lambda 表达式传入命名实参：

lambda将在下一章介绍
Cangjie 
func f(a!: Int64): Unit {}

var x = 1 |> { x: Int64 => f(a: x) } // OK

同样，当命名参数有默认值时，直接与流操作符一起使用也是错误的：

Cangjie 
func f(a!: Int64 = 2): Unit {}

var a = 1 |> f // Error

但如果命名形参都有默认值，且函数只需要传入非命名形参，则可以与流操作符一起使用：

Cangjie 
func f(a: Int64, b!: Int64 = 2): Unit {}

var a = 1 |> f // OK

如果同时想为有默认值的命名参数传入其他值，也需要借助 lambda：

Cangjie 
func f(a: Int64, b!: Int64 = 2): Unit {}

var a = 1 |> { x: Int64 => f(x, b: 3) } // OK

4.2.6 变长参数

变长参数是一种特殊的函数调用语法糖。当形参的最后一个非命名参数是 Array 类型时，实参中对应位置可以直接传入参数序列来代替 Array 字面量（参数个数可以是 0 个或多个）：

Cangjie 
func sum(arr: Array<Int64>) {
    var total = 0
    for (x in arr) {
        total += x
    }
    return total
}

main() {
    println(sum())          // 0 个参数
    println(sum(1, 2, 3))   // 3 个参数
}

程序输出：

Bash 
0
6
❗IMPORTANT
  • 只有最后一个非命名参数可以作为变长参数
  • 命名参数不能使用
Cangjie 
func length(arr!: Array<Int64>) {
    return arr.size
}

main() {
    println(length())        // Error, expected 1 argument, found 0
    println(length(1, 2, 3)) // Error, expected 1 argument, found 3
}

变长参数的使用范围

变长参数可以出现在以下场景的调用处：

  • 全局函数
  • 静态成员函数
  • 实例成员函数
  • 局部函数
  • 构造函数
  • 函数变量
  • lambda
  • 函数调用操作符重载
  • 索引操作符重载

不支持以下调用方式：

  • 其他操作符重载
  • composition（~>）
  • pipeline（|>）
ℹ️NOTE

Class类型将在后续讲解，此处留个印象，学完class再回来复习

Cangjie 
class Counter {
    var total = 0
    init(data: Array<Int64>) {
        total = data.size
    }
    operator func ()(data: Array<Int64>) {
        total += data.size
    }
}

main() {
    let counter = Counter(1, 2)    // 变长参数构造
    println(counter.total)         // 输出：2
    counter(3, 4, 5)               // 变长参数调用操作符
    println(counter.total)         // 输出：5
}

程序输出：

Bash 
2
5

函数重载决议

函数重载决议总是会优先考虑不使用变长参数就能匹配的函数，只有在所有函数都不能匹配时，才尝试使用变长参数解析：

💡TIP

函数重载将在后续讲解，此处只需知道同一个函数名有着不同参数/处理方案/返回值

Cangjie 
func f<T>(x: T) where T <: ToString {
    println("item: ${x}")
}

func f(arr: Array<Int64>) {
    println("array: ${arr}")
}

main() {
    f()       // 无参匹配 → array: []
    f(1)      // 单参匹配 f<T>(x: T) → item: 1
    f(1, 2)   // 无法匹配非变长 → 变长匹配 → array: [1, 2]
}

程序输出：

Bash 
array: []
item: 1
array: [1, 2]

当编译器无法决议时会报错：

Cangjie 
func f(arr: Array<Int64>) {
    arr.size
}

func f(first: Int64, arr: Array<Int64>) {
    first + arr.size
}

main() {
    println(f(1, 2, 3)) // Error：两个变长函数都能匹配，无法确定
}
⚠️WARNING

多个变长参数函数可能导致重载决议失败。如果遇到此类报错，考虑给函数换名或调整参数设计。

4.2.7 总结

知识点 说明
基本调用 f(arg1, arg2, ...)，按位置传参
非命名参数 按位置传参，顺序必须与定义一致
命名参数 按名称传参，使用 p: e 语法，顺序可调换
默认参数 调用时可不传，使用定义时的默认值
Pipeline |> e1 |> e2 等价于 e2(e1)，数据流式处理
Composition ~> f ~> g 等价于 { x => g(f(x)) }，函数组合
变长参数 最后一个非命名 Array 参数可接收 0 或多个实参
重载决议 优先匹配非变长版本，不行才尝试变长版本
