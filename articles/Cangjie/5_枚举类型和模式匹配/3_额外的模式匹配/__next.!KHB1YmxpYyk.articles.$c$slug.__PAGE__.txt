## 5.3.1 模式不只是 match 的专利

之前我们学的模式都是在 `match` 表达式中使用的。但仓颉中，模式还可以用在更多地方：

- **变量定义**：等号左侧可以用模式
- **`for-in` 表达式**：`for` 和 `in` 之间可以用模式
- **`if` 和 `while` 的条件**：也可以使用模式

> [!TIP]
> 模式可以理解为"**解构数据**"的工具——不管数据包装得多复杂，模式都能把它拆开，取出你需要的那部分。

### 只能使用 irrefutable 模式

但是，并不是所有的模式都能用在变量定义和 `for-in` 中——只有 **irrefutable（不可反驳）** 的模式才能在这两处被使用。

| 允许的模式 | 说明 |
|-----------|------|
| 通配符模式 `_` | 匹配任意值 |
| 绑定模式 `id` | 匹配并绑定到变量 |
| irrefutable tuple 模式 | 所有子模式都是 irrefutable 的 tuple |
| irrefutable enum 模式 | 只有一个构造器的 enum |

> [!NOTE]
> 为什么只能 irrefutable？因为变量定义和 `for-in` **必须确保匹配成功**——如果模式可能匹配失败（refutable），那变量就没法确定了。

---

## 5.3.2 通配符模式的额外使用

### 变量定义中使用 `_`

通配符模式在变量定义中表示"定义了一个没有名字的变量"，之后无法访问它：

```Cangjie
main() {
    let _ = 100           // 定义了一个"无名变量"
    // println(_)         // ❌ 无法访问 _
}
```

### for-in 中使用 `_`

在 `for-in` 中使用通配符模式，表示"不关心元素的值，只是循环 N 次"：

```Cangjie
main() {
    for (_ in 1..5) {
        println("0")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
0
0
0
0
```

> [!TIP]
> `for (_ in 1..5)` 的意思是"循环 5 次，每次都不管当前是第几个"。

---

## 5.3.3 绑定模式的额外使用

### 变量定义

变量定义本身就是绑定模式最常见的形式：

```Cangjie
main() {
    let x = 100                    // 绑定模式：把 100 绑定到 x
    println("x = ${x}")
}
```

### for-in 中的绑定模式

```Cangjie
main() {
    for (i in 1..5) {              // 绑定模式：依次把元素绑定到 i
        println(i)
    }
}
```

编译执行上述代码，输出结果为：

```Bash
x = 100
1
2
3
4
```

> [!NOTE]
> 你之前一直在用的 `let x = ...` 和 `for (i in ...)`，其实都是"模式匹配"！只是它们是最简单的绑定模式，所以你一直没意识到。

---

## 5.3.4 Tuple 模式的额外使用

### 变量定义中解构 tuple

变量定义时使用 tuple 模式，表示对右侧的 tuple 值进行**解构**，效果相当于一次性定义多个变量：

```Cangjie
main() {
    let (x, y) = (100, 200)        // 解构：x=100, y=200
    println("x = ${x}")
    println("y = ${y}")
}
```

### for-in 中解构 tuple

```Cangjie
main() {
    for ((i, j) in [(1, 2), (3, 4), (5, 6)]) {
        println("Sum = ${i + j}")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
x = 100
y = 200
Sum = 3
Sum = 7
Sum = 11
```

> [!TIP]
> 元组解构的本质就是模式匹配。`(x, y) = (100, 200)` 中，左侧的 `(x, y)` 是一个 tuple 模式，它匹配右侧的 `(100, 200)`，然后把 `100` 绑定到 `x`，`200` 绑定到 `y`。
>
> 在 `for-in` 中，`(i, j)` 依次匹配 `[(1,2), (3,4), (5,6)]` 中的每个元素，逐个解构。

---

## 5.3.5 Enum 模式的额外使用

对于**只有一个构造器**的 enum（irrefutable enum），也可以在变量定义和 `for-in` 中使用。

```Cangjie
enum RedColor {
    Red(Int64)                     // 只有一个构造器
}

main() {
    // 变量定义中使用 enum 模式
    let Red(red) = Red(0)          // 解构：提取出 0 绑定到 red
    println("red = ${red}")

    // for-in 中使用 enum 模式
    for (Red(r) in [Red(10), Red(20), Red(30)]) {
        println("r = ${r}")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
red = 0
r = 10
r = 20
r = 30
```

> [!CAUTION]
> enum 模式在变量定义和 `for-in` 中使用时，要求 enum **只能有一个构造器**（irrefutable）。如果 enum 有多个构造器（如 `Red \| Green \| Blue`），就不能这样用，因为编译器无法确定一定能匹配到正确的构造器。
>
> 对多个构造器的 enum 进行解构，必须使用 `match` 表达式。

---

## 5.3.6 let pattern（模式匹配）

一个 `let pattern` 的构成为 `let pattern <- expression`，其中各字段含义为：

- `pattern`：模式，用于匹配 expression 的值类型和内容
- `<-`：模式匹配操作符
- `expression`：表达式，该表达式求值后，再和模式进行匹配

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

main() {
    let a = MySome(3)
    let c = if (let MySome(b) <- a) {
        1 // 模式匹配成功，c = 1
    } else {
        2
    }
    println(c)
}
```

### 条件中使用 let pattern

`let pattern` 可以和 `if` 配合，在条件中进行模式匹配。多个条件之间可以用 `&&`（逻辑与）或 `||`（逻辑或）组合，也可以和普通 Bool 表达式混合使用。

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

main() {
    let a = MySome(3)
    let d = MySome(1)

    // 两个 let pattern 用 && 连接：两种模式都匹配，条件的值为真
    if (let MySome(e) <- a && let MySome(f) <- d) {
        println("${e} ${f}") // 输出 3 1
    }

    // let pattern 与普通表达式用 && 连接：模式匹配后继续检查条件
    if (let MySome(f) <- d && f > 3) {
        println("${f}")
    } else {
        println("d is MyNone or value of d is less or equal to 3") // 打印该行
    }

    // 两个 let pattern 用 || 连接：至少一个匹配成功即可
    if (let MySome(_) <- a || let MySome(_) <- d) {
        println("at least one of a and d is MySome") // 打印该行
    } else {
        println("both a and d are MyNone")
    }

    // let pattern 与普通表达式用 || 连接
    let g = 3
    if (let MySome(_) <- a || g > 1) {
        println("this") // 打印该行
    } else {
        println("that")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
3 1
d is MyNone or value of d is less or equal to 3
at least one of a and d is MySome
this
```

### 表达式的优先级

`let pattern` 中表达式部分运算符优先级**不能低于 `..` 运算符**，如果用了优先级更低的运算符（如 `as`），需要用 `()` 包裹。

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

func fun(): Object {
    MySome(3)
}

main() {
    // if (let MySome(a) <- fun() as MyOption) {}  // ❌ 解析错误，as 优先级低于 ..
    if (let MySome(a) <- (fun() as MyOption)) {}    // ✅ 正确，用括号提升优先级
    if (let m <- 0..5) {                           // ✅ 正确，.. 优先级满足要求
        println("m = ${m}")
    }
}
```

### 常见错误示例

以下是一些错误的写法，需要注意避免：

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

main() {
    let b = MySome(3)

    // ❌ || 连接的条件中不能使用会绑定变量的 enum 模式
    // if (let MySome(a) <- b || a > 1) {}

    // ❌ && 右侧既不是 let pattern，也不是 Bool 类型的普通表达式
    // if (let MySome(a) <- b && a + 1) {}

    // ❌ a 在绑定它的 pattern 左侧使用（a 还没绑定）
    // if (a > 3 && let MySome(a) <- b) {}

    // ❌ a 只能在 if 分支使用，不能在 else 分支使用
    if (let MySome(a) <- b && a > 3) {
        println("${a} > 3")
    } else {
        // println("${a} < 3") // ❌ a 在 else 中不可见
    }

    // ❌ 使用 && 表示条件检查，而不是 where
    // if (let MySome(a) <- b where a > 3) {}
    
    println("done")
}
```

> [!CAUTION]
> `let pattern` 有一些使用限制，需要特别注意：
> 1. `||` **不能**连接带变量绑定的 enum 模式
> 2. `&&` 右侧必须是 `let pattern` 或 `Bool` 表达式
> 3. 绑定的变量不能在它本身所在的 pattern **左侧**使用
> 4. `let pattern` 中绑定的变量只在 `if` 分支中可见，**不能在 `else` 分支中使用**
> 5. 条件检查用 `&&`，**不能用 `where`**

## 5.3.7 小结

| 使用场景 | 允许的模式 | 示例 |
|---------|-----------|------|
| 变量定义 `let` | irrefutable 模式 | `let x = 100`、`let (a, b) = (1, 2)`、`let Red(r) = Red(0)` |
| `for-in` | irrefutable 模式 | `for (i in 1..5)`、`for ((a,b) in pairs)`、`for (Red(r) in list)` |
| 不可使用的 | refutable 模式 | 常量模式、类型模式等不能在变量定义中使用 |

> [!TIP]
> 简单记忆：**能保证 100% 匹配成功的模式**，才能在变量定义和 `for-in` 中使用。否则就用 `match`。

## 5.3.1 模式不只是 match 的专利

之前我们学的模式都是在 `match` 表达式中使用的。但仓颉中，模式还可以用在更多地方：

- **变量定义**：等号左侧可以用模式
- **`for-in` 表达式**：`for` 和 `in` 之间可以用模式
- **`if` 和 `while` 的条件**：也可以使用模式

> [!TIP]
> 模式可以理解为"**解构数据**"的工具——不管数据包装得多复杂，模式都能把它拆开，取出你需要的那部分。

### 只能使用 irrefutable 模式

但是，并不是所有的模式都能用在变量定义和 `for-in` 中——只有 **irrefutable（不可反驳）** 的模式才能在这两处被使用。

| 允许的模式 | 说明 |
|-----------|------|
| 通配符模式 `_` | 匹配任意值 |
| 绑定模式 `id` | 匹配并绑定到变量 |
| irrefutable tuple 模式 | 所有子模式都是 irrefutable 的 tuple |
| irrefutable enum 模式 | 只有一个构造器的 enum |

> [!NOTE]
> 为什么只能 irrefutable？因为变量定义和 `for-in` **必须确保匹配成功**——如果模式可能匹配失败（refutable），那变量就没法确定了。

---

## 5.3.2 通配符模式的额外使用

### 变量定义中使用 `_`

通配符模式在变量定义中表示"定义了一个没有名字的变量"，之后无法访问它：

```Cangjie
main() {
    let _ = 100           // 定义了一个"无名变量"
    // println(_)         // ❌ 无法访问 _
}
```

### for-in 中使用 `_`

在 `for-in` 中使用通配符模式，表示"不关心元素的值，只是循环 N 次"：

```Cangjie
main() {
    for (_ in 1..5) {
        println("0")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
0
0
0
0
```

> [!TIP]
> `for (_ in 1..5)` 的意思是"循环 5 次，每次都不管当前是第几个"。

---

## 5.3.3 绑定模式的额外使用

### 变量定义

变量定义本身就是绑定模式最常见的形式：

```Cangjie
main() {
    let x = 100                    // 绑定模式：把 100 绑定到 x
    println("x = ${x}")
}
```

### for-in 中的绑定模式

```Cangjie
main() {
    for (i in 1..5) {              // 绑定模式：依次把元素绑定到 i
        println(i)
    }
}
```

编译执行上述代码，输出结果为：

```Bash
x = 100
1
2
3
4
```

> [!NOTE]
> 你之前一直在用的 `let x = ...` 和 `for (i in ...)`，其实都是"模式匹配"！只是它们是最简单的绑定模式，所以你一直没意识到。

## 5.3.4 Tuple 模式的额外使用

### 变量定义中解构 tuple

变量定义时使用 tuple 模式，表示对右侧的 tuple 值进行**解构**，效果相当于一次性定义多个变量：

```Cangjie
main() {
    let (x, y) = (100, 200)        // 解构：x=100, y=200
    println("x = ${x}")
    println("y = ${y}")
}
```

### for-in 中解构 tuple

```Cangjie
main() {
    for ((i, j) in [(1, 2), (3, 4), (5, 6)]) {
        println("Sum = ${i + j}")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
x = 100
y = 200
Sum = 3
Sum = 7
Sum = 11
```

> [!TIP]
> 元组解构的本质就是模式匹配。`(x, y) = (100, 200)` 中，左侧的 `(x, y)` 是一个 tuple 模式，它匹配右侧的 `(100, 200)`，然后把 `100` 绑定到 `x`，`200` 绑定到 `y`。
>
> 在 `for-in` 中，`(i, j)` 依次匹配 `[(1,2), (3,4), (5,6)]` 中的每个元素，逐个解构。

## 5.3.5 Enum 模式的额外使用

对于**只有一个构造器**的 enum（irrefutable enum），也可以在变量定义和 `for-in` 中使用。

```Cangjie
enum RedColor {
    Red(Int64)                     // 只有一个构造器
}

main() {
    // 变量定义中使用 enum 模式
    let Red(red) = Red(0)          // 解构：提取出 0 绑定到 red
    println("red = ${red}")

    // for-in 中使用 enum 模式
    for (Red(r) in [Red(10), Red(20), Red(30)]) {
        println("r = ${r}")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
red = 0
r = 10
r = 20
r = 30
```

> [!CAUTION]
> enum 模式在变量定义和 `for-in` 中使用时，要求 enum **只能有一个构造器**（irrefutable）。如果 enum 有多个构造器（如 `Red \| Green \| Blue`），就不能这样用，因为编译器无法确定一定能匹配到正确的构造器。
>
> 对多个构造器的 enum 进行解构，必须使用 `match` 表达式。

## 5.3.6 let pattern（模式匹配）

一个 `let pattern` 的构成为 `let pattern <- expression`，其中各字段含义为：

- `pattern`：模式，用于匹配 expression 的值类型和内容
- `<-`：模式匹配操作符
- `expression`：表达式，该表达式求值后，再和模式进行匹配

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

main() {
    let a = MySome(3)
    let c = if (let MySome(b) <- a) {
        1 // 模式匹配成功，c = 1
    } else {
        2
    }
    println(c)
}
```

### 条件中使用 let pattern

`let pattern` 可以和 `if` 配合，在条件中进行模式匹配。多个条件之间可以用 `&&`（逻辑与）或 `||`（逻辑或）组合，也可以和普通 Bool 表达式混合使用。

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

main() {
    let a = MySome(3)
    let d = MySome(1)

    // 两个 let pattern 用 && 连接：两种模式都匹配，条件的值为真
    if (let MySome(e) <- a && let MySome(f) <- d) {
        println("${e} ${f}") // 输出 3 1
    }

    // let pattern 与普通表达式用 && 连接：模式匹配后继续检查条件
    if (let MySome(f) <- d && f > 3) {
        println("${f}")
    } else {
        println("d is MyNone or value of d is less or equal to 3") // 打印该行
    }

    // 两个 let pattern 用 || 连接：至少一个匹配成功即可
    if (let MySome(_) <- a || let MySome(_) <- d) {
        println("at least one of a and d is MySome") // 打印该行
    } else {
        println("both a and d are MyNone")
    }

    // let pattern 与普通表达式用 || 连接
    let g = 3
    if (let MySome(_) <- a || g > 1) {
        println("this") // 打印该行
    } else {
        println("that")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
3 1
d is MyNone or value of d is less or equal to 3
at least one of a and d is MySome
this
```

### 表达式的优先级

`let pattern` 中表达式部分运算符优先级**不能低于 `..` 运算符**，如果用了优先级更低的运算符（如 `as`），需要用 `()` 包裹。

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

func fun(): Object {
    MySome(3)
}

main() {
    // if (let MySome(a) <- fun() as MyOption) {}  // ❌ 解析错误，as 优先级低于 ..
    if (let MySome(a) <- (fun() as MyOption)) {}    // ✅ 正确，用括号提升优先级
    if (let m <- 0..5) {                           // ✅ 正确，.. 优先级满足要求
        println("m = ${m}")
    }
}
```

### 常见错误示例

以下是一些错误的写法，需要注意避免：

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

main() {
    let b = MySome(3)

    // ❌ || 连接的条件中不能使用会绑定变量的 enum 模式
    // if (let MySome(a) <- b || a > 1) {}

    // ❌ && 右侧既不是 let pattern，也不是 Bool 类型的普通表达式
    // if (let MySome(a) <- b && a + 1) {}

    // ❌ a 在绑定它的 pattern 左侧使用（a 还没绑定）
    // if (a > 3 && let MySome(a) <- b) {}

    // ❌ a 只能在 if 分支使用，不能在 else 分支使用
    if (let MySome(a) <- b && a > 3) {
        println("${a} > 3")
    } else {
        // println("${a} < 3") // ❌ a 在 else 中不可见
    }

    // ❌ 使用 && 表示条件检查，而不是 where
    // if (let MySome(a) <- b where a > 3) {}
    
    println("done")
}
```

> [!CAUTION]
> `let pattern` 有一些使用限制，需要特别注意：
> 1. `||` **不能**连接带变量绑定的 enum 模式
> 2. `&&` 右侧必须是 `let pattern` 或 `Bool` 表达式
> 3. 绑定的变量不能在它本身所在的 pattern **左侧**使用
> 4. `let pattern` 中绑定的变量只在 `if` 分支中可见，**不能在 `else` 分支中使用**
> 5. 条件检查用 `&&`，**不能用 `where`**

main() {
    let a = MySome(3)
    let c = if (let MySome(b) <- a) {
        1 // 模式匹配成功，c = 1
    } else {
        2
    }
    println(c)
}
```

### 条件中使用 let pattern

`let pattern` 可以和 `if` 配合，在条件中进行模式匹配。多个条件之间可以用 `&&`（逻辑与）或 `||`（逻辑或）组合，也可以和普通 Bool 表达式混合使用。

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

main() {
    let a = MySome(3)
    let d = MySome(1)

    // 两个 let pattern 用 && 连接：两种模式都匹配，条件的值为真
    if (let MySome(e) <- a && let MySome(f) <- d) {
        println("${e} ${f}") // 输出 3 1
    }

    // let pattern 与普通表达式用 && 连接：模式匹配后继续检查条件
    if (let MySome(f) <- d && f > 3) {
        println("${f}")
    } else {
        println("d is MyNone or value of d is less or equal to 3") // 打印该行
    }

    // 两个 let pattern 用 || 连接：至少一个匹配成功即可
    if (let MySome(_) <- a || let MySome(_) <- d) {
        println("at least one of a and d is MySome") // 打印该行
    } else {
        println("both a and d are MyNone")
    }

    // let pattern 与普通表达式用 || 连接
    let g = 3
    if (let MySome(_) <- a || g > 1) {
        println("this") // 打印该行
    } else {
        println("that")
    }
}
```

编译执行上述代码，输出结果为：

```Bash
3 1
d is MyNone or value of d is less or equal to 3
at least one of a and d is MySome
this
```

### 表达式的优先级

`let pattern` 中表达式部分运算符优先级**不能低于 `..` 运算符**，如果用了优先级更低的运算符（如 `as`），需要用 `()` 包裹。

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

func fun(): Object {
    MySome(3)
}

main() {
    // if (let MySome(a) <- fun() as MyOption) {}  // ❌ 解析错误，as 优先级低于 ..
    if (let MySome(a) <- (fun() as MyOption)) {}    // ✅ 正确，用括号提升优先级
    if (let m <- 0..5) {                           // ✅ 正确，.. 优先级满足要求
        println("m = ${m}")
    }
}
```

### 常见错误示例

以下是一些错误的写法，需要注意避免：

```Cangjie
enum MyOption {
    | MySome(Int64) | MyNone
}

main() {
    let b = MySome(3)

    // ❌ || 连接的条件中不能使用会绑定变量的 enum 模式
    // if (let MySome(a) <- b || a > 1) {}

    // ❌ && 右侧既不是 let pattern，也不是 Bool 类型的普通表达式
    // if (let MySome(a) <- b && a + 1) {}

    // ❌ a 在绑定它的 pattern 左侧使用（a 还没绑定）
    // if (a > 3 && let MySome(a) <- b) {}

    // ❌ a 只能在 if 分支使用，不能在 else 分支使用
    if (let MySome(a) <- b && a > 3) {
        println("${a} > 3")
    } else {
        // println("${a} < 3") // ❌ a 在 else 中不可见
    }

    // ❌ 使用 && 表示条件检查，而不是 where
    // if (let MySome(a) <- b where a > 3) {}
    
    println("done")
}
```

> [!CAUTION]
> `let pattern` 有一些使用限制，需要特别注意：
> 1. `||` **不能**连接带变量绑定的 enum 模式
> 2. `&&` 右侧必须是 `let pattern` 或 `Bool` 表达式
> 3. 绑定的变量不能在它本身所在的 pattern **左侧**使用
> 4. `let pattern` 中绑定的变量只在 `if` 分支中可见，**不能在 `else` 分支中使用**
> 5. 条件检查用 `&&`，**不能用 `where`**

## 5.3.7 小结

| 使用场景 | 允许的模式 | 示例 |
|---------|-----------|------|
| 变量定义 `let` | irrefutable 模式 | `let x = 100`、`let (a, b) = (1, 2)`、`let Red(r) = Red(0)` |
| `for-in` | irrefutable 模式 | `for (i in 1..5)`、`for ((a,b) in pairs)`、`for (Red(r) in list)` |
| 不可使用的 | refutable 模式 | 常量模式、类型模式等不能在变量定义中使用 |
