## 5.1.1 什么是枚举类型？

**枚举类型（enum）** 是一种通过**列出所有可能的取值**来定义的类型。

> [!TIP]
> 生活中的类比：**星期几**就是一个天然的枚举：
> - 星期一、星期二、星期三……星期日
> - 一共就 7 种可能，不可能有"星期八"
>
> 用代码来定义就是：
> ```Cangjie
> enum Weekday {
>     | Monday | Tuesday | Wednesday | Thursday | Friday | Saturday | Sunday
> }
> ```

在很多语言中都有 enum 类型，但不同语言中的 enum 表达力有所不同。仓颉中的 enum 可以理解为函数式编程语言中的**代数数据类型（Algebraic Data Types）**，比传统语言的枚举更强大——每个"选项"还可以携带自己的数据。

---

## 5.1.2 定义 enum

定义 enum 时需要把它所有可能的取值一一列出，称这些值为 enum 的**构造器**（constructor）。

### 基本语法

```Cangjie
enum 类型名 {
    | 构造器1 | 构造器2 | 构造器3
}
```

- 以关键字 `enum` 开头
- 接着是 enum 的名字
- 之后是定义在一对花括号中的 enum 体
- enum 体中定义了若干**构造器**，多个构造器之间使用 `|` 分隔（第一个构造器之前的 `|` 是可选的）
- 构造器可以**有名字**，也可以没有名字（`...`）

### 每个 enum 至少有一个有名字的构造器

有名字的构造器可以：
- **无参数**：就像普通的枚举值
- **有参数**：可以携带若干个参数

### 示例：RGB 颜色

如下示例代码定义了一个名为 `RGBColor` 的 enum 类型，它有 3 个构造器：`Red`、`Green` 和 `Blue`，分别表示 RGB 色彩模式中的红色、绿色和蓝色。每个构造器有一个 `UInt8` 类型的参数，用来表示每个颜色的亮度级别。

```Cangjie
enum RGBColor {
    | Red(UInt8) | Green(UInt8) | Blue(UInt8)
}
```

> [!TIP]
> 枚举参数的获取参考后续的模式匹配

### 支持同名构造器（参数不同即可）

仓颉支持同一个 enum 中定义多个同名构造器，但是要求这些构造器的参数个数不同（认为没有参数的构造器的参数个数等于 0）：

```Cangjie
enum RGBColor {
    | Red
    | Green
    | Blue
    | Red(UInt8)
    | Green(UInt8)
    | Blue(UInt8)
}
```

> [!NOTE]
> 上例中 `Red` 有两个版本：一个无参 `Red`，一个有参 `Red(UInt8)`。编译器会根据使用时传入的参数来决定用的是哪个。

### `...` 构造器（Non-exhaustive enum）

每个 enum 中最多只有一个没有名字的 `...` 构造器，且 `...` **只能是最后一个构造器**。拥有 `...` 构造器的 enum 称为 **non-exhaustive enum**（非穷尽枚举）。

由于没有名字，这个构造器不能被直接匹配，在解构时需要使用可匹配所有构造器的模式，如通配符模式 `_` 或绑定模式。

```Cangjie
enum T {
    | Red | Green | Blue | ...
}
```

> [!TIP]
> `...` 表示"还有其他的可能，但这里没列全"。这在需要扩展的场景下很有用。

### 递归定义 enum

enum 支持递归定义。例如，下面的例子中使用 enum 定义了一种**表达式** `Expr`，此表达式只能有 3 种形式：
- 单独的一个数字 `Num`（携带一个 `Int64` 类型的参数）
- 加法表达式 `Add`（携带两个 `Expr` 类型的参数）
- 减法表达式 `Sub`（携带两个 `Expr` 类型的参数）

对于 `Add` 和 `Sub` 这两个构造器，其参数中递归地使用到了 `Expr` 自身。

```Cangjie
enum Expr {
    | Num(Int64)
    | Add(Expr, Expr)
    | Sub(Expr, Expr)
}
```

### 在 enum 中定义函数

在 enum 体中还可以定义成员函数、操作符函数和成员属性，但是要求构造器、成员函数、成员属性之间**不能重名**。

例如，下面的例子在 `RGBColor` 中定义了一个名为 `printType` 的函数：

```Cangjie
enum RGBColor {
    | Red | Green | Blue

    public static func printType() {
        print("RGBColor")
    }
}
```

> [!NOTE]
> 关于操作符重载和属性的内容，会在后续章节中介绍。

### 定义位置限制

> [!CAUTION]
> enum **只能**定义在源文件的**顶层作用域**，不能定义在函数内部或类内部。

---

## 5.1.3 使用 enum

定义了 enum 类型之后，就可以创建此类型的实例（即 enum 值）。enum 值**只能取 ** enum 类型定义中的某一个构造器。

### 构造 enum 值

enum **没有构造函数**，可以通过 `类型名.构造器`，或者直接使用构造器的方式来构造一个 enum 值（对于有参构造器，需要传实参）。

```Cangjie
enum RGBColor {
    | Red | Green | Blue(UInt8)
}

main() {
    let r = RGBColor.Red       // 使用类型名.构造器
    let g = Green              // 直接使用构造器
    let b = Blue(100)          // 有参构造器，传入参数
}
```

### 名字冲突时的处理

当省略类型名时，enum 构造器的名字可能和类型名、变量名、函数名发生冲突。此时**必须加上 enum 类型名**来使用 enum 构造器，否则只会选择同名的类型、变量、函数定义。

```Cangjie
let Red = 1                    // 变量 Red

func Green(g: UInt8) {         // 函数 Green
    return g
}

enum RGBColor {
    | Red | Green(UInt8) | Blue(UInt8)
}

let r1 = Red                   // 选了变量 let Red
let r2 = RGBColor.Red          // ✅ 加上类型名，才是 enum 构造器

let g1 = Green(100)            // 选了函数 func Green
let g2 = RGBColor.Green(100)   // ✅ 加上类型名，才是 enum 构造器

let b = Blue(100)              // ✅ 没有冲突，可以唯一识别为 enum 构造器
```

```Cangjie
class Blue {}

enum RGBColor {
    | Red | Green(UInt8) | Blue(UInt8)
}

let r = Red                    // ✅ 没有冲突，可以唯一识别

let g = Green(100)             // ✅ 没有冲突

let b = Blue(100)              // 选了 class Blue 的构造器，报错
```

> [!WARNING]
> 如果 enum 构造器的名字和另一个类名相同，且都有参数，编译器会优先选择类的构造器，导致报错。这时必须使用 `RGBColor.Blue(100)` 来明确指定。

## 5.1.1 什么是枚举类型？

**枚举类型（enum）** 是一种通过**列出所有可能的取值**来定义的类型。

> [!TIP]
> 生活中的类比：**星期几**就是一个天然的枚举：
> - 星期一、星期二、星期三……星期日
> - 一共就 7 种可能，不可能有"星期八"
>
> 用代码来定义就是：
> ```Cangjie
> enum Weekday {
>     | Monday | Tuesday | Wednesday | Thursday | Friday | Saturday | Sunday
> }
> ```

在很多语言中都有 enum 类型，但不同语言中的 enum 表达力有所不同。仓颉中的 enum 可以理解为函数式编程语言中的**代数数据类型（Algebraic Data Types）**，比传统语言的枚举更强大——每个"选项"还可以携带自己的数据。

---

## 5.1.2 定义 enum

定义 enum 时需要把它所有可能的取值一一列出，称这些值为 enum 的**构造器**（constructor）。

### 基本语法

```Cangjie
enum 类型名 {
    | 构造器1 | 构造器2 | 构造器3
}
```

- 以关键字 `enum` 开头
- 接着是 enum 的名字
- 之后是定义在一对花括号中的 enum 体
- enum 体中定义了若干**构造器**，多个构造器之间使用 `|` 分隔（第一个构造器之前的 `|` 是可选的）
- 构造器可以**有名字**，也可以没有名字（`...`）

### 每个 enum 至少有一个有名字的构造器

有名字的构造器可以：
- **无参数**：就像普通的枚举值
- **有参数**：可以携带若干个参数

### 示例：RGB 颜色

如下示例代码定义了一个名为 `RGBColor` 的 enum 类型，它有 3 个构造器：`Red`、`Green` 和 `Blue`，分别表示 RGB 色彩模式中的红色、绿色和蓝色。每个构造器有一个 `UInt8` 类型的参数，用来表示每个颜色的亮度级别。

```Cangjie
enum RGBColor {
    | Red(UInt8) | Green(UInt8) | Blue(UInt8)
}
```

> [!TIP]
> 枚举参数的获取参考后续的模式匹配

### 支持同名构造器（参数不同即可）

仓颉支持同一个 enum 中定义多个同名构造器，但是要求这些构造器的参数个数不同（认为没有参数的构造器的参数个数等于 0）：

```Cangjie
enum RGBColor {
    | Red
    | Green
    | Blue
    | Red(UInt8)
    | Green(UInt8)
    | Blue(UInt8)
}
```

> [!NOTE]
> 上例中 `Red` 有两个版本：一个无参 `Red`，一个有参 `Red(UInt8)`。编译器会根据使用时传入的参数来决定用的是哪个。

### `...` 构造器（Non-exhaustive enum）

每个 enum 中最多只有一个没有名字的 `...` 构造器，且 `...` **只能是最后一个构造器**。拥有 `...` 构造器的 enum 称为 **non-exhaustive enum**（非穷尽枚举）。

由于没有名字，这个构造器不能被直接匹配，在解构时需要使用可匹配所有构造器的模式，如通配符模式 `_` 或绑定模式。

```Cangjie
enum T {
    | Red | Green | Blue | ...
}
```

> [!TIP]
> `...` 表示"还有其他的可能，但这里没列全"。这在需要扩展的场景下很有用。

### 递归定义 enum

enum 支持递归定义。例如，下面的例子中使用 enum 定义了一种**表达式** `Expr`，此表达式只能有 3 种形式：
- 单独的一个数字 `Num`（携带一个 `Int64` 类型的参数）
- 加法表达式 `Add`（携带两个 `Expr` 类型的参数）
- 减法表达式 `Sub`（携带两个 `Expr` 类型的参数）

对于 `Add` 和 `Sub` 这两个构造器，其参数中递归地使用到了 `Expr` 自身。

```Cangjie
enum Expr {
    | Num(Int64)
    | Add(Expr, Expr)
    | Sub(Expr, Expr)
}
```

### 在 enum 中定义函数

在 enum 体中还可以定义成员函数、操作符函数和成员属性，但是要求构造器、成员函数、成员属性之间**不能重名**。

例如，下面的例子在 `RGBColor` 中定义了一个名为 `printType` 的函数：

```Cangjie
enum RGBColor {
    | Red | Green | Blue

    public static func printType() {
        print("RGBColor")
    }
}
```

> [!NOTE]
> 关于操作符重载和属性的内容，会在后续章节中介绍。

### 定义位置限制

> [!CAUTION]
> enum **只能**定义在源文件的**顶层作用域**，不能定义在函数内部或类内部。

---

## 5.1.3 使用 enum

定义了 enum 类型之后，就可以创建此类型的实例（即 enum 值）。enum 值**只能取 ** enum 类型定义中的某一个构造器。

### 构造 enum 值

enum **没有构造函数**，可以通过 `类型名.构造器`，或者直接使用构造器的方式来构造一个 enum 值（对于有参构造器，需要传实参）。

```Cangjie
enum RGBColor {
    | Red | Green | Blue(UInt8)
}

main() {
    let r = RGBColor.Red       // 使用类型名.构造器
    let g = Green              // 直接使用构造器
    let b = Blue(100)          // 有参构造器，传入参数
}
```

### 名字冲突时的处理

当省略类型名时，enum 构造器的名字可能和类型名、变量名、函数名发生冲突。此时**必须加上 enum 类型名**来使用 enum 构造器，否则只会选择同名的类型、变量、函数定义。

```Cangjie
let Red = 1                    // 变量 Red

func Green(g: UInt8) {         // 函数 Green
    return g
}

enum RGBColor {
    | Red | Green(UInt8) | Blue(UInt8)
}

let r1 = Red                   // 选了变量 let Red
let r2 = RGBColor.Red          // ✅ 加上类型名，才是 enum 构造器

let g1 = Green(100)            // 选了函数 func Green
let g2 = RGBColor.Green(100)   // ✅ 加上类型名，才是 enum 构造器

let b = Blue(100)              // ✅ 没有冲突，可以唯一识别为 enum 构造器
```

```Cangjie
class Blue {}

enum RGBColor {
    | Red | Green(UInt8) | Blue(UInt8)
}

let r = Red                    // ✅ 没有冲突，可以唯一识别

let g = Green(100)             // ✅ 没有冲突

let b = Blue(100)              // 选了 class Blue 的构造器，报错
```

> [!WARNING]
> 如果 enum 构造器的名字和另一个类名相同，且都有参数，编译器会优先选择类的构造器，导致报错。这时必须使用 `RGBColor.Blue(100)` 来明确指定。
